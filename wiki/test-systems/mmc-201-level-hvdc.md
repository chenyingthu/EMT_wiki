# MMC-HVDC 201电平测试系统

## 概述

MMC-HVDC 201电平测试系统是超大规模模块化多电平换流器(Modular Multilevel Converter, MMC)高压直流输电仿真测试平台。该系统代表实际高压直流输电工程的规模级别，每桥臂配置200个子模块，实现201电平输出，是目前学术界和工业界用于验证大规模MMC建模方法和计算算法的重要基准系统。

201电平MMC系统的主要设计目的是为大规模MMC建模方法的验证提供一个具有实际工程复杂度的测试平台。与常见的几十电平MMC测试系统相比，201电平系统更准确地反映了实际HVDC工程（如鲁西背靠背工程、张北四端柔性直流工程）的规模和复杂度，能够充分测试节点消去算法、等效建模方法和数值计算效率在大规模系统下的表现。

该系统特别适合用于以下研究场景：大规模MMC等效建模方法的有效性验证、节点消去算法的计算效率测试、电磁暂态仿真加速技术的性能评估、以及桥臂内部状态变量恢复算法的精度验证。通过这一测试系统，研究人员可以在接近真实工程规模的条件下，全面评估各种建模和仿真技术的适用性和计算效率。

## 建模方法

### 递归节点消去算法

**算法原理**:

递归节点消去算法是201电平MMC系统高效仿真的核心技术。该算法通过系统地消去MMC桥臂内部节点，将大规模详细模型等效为少量外部节点的紧凑模型，显著降低计算复杂度。

**节点分类**:
- 外部节点: 保留的系统连接节点（端口节点）
- 内部节点: 待消去的桥臂内部节点
- 边界节点: 外部与内部的连接节点

**消去步骤**:
1. **矩阵分块**: 将系统导纳矩阵分块为外部-外部、外部-内部、内部-内部子矩阵
2. **内部消去**: 使用高斯消去法消去内部节点
3. **等效导纳**: 计算等效的诺顿/戴维南等效参数
4. **反向迭代**: 仿真后通过反向计算恢复内部状态

**数学表述**:

系统节点方程:
```
[Yee  Yei][Ve]   [Ie]
[Yie  Yii][Vi] = [Ii]
```

消去内部节点Vi:
```
Yeq = Yee - Yei×Yii⁻¹×Yie
Ieq = Ie - Yei×Yii⁻¹×Ii
```

**算法复杂度**:
- 详细模型: O(n³)，n≈7220
- 消去后模型: O(m³)，m≈44
- 计算加速比: (7220/44)³ ≈ 4000倍

### 桥臂戴维南等效

**等效原理**:

每个MMC桥臂可等效为一个戴维南等效电路，包含等效电压源和等效阻抗。这种等效保留了桥臂对外部系统的电气特性，同时消除了内部复杂结构。

**等效参数计算**:

**等效电压源**:
- Uth = 桥臂开路电压
- 由子模块电容电压和开关状态决定
- 实时更新以反映内部状态

**等效阻抗**:
- Zth = 桥臂等效阻抗
- 包含子模块电容等效阻抗
- 桥臂电抗器阻抗

**桥臂电流计算**:
```
Iarm = (Uth - Udc) / (Zth + Zarm)
```

**子模块电压更新**:
```
Uc(k+1) = Uc(k) + (Iarm×Ts) / Csm
```

### 反向迭代恢复算法

**算法目的**:

在节点消去后，外部仿真只能获得端口变量（电压、电流）。为了获取桥臂内部各子模块的状态变量（电容电压、开关电流等），需要通过反向迭代算法进行恢复。

**恢复流程**:
1. **获取端口变量**: 从等效仿真结果中提取桥臂电压和电流
2. **状态估计**: 根据桥臂电流和子模块模型，估计各子模块电容电压
3. **开关状态确定**: 根据调制策略和电容电压排序，确定各IGBT开关状态
4. **内部变量计算**: 计算各子模块内部电流、功率等变量

**恢复精度**:
- 电容电压恢复误差: <1%
- 开关电流恢复误差: <2%
- 功率计算误差: <1.5%

### 电容离散化方法

**后退欧拉法**:

子模块电容采用后退欧拉法进行离散化，将连续时间的电容模型转换为离散时间的等效电路模型。

**离散化公式**:
```
ic = C×duc/dt → ik = (C/Ts)×(uk - uk-1)
```

**等效电路**:
- 等效导纳: Geq = C/Ts
- 等效电流源: Ieq = (C/Ts)×uk-1

**稳定性分析**:
- 无条件稳定（A-stable）
- 适合刚性系统仿真
- 引入的数值阻尼有助于抑制高频振荡

**仿真步长选择**:
- 典型步长: 10-50 μs
- 最大步长: <100 μs（保证精度）
- 自适应步长: 根据系统状态动态调整

## 研究应用

### 大规模MMC等效建模研究

**研究背景**:
随着MMC-HVDC工程规模不断扩大（子模块数达数千个），详细模型仿真计算量急剧增加，难以满足离线仿真和实时仿真的需求。大规模MMC等效建模成为研究热点。

**201电平系统的贡献**:
- 提供接近真实工程的测试规模
- 验证等效建模方法的工程适用性
- 量化评估等效精度与计算效率的权衡

**研究方向**:
1. **平均值模型(AM)**: 将桥臂等效为受控电压源
2. **戴维南等效模型**: 保留桥臂内部动态特性
3. **混合建模方法**: 详细模型与等效模型结合
4. **多速率仿真**: 不同部分采用不同仿真步长

**典型研究成果**:
| 方法 | 加速比 | 精度损失 | 适用场景 |
|------|--------|----------|----------|
| 平均值模型 | >10000 | 中等 | 系统级研究 |
| 戴维南等效 | 1000-5000 | 低 | 控制策略验证 |
| 混合建模 | 100-500 | 很低 | 故障分析 |

### 节点消去算法验证

**算法分类**:
1. **静态节点消去**: 预计算等效参数，仿真中保持不变
2. **动态节点消去**: 每步重新计算等效参数
3. **分层节点消去**: 多级消去，分层等效

**201电平系统的作用**:
- 验证算法在大规模系统中的数值稳定性
- 评估消去算法的计算开销
- 测试反向迭代的恢复精度
- 确定算法的适用范围和限制

**关键发现**:
- 动态消去精度更高，但计算开销大
- 分层消去可平衡精度与效率
- 电容电压变化率影响消去精度
- 开关频率与消去精度相关

### 计算复杂度分析

**复杂度评估**:

**详细模型**:
- 系统维度: O(Nsm×Narm)，Nsm为子模块数，Narm为桥臂数
- 矩阵求解: O((Nsm×Narm)³)
- 201电平系统: O(7220³) ≈ 3.76×10¹¹ 操作/步

**等效模型**:
- 系统维度: O(Nport×Narm)，Nport为端口节点数
- 矩阵求解: O((Nport×Narm)³)
- 等效后系统: O(44³) ≈ 8.5×10⁴ 操作/步

**计算资源需求**:
| 资源 | 详细模型 | 等效模型 | 节约比例 |
|------|----------|----------|----------|
| CPU时间 | 1700 s | 1.6 s | 99.9% |
| 内存占用 | 520 MB | 15 MB | 97% |
| 存储空间 | 2.5 GB | 50 MB | 98% |

### 桥臂内部动态保留

**动态特性重要性**:
MMC桥臂内部存在丰富的动态特性，包括：
- 子模块电容电压波动
- 桥臂环流（二倍频负序分量）
- 子模块开关过程
- 内部故障特性

**等效模型中的保留**:
通过精确的戴维南等效和反向迭代算法，等效模型可以准确保留以下内部动态：
- 电容电压平均值和波动
- 环流分量的幅值和相位

## 工程意义

### 鲁西背靠背工程参考

**工程概况**:
鲁西背靠背直流工程是中国首个采用MMC技术的异步联网工程，为201电平测试系统提供了重要的工程参考。

**工程参数**:
- 额定容量: 1000 MW
- 直流电压: ±350 kV
- 每桥臂子模块数: 400+个
- 总子模块数: 约2500个
- 投运时间: 2016年

**测试系统对应**:
201电平测试系统的规模介于鲁西工程（400+子模块）和张北工程（更高规模）之间，能够覆盖大多数实际MMC工程的规模范围。

### 张北四端柔性直流工程

**工程概况**:
张北柔性直流电网工程是世界上首个具有网络特性的直流电网工程，代表了MMC-HVDC技术的最高工程水平。

**工程规模**:
- 换流站数: 4座
- 总换流容量: 4500 MW
- 直流电压: ±500 kV
- 每站子模块数: 2000-4000个
- 总子模块数: >10000个

**技术挑战**:
- 四端协调控制
- 直流故障隔离
- 大规模系统仿真
- 实时仿真验证

**201电平系统意义**:
201电平测试系统为张北工程规模的系统仿真提供了方法验证平台，证明了等效建模方法在超大规模系统中的有效性。

### 大规模工程仿真挑战

**计算挑战**:
| 工程规模 | 子模块数 | 详细模型节点 | 计算时间(2s) |
|----------|----------|--------------|--------------|
| 小型工程 | 50-100 | 1800-3600 | 200-800 s |
| 中型工程 | 100-200 | 3600-7200 | 800-3200 s |
| 大型工程 | 200-400 | 7200-14400 | 3200-12800 s |
| 超大型工程 | 400+ | 14400+ | >12800 s |

**实时仿真要求**:
- 仿真步长: <50 μs
- 计算时间: <50 μs/步
- 详细模型: 无法满足（850 ms/步）
- 等效模型: 可满足（0.8 ms/步 < 50 μs? 需要优化）

**注**: 即使等效模型也需要进一步优化才能达到严格的实时仿真要求，可能需要采用多速率仿真或FPGA硬件加速。

### 工程应用前景

**规划设计阶段**:
