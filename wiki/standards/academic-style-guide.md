# 学术表达规范

本文档规定 EMT Wiki 的学术表达标准，与 `academic-polish-guidelines.md`（语言润色、PEEL结构）和 `symbol-table.md`（符号规范）配合使用。本指南侧重**可引用性**——使 wiki 页面达到接近教科书的技术文档水准。

---

## 1. 语言规范

### 1.1 陈述语气对照

| 场景 | ✅ 正确 | ❌ 避免 |
|------|---------|--------|
| 论文引用 | "X 等 (2024) 提出了..." | "论文提出..."（拟人化）|
| 方法描述 | "该方法适用于..." | "该方法可以完美解决..."（过度承诺）|
| 对比 | "与 A 相比，B 的误差减小了 X%" | "B 明显优于 A"（模糊比较）|
| 边界 | "验证限于 X 算例" | "该方法通用有效"（无边界）|
| 量化 | "加速比为 3.47×（5 核配置）" | "显著加速"（模糊量词）|
| 推导 | "由式 (12) 可得..." | "显然..."（缺乏推导线索）|
| 局限 | "当频率 > 2 kHz 时误差增大" | "该方法存在一定局限"（模糊）|
| 目的 | "为分析 X，引入 Y" | "为了大家理解..." |

### 1.2 强断言降级表

实体页（组织/学者/工具介绍）必须遵循此降级规则：

| ❌ 不可接受 | ✅ 可接受 | 说明 |
|------------|---------|------|
| 全球领先 / 国际领先 | 有研究成果 / 用于 | 删除无比较基准的领先声明 |
| 最经典 | 经典 | 删除最高级 |
| 突破 / 重大突破 | 贡献 / 技术贡献 | 删除颠覆性暗示 |
| 泰斗 / 权威 | 学者 / 研究者 | 删除个人崇拜式表述 |
| 广泛使用 / 被广泛认可 | 用于 / 被应用于 | 除非有引用支持 |
| 彻底解决 / 完全克服 | 能够处理 / 适用于 | 删除绝对化表述 |
| 最重要的 / 最具影响力的 | 主要的 / 有影响力的 / 重要之一 | 删除最高级 |
| 核心角色 / 不可或缺 | 常用于 / 参与 | 除非有数据证明 |
| 金标准 / 黄金标准 | 参考标准 / 被用作基准 | 删除市场宣传用语 |
| 独具特色 / 独一无二 | 具有特征 / 区别于 | 删除排他性声明 |

**原则**：实体页介绍应基于可验证事实，避免广告化语言。所有正面陈述需有来源支撑。

### 1.3 量化要求

- 所有性能比较必须有**数值证据**（百分比、时间、误差指标）
- 使用 `>`, `<`, `≈`, `×` 等精确符号替代模糊描述
- 无法获取数值时明确标注"原文未报告可核验的数值结果"
- **禁止编造数字**

## 2. 引用规范

### 2.1 文内引用格式

| 类型 | 格式 | 示例 |
|------|------|------|
| 单篇 | （作者, 年份） | （Gustavsen 等, 1999）|
| 多篇 | （作者1, 年份; 作者2, 年份） | （Noda 等, 2014; Watson, 2003）|
| 多人 | 作者 等（年份） | Gustavsen 等（1999）提出了... |
| 机构 | 机构名（年份） | CIGRE（2014）发布了... |
| 间接引用 | 参见 作者（年份） | 详细推导参见 Mahseredjian（2010）|

### 2.2 规范示例

```markdown
矢量拟合（Vector Fitting）由 Gustavsen 等（1999）提出，
用于将频域采样数据拟合为有理函数。后续改进包括
Martinez 等（2002）引入的迭代极点移位策略。

仿真验证采用 CIGRE 提出的 HVDC 基准系统
（CIGRE, 2014）。
```

### 2.3 来源页链接

对于 wiki 中已有对应来源页的文献：

```markdown
- [[gustavsen-1999-vector-fitting]] — 矢量拟合算法原始论文
- [[noda-2014-2s-dirk]] — 2S-DIRK 在 EMT 中的实现
```

### 2.4 引用标注位置

- **方法/模型页面**：在技术描述后标注来源，而非在标题处
- **主题页面**：综述性陈述无需逐句标注，但关键结论和数据需要
- **实体页面**：所有关于实体的具体事实需标注来源

## 3. 数学公式规范

### 3.1 格式

- 所有公式必须使用 LaTeX：块级 `$$...$$`，行内 `$...$`
- 禁止使用图片代替公式
- 公式中的变量首次出现需解释其含义

### 3.2 编号

关键公式需编号，编号格式为 `(n)`：

```markdown
$$
\mathbf{Y}_n \mathbf{v}_n = \mathbf{i}_n \tag{1}
$$
```

或手动：

```markdown
$$
\mathbf{Y}_n \mathbf{v}_n = \mathbf{i}_n \qquad (1)
$$
```

引用编号："由式 (1) 可得节点电压..."

### 3.3 推导连续性

- 关键推导步骤不可跳跃
- 相邻公式间应有简短文字说明转换逻辑
- 较长的推导应标出中间步骤编号

```markdown
由梯形积分公式：
$$
v_{n+1} = v_n + \frac{h}{2}(f_n + f_{n+1})
$$

代入 $f = \frac{1}{L}v$ 并整理得：
$$
i_L(t+h) = G_{\text{eq}} v_L(t+h) + I_{\text{hist}}
$$

其中 $G_{\text{eq}} = h/(2L)$，$I_{\text{hist}} = i_L(t) + G_{\text{eq}} v_L(t)$。
```

### 3.4 符号首现标注

每个页面中，符号首次出现时需用以下格式之一解释：

- 行内说明："其中 $h$ 为仿真步长"
- 列表说明："$h$：仿真步长（单位：s）"
- 表格式说明（多个符号时优先）：

```markdown
| 符号 | 含义 | 单位 |
|------|------|------|
| $h$ | 仿真步长 | s |
| $\mathbf{Y}_n$ | 节点导纳矩阵 | S |
```

## 4. 实体页风格规范

### 4.1 总体原则

实体页（工具、机构、学者、标准组织）的语调应为**客观描述**，避免：

- ✅ 事实陈述："该工具支持 256 个子模块仿真"
- ❌ 营销用语："该工具是目前最强大的仿真平台"
- ✅ 对比陈述："测试表明在 50 子模块以下性能优于 X"（需注明来源）
- ❌ 泛化评价："性能远优于同类产品"

### 4.2 模板结构

```markdown
# 实体名称

## 概述
[客观介绍：是什么、创立时间、核心领域]

## 核心产品/贡献
[基于事实的描述，每个条目附来源]

## 在 EMT 领域的位置
[与 wiki 主题/方法/模型的关联]

## 评价与局限性
[如有负面信息需平衡呈现]

## 代表性来源
[- [[source-1]]
 - [[source-2]]]
```

### 4.3 敏感表述检查

实体页面修订后对照以下检查：

- [ ] 是否含"最"字头最高级表述（最先进、最重要、最具影响力）
- [ ] 是否含无比较基准的领先声明（国际领先、全球领先）
- [ ] 是否含绝对化表述（彻底、完全、所有）
- [ ] 是否含市场宣传用语（金标准、全新一代）
- [ ] 是否含个人崇拜式表述（泰斗、权威、大师）
- [ ] 所有正面评价是否有引用支持

## 5. 页面元数据规范

### 5.1 YAML frontmatter

```yaml
---
title: "页面标题"
type: method | model | topic | entity | source | test-system
tags: [标签1, 标签2]
created: "YYYY-MM-DD"
updated: "YYYY-MM-DD"
---
```

- `type` 字段必须使用规范类型之一
- `tags` 使用小写，连字符分词
- 来源页需额外包含 `year`, `sources` 字段

### 5.2 类型检查

| 页面类型 | type 值 | tags 建议 |
|---------|---------|----------|
| 方法 | method | numerical-integration, stability, ... |
| 模型 | model | mmc, transmission-line, ... |
| 主题 | topic | co-simulation, real-time, ... |
| 实体 | entity | tool, institution, scholar, std-body |
| 来源 | source | 方法/模型/主题关键词 |
| 测试系统 | test-system | benchmark, hvdc, mmc |

## 6. 快速检查清单

修订完成后逐项核对：

### 表达
- [ ] 所有最高级/强断言已降级为事实陈述
- [ ] 量化描述含数值证据，无模糊量词
- [ ] 边界条件明确（适用/不适用场景）
- [ ] 无拟人化表述（"论文提出"→"X 等提出"）

### 引用
- [ ] 间接引用使用"参见"而非直接引用格式
- [ ] 关键数据/结论标注来源
- [ ] 术语首次出现时定义缩写

### 公式
- [ ] 所有公式使用 LaTeX 格式
- [ ] 关键公式已编号
- [ ] 首次出现的变量已解释含义
- [ ] 推导步骤连续无跳跃

### 实体
- [ ] 实体页面无营销/广告化语言
- [ ] 所有正面陈述有可核查来源
- [ ] 评价保持客观中立
