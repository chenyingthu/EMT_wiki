# EMT Wiki 学术润色规范

## 一、语言风格规范

### 1.1 口语化 → 学术化

| 原表达 | 学术表达 | 说明 |
|--------|---------|------|
| 这种方法 | 该方法 | 特指前文提及的方法 |
| 这个东西 | 该要素/该实体 | 明确指代对象 |
| 可以看到 | 分析表明 | 基于证据的陈述 |
| 我们发现 | 研究表明 | 引述研究发现 |
| 很明显 | 显然 | 简洁的学术表达 |
| 怎么做 | 实现机制 | 技术层面的描述 |
| 结果怎么样 | 性能指标 | 量化描述 |
| 有问题 | 存在局限性 | 学术委婉表达 |

### 1.2 句式结构优化

**原句**: "这个方法能够有效地降低计算量，提高仿真速度。"

**优化**: "该方法通过降低计算复杂度，显著提升仿真效率。"

**原则**:
- 避免口语化副词（"很"、"非常"、"比较"）
- 使用专业动词（"降低"→"抑制"、"提高"→"优化"）
- 明确因果关系（"通过...实现..."）

## 二、符号规范统一

### 2.1 电压电流

```
✅ 正确: $v$ (瞬时电压), $V$ (有效值/直流)
❌ 错误: $u$, $U$ (非标准)

✅ 正确: $i$ (瞬时电流), $I$ (有效值/直流)
❌ 错误: $j$ (与虚数单位冲突)
```

### 2.2 向量与矩阵

```
✅ 正确: $\mathbf{v}$ (粗体小写)
❌ 错误: $\vec{v}$, $\overrightarrow{v}$ (非标准)

✅ 正确: $\mathbf{Y}$ (粗体大写)
❌ 错误: $[Y]$, $\overline{Y}$
```

### 2.3 微分与积分

```
✅ 正确: $\frac{\mathrm{d}v}{\mathrm{d}t}$ (d正体)
❌ 错误: $\frac{dv}{dt}$ (d斜体)

✅ 正确: $\mathrm{j}\omega$ (j正体)
❌ 错误: $j\omega$ (j斜体)
```

### 2.4 单位格式

```
✅ 正确: $10\,\mathrm{kV}$, $100\,\mathrm{MW}$
❌ 错误: $10kV$, $100MW$ (缺少间距)

✅ 正确: $0.5\,\mathrm{ms}$
❌ 错误: $0.5ms$ (可能误解为毫西门子)
```

## 三、逻辑结构规范

### 3.1 章节逻辑

每个method/topic/model页面应遵循：

```
1. 定义与边界
   - 准确定义（是什么）
   - 边界限定（适用/不适用场景）

2. EMT中的作用
   - 问题背景（为什么需要）
   - 核心功能（解决什么问题）

3. 主要分支/机制
   - 分类体系（有哪些类型）
   - 比较分析（优缺点对比）

4. 形式化表达
   - 数学模型（公式推导）
   - 参数说明（符号定义）

5. 适用边界与失败模式
   - 前提假设（必要条件）
   - 失效边界（极限条件）
   - 数值问题（稳定性/精度）

6. 代表性来源
   - 关键文献（3-5篇）
   - 验证证据（测试系统/结果）
```

### 3.2 段落逻辑

**PEEL结构**:
- **P**oint: 核心观点句
- **E**vidence: 证据/公式支持
- **E**xplanation: 解释说明
- **L**ink: 与上下文连接

**示例**:
```
梯形积分法在EMT仿真中广泛应用（观点）。
其离散形式为：$v_{n+1} = v_n + \frac{h}{2}(f_n + f_{n+1})$（证据）。
该方法具有二阶精度，且数值稳定（解释）。
然而，在开关事件处可能产生数值振荡（连接）。
```

## 四、常见问题清单

### 4.1 语言问题

| 问题 | 示例 | 修正 |
|------|------|------|
| 的的不当 | "该方法的的性能" | "该方法的性能" |
| 重复表达 | "EMT仿真中的的精度问题" | "EMT仿真中的精度问题" |
| 标点混乱 | "电压,电流和功率," | "电压、电流和功率，" |
| 术语不一致 | 前用"换流器"后用"变流器" | 统一为"换流器" |
| 缩写未定义 | 首次出现"MMC" | "模块化多电平换流器(MMC)" |

### 4.2 数学问题

| 问题 | 示例 | 修正 |
|------|------|------|
| 变量未定义 | "$x$ 表示状态" | "$x \in \mathbb{R}^n$ 为状态向量" |
| 单位缺失 | "$P = 100$" | "$P = 100\,\mathrm{MW}$" |
| 公式不连贯 | 公式单独成段无解释 | 公式前后加文字说明 |
| 符号混用 | 一处用$P$一处用$p$ | 统一使用规范符号 |

### 4.3 逻辑问题

| 问题 | 示例 | 修正 |
|------|------|------|
| 因果不清 | "使用了该方法，仿真变快" | "采用该方法后，仿真效率提升30%" |
| 跳跃过大 | 公式A直接跳到结论C | 补充中间推导步骤B |
| 缺少引用 | "研究表明..." | "Zhang等[1]研究表明..." |
| 假设不明 | "在一般情况下..." | "在满足线性假设条件下..." |

## 五、配图规范

### 5.1 配图需求标记

当页面需要配图时，使用以下标记：

```markdown
<!-- figure-needed: 类型 | 描述 | 优先级 -->
<!-- figure-needed: diagram | MMC桥臂等效电路示意图 | high -->
<!-- figure-needed: plot | 不同步长下的数值误差对比曲线 | medium -->
<!-- figure-needed: photo | 实际MMC换流阀照片 | low -->
```

### 5.2 常见配图类型

| 页面类型 | 建议配图 | 说明 |
|---------|---------|------|
| method | 原理框图 | 方法流程/结构 |
| method | 公式推导图 | 关键推导步骤 |
| method | 对比曲线 | 性能对比/验证 |
| model | 等效电路 | 模型拓扑结构 |
| model | 参数表格 | 关键参数取值 |
| topic | 系统架构 | 整体框架图 |
| topic | 分类图谱 | 分类体系图 |

## 六、检查清单

页面润色完成后，检查：

### 内容完整性
- [ ] 定义章节是否明确回答了"是什么"
- [ ] 是否说明了该方法/模型在EMT中的独特作用
- [ ] 形式化表达是否有清晰的数学推导
- [ ] 边界条件是否明确（何时有效/失效）
- [ ] 是否有代表性来源引用

### 语言规范性
- [ ] 是否消除了口语化表达
- [ ] 专业术语是否统一
- [ ] 缩写是否在首次出现时定义
- [ ] 标点符号使用是否正确

### 符号一致性
- [ ] 电压是否统一使用 $v$/$V$
- [ ] 向量是否使用粗体 $\mathbf{x}$
- [ ] 微分符号 $d$ 是否为正体
- [ ] 单位是否有正体标注
- [ ] 下标是否为正体（如 $v_d$）

### 逻辑连贯性
- [ ] 段落之间是否有过渡句
- [ ] 公式是否有前后文字说明
- [ ] 结论是否有证据支持
- [ ] 是否有明确的适用边界
