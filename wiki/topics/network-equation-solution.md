# 网络方程的建立与求解 (Network Equation Formulation and Solution)

## 概述

网络方程的建立与求解是EMT仿真的核心计算环节。在每个仿真时步，EMT软件需要将电力系统中的各种元件（发电机、变压器、线路、负荷、电力电子设备等）转化为等效电路模型，组装成节点导纳矩阵，并求解大型稀疏线性方程组得到节点电压。这一过程决定了仿真的精度、效率和可扩展性。

在EMT语境下，网络方程求解面临独特挑战：需要处理开关事件导致的拓扑频繁变更、非线性元件的迭代求解、大规模系统的稀疏矩阵技术，以及实时仿算的确定性延迟要求。理解这些技术的原理和适用边界，是掌握EMT仿真的关键。

## 作用机制

### 3.1 节点导纳矩阵的构建

**基本节点电压方程**

EMT仿真采用改进增广节点分析法（MANA），统一的矩阵形式为：

$$
\begin{bmatrix} Y_n & A \\ B & C \end{bmatrix}
\begin{bmatrix} v \\ i_x \end{bmatrix} =
\begin{bmatrix} j \\ e \end{bmatrix}
$$

其中：
- $Y_n \in \mathbb{R}^{n \times n}$：节点导纳矩阵（Nodal Admittance Matrix）
- $v \in \mathbb{R}^n$：节点电压向量
- $j \in \mathbb{R}^n$：注入电流源向量
- $i_x$：附加支路电流（如电压源、电感等需要增广的变量）

**伴随电路离散化**

各动态元件通过数值积分转换为伴随电路（Companion Circuit）：

**电感元件（梯形积分）**：
$$
i_L(t) = G_{eq,L} v_L(t) + I_{hist,L}, \quad G_{eq,L} = \frac{\Delta t}{2L}
$$

$$
I_{hist,L} = i_L(t-\Delta t) + \frac{\Delta t}{2L} v_L(t-\Delta t)
$$

**电容元件（梯形积分）**：
$$
i_C(t) = G_{eq,C} v_C(t) + I_{hist,C}, \quad G_{eq,C} = \frac{2C}{\Delta t}
$$

$$
I_{hist,C} = -\left[\frac{2C}{\Delta t} v_C(t-\Delta t) + i_C(t-\Delta t)\right]
$$

**导纳矩阵组装**

$$
Y_n = A_a Y_b A_a^T
$$

其中：
- $A_a$：节点-支路关联矩阵
- $Y_b$：支路导纳对角矩阵

### 3.2 开关事件处理：拓扑变更的方法

**变导纳方法（传统方法）**

开关状态改变时直接修改导纳矩阵：

- **闭合**：$G_{sw} = G_{on}$（大电导，如$10^6$ S）
- **断开**：$G_{sw} = G_{off}$（小电导，如$10^{-6}$ S）

**问题**：每次开关动作都需要重新分解导纳矩阵，计算开销大。

**临界阻尼调整（CDA）**

在开关事件点插入半步长后向欧拉：

$$
x(t + \Delta t/2) = x(t) + \frac{\Delta t}{2} f(t + \Delta t/2, x(t + \Delta t/2))
$$

**作用**：
- 抑制梯形法数值振荡
- 保持整体2阶精度
- 需要检测开关事件

**恒定导纳矩阵技术（ADC模型）**

通过约束开关等效电感电容乘积等于时间步长平方：

$$
L_{sw}C_{sw} = (\Delta t)^2
$$

确保开关状态变化时系统导纳矩阵保持不变。

**预计算策略（GENE方法）**

- 枚举子模块所有开关状态组合
- 为每种组合预存LU分解结果
- 运行时直接查表，避免在线分解

**插值技术**

精确计算开关时刻：

$$
t_{sw} = t_n + \frac{v_{sw} - v(t_n)}{v(t_{n+1}) - v(t_n)} \Delta t
$$

### 3.3 非线性元件的处理：迭代法 vs 线性化

**牛顿-拉夫逊迭代**

求解非线性方程：

$$
F(x) = 0
$$

迭代格式：

$$
x^{(k+1)} = x^{(k)} - J^{-1}(x^{(k)})F(x^{(k)})
$$

其中雅可比矩阵：

$$
J_{ij} = \frac{\partial F_i}{\partial x_j}
$$

**应用实例：饱和变压器**

非线性磁化特性：$\Phi = f(i_m)$

牛顿迭代：

$$
i_m^{(k+1)} = i_m^{(k)} - \frac{\Phi(i_m^{(k)}) - \Phi_{target}}{d\Phi/di_m}
$$

**简化牛顿法**

固定雅可比矩阵：

$$
J_0 \Delta x^{(k)} = -F(x^{(k)})
$$

减少LU分解次数，适用于弱非线性系统。

**分段线性化**

将非线性V-I特性分段线性近似：

| 区域 | 特性 | 等效参数 |
|------|------|---------|
| 导通区 | 线性 | $R = R_{on}$ |
| 关断区 | 线性 | $R = R_{off}$ |
| 过渡区 | 线性插值 | $R = f(V)$ |

避免迭代，适合电力电子详细模型。

**预测-校正策略**

```
预测（Predictor）：
  x^(0) = x_n + h f(x_n)  （显式欧拉）

校正（Corrector）：
  for k = 0, 1, ..., max_iter
    F = x^(k) - x_n - h f(x^(k))
    if ||F|| < tol: break
    J = I - h ∂f/∂x
    solve J Δx = -F
    x^(k+1) = x^(k) + Δx
```

### 3.4 稀疏矩阵技术

**稀疏性原理**

电力系统导纳矩阵的稀疏特性：

| 系统规模 | 非零元素占比 |
|---------|-------------|
| 100节点 | ~2% |
| 1,000节点 | ~0.5% |
| 10,000节点 | ~0.05% |

**稀疏存储格式**

| 格式 | 存储方式 | 适用场景 |
|------|----------|----------|
| COO | (行,列,值)三元组 | 构建阶段 |
| CSR | 行指针+列索引+值 | 行访问频繁 |
| CSC | 列指针+行索引+值 | 列访问频繁（EMT常用）|

**LU分解与填充元控制**

高斯消去过程中产生的填充元（Fill-in）会严重影响计算效率。

**最小度算法（AMD）**：
- 每次选择度数最小的节点进行消去
- 目标：最小化填充元数量
- 比精确MD快10-100倍，质量接近

**BTF预处理（块三角形式）**：

$$
P^T Y Q = \begin{bmatrix} A_{11} & & \\ A_{21} & A_{22} & \\ \vdots & \vdots & \ddots \end{bmatrix}
$$

**KLU求解器流程**：

```
1. AMD排序（首次）
2. BTF分解（首次）
3. 符号分解（首次）：确定L/U结构
4. 数值分解（拓扑变化时）：计算L/U数值
5. 前代回代（每步执行）
```

**避免重分解技术**

| 技术 | 原理 | 加速效果 |
|------|------|---------|
| 伴随模型 | 将动态元件离散为等效导纳 | 矩阵恒定 |
| 恒导纳ADC | 约束开关LC乘积 | 开关不触发重分解 |
| 预计算存储 | 枚举开关状态 | 查表<1μs |
| 节点分裂 | 将开关限制在子网内 | 局部重分解 |

### 3.5 直接法 vs 迭代法在网络求解中的取舍

**直接法（LU分解）**

| 特性 | 说明 |
|------|------|
| 计算复杂度 | $O(N^3)$（稠密），$O(N^{1.2})$（稀疏） |
| 内存需求 | $O(N^2)$（填充后） |
| 确定性 | 计算时间确定，适合实时仿真 |
| 多右端项 | 分解一次可求解多组右端项 |

**迭代法（Krylov子空间）**

| 特性 | 说明 |
|------|------|
| 计算复杂度 | $O(kN)$，$k$为迭代次数 |
| 内存需求 | $O(N)$ |
| 并行性 | 适合并行，加速比高 |
| 预处理 | 需要有效的预处理器 |

**选择指南**

| 系统规模 | 推荐方法 | 理由 |
|---------|---------|------|
| < 1,000节点 | KLU直接法 | 简单、确定、高效 |
| 1,000-10,000节点 | KLU/GMRES混合 | 稀疏技术+预处理 |
| > 10,000节点 | GMRES/ILU | 内存效率、可扩展性 |
| 实时仿真 | KLU+恒导纳 | 确定性计算时间 |

**混合策略**

```
大规模系统：
├─ 子网内部：KLU直接求解（稀疏优化）
├─ 边界网络：GMRES迭代求解（避免稠密化）
└─ 开关处理：恒定导纳 + 查表
```

## 适用边界

- 变导纳方法适合开关不频繁的系统，频繁开关会导致计算效率急剧下降
- 恒定导纳方法适合电力电子系统，但需要配合插值技术处理开关时刻
- 牛顿迭代对初值敏感，强非线性系统可能不收敛，需采用阻尼牛顿或分段线性化
- 稀疏矩阵技术对拓扑结构有要求，特殊拓扑可能导致填充元过多
- 迭代法收敛性依赖预处理，条件数大的系统可能需要特殊的预处理器

- **Dommel节点法** (1969)

  - 节点导纳矩阵求解框架
- **稀疏矩阵概念引入** (1970s)
  - 利用电力系统稀疏性
  - 减少计算量和存储需求

### 1980s-1990s：稀疏技术成熟
- **最小度算法（MD/AMD）** (1980s)
  - 填充元最小化
  - 符号分解优化
- **BTF预处理** (1990s)
  - 块三角形式分解
  - 并行求解基础

### 2000s-2010s：开关处理突破
- **GENE方法** (2006)
  - 预计算开关状态
  - 避免在线LU分解

  - 开关不触发重分解
  - 实时仿真实现

### 2010s-2026年：大规模与并行
- **KLU求解器** (2010s)

## 深度增强内容

### 1. 节点导纳矩阵构建流程

```
输入：网络拓扑、元件参数、时间步长
输出：节点导纳矩阵Y、注入电流向量I

步骤：
1. 拓扑分析
   - 识别独立节点
   - 建立节点-支路关联矩阵Aa

2. 元件离散化
   - 对每个动态元件应用数值积分
   - 计算等效导纳Geq和历史电流源Ihist

3. 矩阵组装
   - Yn = Aa * Yb * Aa^T
   - Iinj = Aa * (Isrc - Ihist)

4. 边界条件处理
   - 接地节点处理
   - 电压源增广

5. 稀疏化存储
   - 转换为CSC/CSR格式
   - 去除零元素
```

### 2. 开关处理策略对比

| 方法 | 原理 | 计算开销 | 适用场景 |
|------|------|---------|---------|
| 变导纳 | 直接修改Y矩阵 | 高（每步重分解）| 开关少 |
| CDA | 事件点插入BE | 中 | 精确性要求高 |
| 恒导纳ADC | 约束LC乘积 | 低 | 电力电子 |
| 预计算GENE | 枚举状态查表 | 极低 | 子模块有限 |
| 补偿法 | 电流源补偿 | 低 | 快速近似 |

### 3. 非线性求解参数

| 应用场景 | 最大迭代 | 绝对容差 | 雅可比更新策略 |
|----------|----------|----------|----------------|
| 饱和变压器 | 5-10 | 1e-6 | 每步更新 |
| 避雷器 | 3-5 | 1e-4 | 固定/简化 |
| 开关分段 | 1-2 | 1e-3 | 线性化 |
| 混合仿真接口 | 3-10 | 1e-3 | 预测-校正 |

### 4. 大规模系统求解策略

```
系统规模评估：
├─ N < 1,000：KLU直接求解
│   └─ 单处理器，简单高效
├─ 1,000 ≤ N < 10,000：KLU+优化
│   ├─ AMD排序
│   ├─ BTF预处理
│   └─ 并行前代回代
└─ N ≥ 10,000：混合策略
    ├─ 分网分解
    ├─ 子网KLU
    ├─ 边界GMRES
    └─ 并行计算
```

### 5. 实时仿真特殊考量

**确定性要求**：
- 计算时间必须小于步长
- 避免迭代（使用预测-校正）
- 固定导纳矩阵
- 预计算查表

**性能指标**：
- 100节点：~10μs
- 1,000节点：~50μs
- 10,000节点：~200μs（需并行）
