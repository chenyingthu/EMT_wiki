# 最近电平控制 (Nearest Level Control)

## 定义

最近电平控制（Nearest Level Control, NLC），又称最近电平调制（Nearest Level Modulation, NLM），是模块化多电平换流器（Modular Multilevel Converter, MMC）中最基本的调制策略之一。其核心思想是：根据交流侧参考电压的幅值，选择最接近该电压值的整数个子模块投入运行，从而在桥臂上产生阶梯波形的输出电压。NLC算法本质上是将连续模拟量映射为离散电平序列，属于低开关频率调制方法的典型代表。

NLC的数学表达极为简洁：对于给定的参考电压 $v_{ref}$ 和子模块电容电压 $V_{SM}$，首先计算目标电平数：

$$n_{ref} = \frac{|v_{ref}|}{V_{SM}}$$

然后取最近的整数作为实际投入的子模块数：

$$n_{on} = \text{round}(n_{ref}) = \left\lfloor n_{ref} + 0.5 \right\rfloor$$

其中 $\text{round}(\cdot)$ 为四舍五入取整运算。当参考电压为正时投入上桥臂子模块，当参考电压为负时投入下桥臂子模块，通过上下桥臂的互补配合在交流侧产生对称的多电平波形。

**边界限定**：NLC方法适用于子模块数较多（$N > 20$）的MMC换流器。当电平数较低时，量化误差导致的谐波含量较高，性能不如载波移相PWM（Carrier Shift PWM, CPS-PWM）。在极高电平数（$N > 200$）场景下，NLC的计算优势尤为明显——相比CPS-PWM，计算量可降低约5000倍。

## EMT中的作用

NLC在MMC电磁暂态仿真中扮演双重角色：一方面作为调制算法产生PWM脉冲序列，另一方面作为电容电压均衡策略的触发机制。在EMT仿真中，NLC的处理方式直接影响仿真的精度和速度。

### NLC的核心功能

1. **桥臂电压生成**：NLC将控制系统给出的调制波 $m \cdot \sin(\omega t + \theta)$ 转换为桥臂子模块的投入/切除决策。对于有 $N$个子模块的桥臂，上桥臂投入数量 $n_p$ 和下桥臂投入数量 $n_n$ 满足：

$$n_p = \begin{cases} \text{round}\left(\dfrac{N}{2} + \dfrac{v_a}{V_{SM}}\right) & v_a \geq 0 \\\\ 0 & v_a < 0 \end{cases}$$

$$n_n = \begin{cases} 0 & v_a \geq 0 \\\\ \text{round}\left(\dfrac{N}{2} - \dfrac{v_a}{V_{SM}}\right) & v_a < 0 \end{cases}$$

2. **子模块电容电压均衡**：每个开关周期内，NLC根据当前桥臂电流方向和电容电压排序结果，决定哪些子模块投入、哪些旁路。电流正向时优先投入电压低的子模块（充电模式），电流反向时优先投入电压高的子模块（放电模式）。

3. **环流动态耦合**：NLC在选择子模块投入数量时，会影响桥臂等效阻抗，进而影响桥臂环流 $i_{circ}$ 的二倍频分量大小。环流抑制控制（Circulating Current Suppression Control, CCSC）需要与NLC协调设计。

### NLC在EMT仿真中的关键挑战

| 挑战类型 | 具体表现 | 对仿真的影响 |
|---------|---------|-------------|
| 开关事件处理 | 每个子模块的IGBT通断产生大量开关事件 | 仿真步长受开关速度限制 |
| 电容电压排序 | $O(N \log N)$ 排序计算与仿真步长耦合 | 大电平数时计算瓶颈 |
| 状态更新时序 | 排序→决策→调制→状态更新顺序执行 | 并行化困难 |
| 谐波特性 | 量化误差引入的非特征谐波 | THD分析需额外处理 |

## EMT建模方法

NLC在EMT中的建模方法主要分为三类：**详细开关模型**、**等效平均值模型**和**快速NLM等效模型**。三者精度依次降低，但计算效率依次提升。

### 方法1：详细NLC开关模型

详细NLC模型严格模拟每个子模块的IGBT通断过程和电容电压排序逻辑，是EMT仿真的"金标准"模型。

**算法步骤**：
1. 读取当前时刻的调制波采样值 $v_{ref}(t)$
2. 计算目标电平数 $n_{ref} = |v_{ref}(t)| / V_{SM}$
3. 取整得到投入子模块数 $n_{on} = \text{round}(n_{ref})$
4. 读取所有子模块电容电压值 $U_{C1}, U_{C2}, \ldots, U_{CN}$
5. 根据桥臂电流方向 $i_{arm}$ 执行排序算法
   - 若 $i_{arm} > 0$：按电压升序排序，投入电压最低的 $n_{on}$ 个子模块
   - 若 $i_{arm} < 0$：按电压降序排序，投入电压最高的 $n_{on}$ 个子模块
6. 更新子模块状态和电容电压

**计算复杂度**：$O(N \log N)$ 排序 + $O(N)$ 状态更新，每次仿真步长都需要完整执行。当 $N = 200$ 时，单步排序计算量约为 $200 \times \log_2(200) \approx 1720$ 次比较操作。

**量化性能边界**（Gnanarathna 2011）：
- 仿真步长限制：$\Delta t \leq 50 \, \mu s$（受IGBT开关速度约束）
- 开关频率：$f_{sw} \approx f_0 / N$（基频 $f_0 = 50 Hz$，$N=200$ 时 $f_{sw} = 0.25 Hz$）
- 计算效率：100子模块系统实时仿真需FPGA主频 $> 200 MHz$

### 方法2：NLM等效平均值模型

Zhao 2023提出的NLM等效模型将NLM生成的子模块等效占空比分解为**稳态分量**与**波动分量**两部分，通过直流电压闭环控制替代离散排序逻辑。

**核心公式**：
等效占空比分解：
$$d_{eq} = d_{stable} + d_{fluctuation}$$

稳态分量由调制波直接生成：
$$d_{stable} = \frac{u_{ref\_pa}}{U_{dc}}$$

其中 $u_{ref\_pa}$ 为上桥臂调制波瞬时值，$U_{dc}$ 为直流侧电压。

波动分量由子模块直流电压闭环自动产生：
$$d_{fluctuation} = K_p \cdot (U_{dc}/N - u_{dcpa\_i}) + K_i \cdot \int (U_{dc}/N - u_{dcpa\_i}) dt$$

**A相上桥臂电流解析表达式**（Zhao 2023原文式(1)）：
$$i_{pa} = -\frac{I_{dc}}{3} - \frac{I_{vm}}{2}\sin(\omega t + \theta_{a1} - \alpha) - I_{cirm}\cos(2\omega t + \theta_{a2} - \beta) - \frac{4U_{dc}}{\pi N(2L_{ac}+L_0)}\sum_{k=6i\pm1}^{\infty}\frac{f(k)}{k^2\omega}\cos(k\omega t + \theta_{ak})$$

**A相环流电流表达式**（Zhao 2023原文式(2)）：
$$i_{cira} = -\frac{I_{dc}}{3} - I_{cirm}\cos(2\omega t + \theta_{a2} - \beta)$$

**量化性能边界**（Zhao 2023）：
| 指标 | 数值 | 说明 |
|------|------|------|
| 仿真步长 | 1~5 ms | 可放宽至毫秒级（传统NLC需 $\leq 50 \, \mu s$） |
| 电压误差 | < 0.5% | 0~500Hz低频范围内与传统NLM一致 |
| 电容电压均衡偏差 | < ±2% | 稳态波动幅值 |
| 仿真加速比 | 15~40× | 随子模块数N增加而显著提升 |
| 计算复杂度 | O(N) | 线性于子模块数（vs 开关模型O(N log N)） |

### 方法3：快速排序优化算法

针对大电平数MMC实时仿真的排序计算瓶颈，樊强2020提出分组并行排序算法，将排序过程分为**组内排序**和**组间排序**两个阶段。

**分组策略**：将 $N$ 个子模块分为 $n$ 组，每组 $m = N/n$ 个子模块。组内采用并行全比较算法（4时钟周期完成排序），组间采用能量平衡因子方法协调。

**能量平衡因子**定义为组内子模块电容电压最大值、最小值和中间值之和：
$$f = U_{Cmax} + U_{Cmin} + U_{Cmid}$$

**最优分组数**（樊强2020原文式(3)）：
$$n_0 = \sqrt[3]{\frac{N}{4} + \frac{1}{2}\sqrt{\frac{N^2}{16} + \frac{216}{(6N+1)^2}}} + \sqrt[3]{\frac{N}{4} - \frac{1}{2}\sqrt{\frac{N^2}{16} + \frac{216}{(6N+1)^2}}} - \frac{1}{6}$$

**比较器数量**（樊强2020原文式(1)）：
$$N_{com} = n(n-1) + \frac{N}{n} - 1$$

**混合脉冲触发模式**：串行触发与并行触发结合。设分组排序后有序序列 $U_{Cn_i}$ $(i=1,2,3)$ 对应的子模块编号序列为 $NC_{n_i}$，根据"充小放大"原则，组内子模块采用串行触发模式确定触发脉冲，组间采用并行触发模式（$NC_{n_i}=0$ 表示全部旁路，$NC_{n_i}=1$ 表示全部投入）。

**量化性能边界**（樊强2020）：
| 算法 | 计算步长 | $N_{com}$ | 子模块电容电压波动范围 |
|------|---------|-----------|---------------------|
| 冒泡排序 | 1 | $N(N-1)/2$ | [2.917, 3.502] kV |
| 并行全比较 | 4 | $N(N-1)$ | [2.917, 3.480] kV |
| 本文算法 | 10 | 按式(1)计算 | [2.917, 3.490] kV |

当 $N=100$ 时，取 $n=10$（$m=10$），所需比较器数量远少于并行全比较算法，且FPGA资源占用（ALM）减少约0.48%。

## 形式化表达

### 量化误差分析

NLC引入的量化误差定义：
$$\epsilon = v_{ref} - n_{on} \cdot V_{SM}$$

最大量化误差（理论极限）：
$$|\epsilon|_{max} = 0.5 V_{SM}$$

相对误差随电平数增加而减小：
$$\frac{|\epsilon|_{max}}{V_{ref}} = \frac{0.5 V_{SM}}{n_{on} V_{SM}} = \frac{0.5}{n_{on}} \leq \frac{0.5}{N/2} = \frac{1}{N}$$

### 谐波特性

输出电压THD与子模块数的关系（理想情况）：
$$THD \approx \frac{1}{\sqrt{6} N}$$

当 $N > 100$ 时，THD 通常 < 1%。

### 开关频率

平均开关频率：
$$f_{sw} = \frac{f_0}{N} \cdot \frac{V_{dc}}{2 V_{SM}}$$

其中 $f_0$ 为基频（50Hz或60Hz），$V_{dc}$ 为直流电压，$V_{SM}$ 为子模块电容电压额定值。

### 电平数选择准则

| 应用场景 | 推荐最小电平数 | 说明 |
|---------|---------------|------|
| 高压直流输电 | $N \geq 200$ | THD < 0.5%，开关频率极低 |
| 中压配电 | $N \geq 50$ | THD < 2%，可接受 |
| 低压变换器 | $N \geq 20$ | THD < 5%，经济性最优 |
| 故障穿越 | $N \geq 100$ | 快速暂态响应需要较多电平 |

## 关键技术挑战

### 挑战1：低电平数时的谐波抑制

当 $N < 20$ 时，NLC的量化误差导致谐波含量显著增加。**解决方案**：
- 采用混合调制策略：低电平数时切换为CPS-PWM
- 添加谐波注入算法：在调制波中注入特征次谐波以抵消量化谐波
- 使用最近电平优化（Optimized NLC）：根据电压波形质量最优选择投入电平

### 挑战2：大电平数时的排序效率

当 $N > 500$ 时，$O(N \log N)$ 排序成为计算瓶颈。**解决方案**：
- 采用并行全比较算法（4时钟周期完成排序）
- 分组排序策略：将全局排序分解为组内+组间两级排序
- 降频排序：每隔若干仿真步长执行一次完整排序，中间步长仅做微调

### 挑战3：电容电压不平衡治理

实际工程中，子模块电容电压可能因制造差异、温度不均或控制延迟而出现不平衡。**量化标准**：
- 电压不平衡度：$\delta U = \dfrac{U_{Cmax} - U_{Cmin}}{U_{Cavg}} \times 100\%$
- 工程允许限值：$\delta U \leq 10\%$
- 均衡控制目标：$\delta U \leq 2\%$（稳态）

### 挑战4：环流与NLC的耦合动态

NLC在选择子模块投入数量时，会改变桥臂的等效导纳，从而影响环流路径。**分析要点**：
- 桥臂环流包含直流偏置、二倍频分量及高次谐波
- 二倍频环流与NLC的耦合导致子模块电压出现2次谐波波动
- CCSC与NLC需要协调设计以抑制环流

### 挑战5：实时仿真的步长约束

传统NLC的开关事件导致仿真步长受IGBT开关速度限制。**突破方法**：
- 等效平均值模型：用受控电压源替代开关动作
- 事件检测与定位：仅在开关事件附近细化步长
- 并行计算：利用GPU/FPGA并行处理多子模块状态更新

## 适用边界与选择指南

### NLC vs CPS-PWM选择

| 条件 | NLC | CPS-PWM |
|------|-----|---------|
| 电平数 $N > 100$ | ✓ 首选 | 可选 |
| 电平数 $20 < N < 100$ | ✓ 可用 | ✓ 较好 |
| 电平数 $N \leq 20$ | ✗ 谐波大 | ✓ 首选 |
| 计算资源受限 | ✓ 简单高效 | ✗ 计算量大 |
| 开关频率要求极低 | ✓ 首选 | ✗ 开关频率高 |

### 建模精度选择

| 应用场景 | 推荐模型 | 步长 | 精度要求 |
|---------|---------|------|---------|
| 开关器件级暂态分析 | 详细开关模型 | $\leq 1 \, \mu s$ | 最高 |
| 系统级机电暂态 | 等效平均值模型 | 0.1~1 ms | 中等 |
| 控制器硬件在环测试 | 等效模型+实时平台 | 0.1~1 ms | 中等 |
| 故障穿越仿真 | 详细模型或等效模型 | 取决于关注动态 | 按需选择 |
| 稳态谐波分析 | 等效模型 | 1~5 ms | 足够 |

### 快速NLM等效模型的适用条件

**适用条件**：
- 仿真关注低频动态（0~500Hz）
- 时间步长可放宽至毫秒级
- 需要保留子模块电容电压状态作为可观测量
- 系统规模较大，需要仿真加速

**不适用条件**：
- 开关器件级暂态分析
- 高频谐波（>1kHz）研究
- 精确开关时序研究
- 故障闭锁瞬间分析
