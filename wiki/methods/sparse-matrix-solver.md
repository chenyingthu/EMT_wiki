# 稀疏矩阵求解方法 (Sparse Matrix Solver)

## 定义与概述

稀疏矩阵求解方法是电磁暂态（EMT）仿真中处理大规模电力系统网络方程的核心计算技术。电力系统节点导纳矩阵具有高度稀疏性（非零元素占比通常<1%），稀疏矩阵技术通过利用这一特性，将求解复杂度从$O(N^3)$降至接近$O(N)$，使大规模电网（10,000+节点）的实时仿真成为可能。稀疏矩阵求解涵盖矩阵重排序、符号分解、数值分解、前代回代等关键步骤，是EMTP、PSCAD、ADPSS等主流仿真器的计算引擎基础。

## 1. 理论基础

### 1.1 稀疏性原理

**电力系统导纳矩阵稀疏性**:
- 典型500节点系统：非零元素占比约0.5-1%
- 10,000节点系统：非零元素占比约0.05-0.1%
- 稀疏模式：对称（无互感）或近似对称（有互感）

**稀疏存储格式**:
| 格式 | 存储方式 | 适用场景 |
|------|----------|----------|
| COO | (行,列,值)三元组 | 构建阶段 |
| CSR | 行指针+列索引+值 | 行访问频繁 |
| CSC | 列指针+行索引+值 | 列访问频繁（EMT常用）|
| 对角存储 | 主对角+副对角 | 三对角/带状矩阵 |

### 1.2 LU分解基础

**节点电压方程**:
$$YU = I$$

其中：
- $Y \in \mathbb{C}^{N \times N}$：节点导纳矩阵（稀疏）
- $U \in \mathbb{C}^{N}$：节点电压向量（未知）
- $I \in \mathbb{C}^{N}$：节点注入电流向量（已知）

**LU分解**:
$$Y = LU$$

- $L$：下三角矩阵（对角元素为1）
- $U$：上三角矩阵

**求解步骤**:
```
1. 符号分解：确定L/U的非零结构（仅执行一次）
2. 数值分解：计算L/U的具体数值（拓扑变化时执行）
3. 前代求解：Ly = I，求y
4. 回代求解：Ux = y，求U
```

### 1.3 填充元（Fill-in）控制

**填充元产生**: 高斯消去过程中，原零元素位置变为非零。

**最小度算法（MD, Minimum Degree）**:
- 每次选择度数最小的节点进行消去
- 目标：最小化填充元数量

**近似最小度算法（AMD, Approximate Minimum Degree）**:
- 采用近似度数以加速排序
- 比精确MD快10-100倍，质量接近

**嵌套剖分（Nested Dissection）**:
- 基于图划分思想
- 适用于高度结构化的大规模矩阵

### 1.4 网络撕裂与分块

**Diakoptics方法（Kron）**:
将网络分割为子系统，通过边界协调求解：

$$\begin{bmatrix} Y_{11} & Y_{12} \\ Y_{21} & Y_{22} \end{bmatrix} \begin{bmatrix} U_1 \\ U_2 \end{bmatrix} = \begin{bmatrix} I_1 \\ I_2 \end{bmatrix}$$

消去内部节点后边界方程：
$$\hat{Y}_{22}U_2 = \hat{I}_2$$

其中：
$$\hat{Y}_{22} = Y_{22} - Y_{21}Y_{11}^{-1}Y_{12}$$

**优势**: 子系统可并行求解，开关变化仅影响局部。

## 2. EMT仿真应用

### 2.1 实时仿真的计算瓶颈

**单步计算时间分解**:
| 计算阶段 | 典型耗时占比 | 关键优化 |
|----------|-------------|----------|
| 历史项更新 | 10-20% | 向量化 |
| 导纳矩阵组装 | 5-15% | 预计算映射 |
| LU分解（重分解） | 40-60% | 避免重分解/稀疏技术 |
| 前代回代 | 20-40% | 稀疏向量法 |
| 开关处理 | 5-20% | 预计算/分段线性 |

**实时性要求**:
$$T_{step} < \Delta t$$

典型步长：50μs，单步计算时间需<30-40μs以满足裕度。

### 2.2 避免重分解技术

**伴随模型（Companion Model）**:
将动态元件离散化为等效导纳并联历史电流源：
- 电感：$G_{eq} = \frac{\Delta t}{2L}$，$I_{hist} = i_L(t-\Delta t) + G_{eq}v_L(t-\Delta t)$
- 电容：$G_{eq} = \frac{2C}{\Delta t}$，$I_{hist} = -i_C(t-\Delta t) - G_{eq}v_C(t-\Delta t)$

**恒导纳矩阵（ADC模型）**:
约束开关等效电感电容乘积等于时间步长平方：
$$L_{sw}C_{sw} = (\Delta t)^2$$

确保开关状态变化时系统导纳矩阵保持不变。

**预计算策略（GENE方法）**:
- 枚举子模块所有开关状态组合
- 为每种组合预存LU分解结果
- 运行时直接查表，避免在线分解

### 2.3 节点分裂并行求解

**ADPSS实现架构**:
```
┌──────────────────────────────────────┐
│  父层：边界网络（稀疏求解）              │
│  ┌─────┐ ┌─────┐ ┌─────┐            │
│  │子网1│ │子网2│ │子网3│  ← 并行求解 │
│  └─────┘ └─────┘ └─────┘            │
└──────────────────────────────────────┘
```

**量化成果**:
- 10,000节点系统分割为3个子网
- 各子网约3,000节点，矩阵规模降至1/9
- 单步计算时间从~100μs降至~30μs

### 2.4 稀疏向量法

**应用场景**: 仅需计算部分节点电压（如故障点、测量点）。

**原理**: 利用稀疏右端项$I$的结构，跳过大量零元素的运算。

**加速效果**: 当只需1%节点电压时，求解速度提升10-50倍。

### 3.2 KLU算法详解

**BTF预处理（Block Triangular Form）**:
将矩阵置换为块三角形式：
$$P^T Y Q = \begin{bmatrix} A_{11} & & \\ A_{21} & A_{22} & \\ \vdots & \vdots & \ddots \end{bmatrix}$$

**算法流程**:
```c
void KLU_solve(Y, I, U, N) {
    // 1. AMD排序（首次）
    ordering = AMD(Y);
    
    // 2. BTF分解（首次）
    [P, Q, blocks] = BTF(Y, ordering);
    
    // 3. 符号分解（首次）：确定L/U结构
    [L_sym, U_sym] = SymbolicFactor(Y, P, Q, blocks);
    
    // 4. 数值分解（拓扑变化时）
    [L, U] = NumericFactor(Y, L_sym, U_sym);
    
    // 5. 前代回代（每步执行）
    y = ForwardSolve(L, P*I);
    U = BackwardSolve(U, y);
    U = Q^T * U;
}
```

**性能对比**:
- KLU vs 标准LU：稀疏矩阵加速10-1000倍
- AMD vs 自然排序：填充元减少50-90%
- BTF vs 无预处理：块对角结构额外加速2-5倍

### 3.3 开关处理优化

**问题**: 电力电子开关导致拓扑频繁变化，需多次重分解。

**解决方案**:
| 方法 | 原理 | 加速效果 |
|------|------|----------|
| 预计算存储 | 枚举所有开关状态，预存LU因子 | 查表时间<1μs |
| 分段线性化 | 用线性段逼近非线性特性 | 避免迭代 |
| 节点分裂 | 将开关限制在子网内 | 局部重分解 |
| 补偿法 | 用等效电流源补偿开关变化 | 无需重分解 |

### 3.4 并行稀疏求解

**共享内存并行**:
```c
// OpenMP并行前代回代
#pragma omp parallel for
for (int i = 0; i < N; i++) {
    y[i] = b[i];
    for (int j = L_row[i]; j < L_row[i+1]; j++) {
        y[i] -= L_val[j] * y[L_col[j]];
    }
    y[i] /= L_diag[i];
}
```

**限制**: 稀疏前代回代的依赖性限制了并行效率，通常加速比<4（8核）。

**分块并行（MATE/分网法）**:
```
系统分割 → 各子网独立LU分解（并行） → 边界协调求解
```

优势：子网间并行度高，加速比接近线性。

## 4. 仿真软件实现

### 4.1 EMTP-RV实现

```fortran
! EMTP稀疏求解核心
SUBROUTINE SPARSE_SOLVE(N, Y, I, U, IPARM, DPARM)
    INTEGER N, IPARM(*)
    REAL Y(*), I(*), U(*), DPARM(*)
    
    ! 检查是否需要符号分解
    IF (IPARM(1) .EQ. 1) THEN
        CALL SYMBOLIC_FACTOR(N, Y, IPARM)
        IPARM(1) = 0
    ENDIF
    
    ! 数值分解
    CALL NUMERIC_FACTOR(N, Y, DPARM)
    
    ! 前代回代求解
    CALL FORWARD_BACKWARD(N, Y, I, U, DPARM)
END SUBROUTINE
```

**关键优化**:
- 仅首次求解执行符号分解
- 拓扑变化时仅局部更新因子
- 复用稀疏模式减少内存分配

### 4.2 ADPSS并行实现

```python
## ADPSS分网并行求解伪代码
class ADPSS_Sparse_Solver:
    def __init__(self, subnetworks):
        self.subs = subnetworks
        self.boundary_solver = KLUSolver()
        
    def solve_step(self, I_total):
        # 1. 各子网并行计算等值
        equivalents = parallel_map(
            lambda sub: sub.compute_thevenin_equivalent(),
            self.subs
        )
        
        # 2. 组装边界网络
        Y_bound, I_bound = self.assemble_boundary(equivalents)
        
        # 3. 求解边界电压
        U_bound = self.boundary_solver.solve(Y_bound, I_bound)
        
        # 4. 各子网并行更新内部状态
        parallel_map(
            lambda sub: sub.update_internal(U_bound),
            self.subs
        )
```

## 7. 适用边界与限制

### 7.1 适用条件
- **大规模稀疏矩阵**: 非零元占比<5%，稀疏技术才能发挥优势
- **结构相对稳定**: 频繁拓扑变化会增加符号分解开销
- **直接求解**: 适用于规模<100,000节点，更大规模需迭代法

### 7.2 失效边界
- **稠密矩阵**: 非零元占比>20%，稀疏开销超过收益
- **极频繁开关**: 每步拓扑变化导致符号分解成为瓶颈
- **病态矩阵**: 条件数>1e12时，LU分解精度下降
- **超大规模**: >100,000节点，直接法存储需求过高

### 7.3 精度边界
| 因素 | 影响 | 缓解方法 |
|------|------|----------|
| 矩阵条件数 | 误差放大 | 标度化、预处理 |
| 数值分解累积 | 长期漂移 | 定期重分解 |
| 复数运算 | 精度损失 | 双精度计算 |
| 并行竞争条件 | 非确定性 | 原子操作、锁机制 |
