# 广域监测保护与控制 (WAMPAC)

## 定义与边界

广域监测保护与控制（WAMPAC）是把同步测量、通信、状态评估、保护判据和协调控制组合起来的方法体系。它以 [[phasor-measurement-unit]]（同步相量测量单元，PMU）、主站数据集中、通信网络和控制执行设备为基础，用于观察跨区域动态、辅助保护决策或触发协调控制。

WAMPAC 不是单一设备或保证性能力。广域信息可以扩展观测范围，但也引入通信延迟、时标误差、数据缺失、网络安全和控制协调风险。任何保护或控制结论都需要绑定测量配置、通信条件、应用算法和验证工况。

本页不把 WAMPAC 写成单一设备或保证性能力。广域信息可以扩展观测范围，但也引入通信延迟、时标误差、数据缺失、网络安全和控制协调风险。任何保护或控制结论都需要绑定测量配置、通信条件、应用算法和验证工况。

## EMT 中的作用

EMT 仿真中的 WAMPAC 研究通常用于两类问题：

1. **保护与控制闭环测试**：用详细时域模型产生故障、振荡、HVDC 控制和保护动作波形，测试广域测量和保护逻辑是否按预期响应。保护系统作为会改变电网状态的动态子系统进入 EMT 仿真，形成"系统暂态计算—互感器变换—继电器算法处理—保护决策反馈—系统状态更新"的闭环。

2. **离线到实时移植**：将离线 EMT 模型迁移到硬件在环（HIL）或准在线环境，以便把真实保护控制设备、通信仿真器和电网模型连接起来。参考 [[lessons-learned-in-porting-offline-large-scale-power-system-simulation-to-real-t]] 的移植框架：模型库兼容、控制模块等价、用户代码重建、原理图修改、信号校核和性能约束共同作用的系统工程。

## 系统组成

WAMPAC 按功能分为四层：

| 层次 | 主要对象 | 关键边界 |
|------|----------|----------|
| 测量层 | PMU、故障录波、SCADA、开关量 | 采样率、同步误差、互感器暂态和数据质量 |
| 通信层 | 数据集中、时标对齐、网络传输 | 延迟、丢包、时钟源、网络安全 |
| 应用层 | 状态估计、振荡识别、稳定评估、保护逻辑 | 算法假设和验证场景 |
| 执行层 | HVDC、FACTS、发电机励磁、切负荷、解列 | 控制权限、动作时序和失效后果 |

### 广域闭环时间约束

广域闭环的时间约束为：

$$T_{\mathrm{meas}} + T_{\mathrm{comm}} + T_{\mathrm{alg}} + T_{\mathrm{act}} \le T_{\mathrm{allow}}$$

其中 $T_{\mathrm{meas}}$ 是测量与相量估计延迟，$T_{\mathrm{comm}}$ 是通信传输和排队延迟，$T_{\mathrm{alg}}$ 是应用算法计算时间，$T_{\mathrm{act}}$ 是执行器动作时间，$T_{\mathrm{allow}}$ 必须来自保护或控制目标。该式不提供通用动作时限，只提醒 WAMPAC 结论需要同时报告电网响应和信息链路时序。

### PMU 动态相量估计的频率偏移问题

PMU 在扰动期间仍要给出可信同步相量。电网负荷波动、发电与负载不平衡、故障等会使基波频率偏离标称值，并伴随电压/电流幅值振荡；此时按标称频率做 DFT 会产生非同步采样和频谱泄漏，导致幅值、相角和频率估计误差迅速增大。

传统 DFT 计算快但默认频率锁定在标称值；泰勒级数模型能描述动态变化，却未充分处理大范围频偏；插值 DFT 可提高频率估计精度，但依赖窗函数和较多采样点，运算负担更高。

多频率相量模型（MFPM）的核心思想是：用中心粗估频率及其两侧按 $\Delta f$ 间隔分布的若干子相量共同拟合动态基波信号。其核心未知量是各子相量复系数 $A_m$；输入是采样信号 $x(n)$、标称频率 $f_0$、采样频率、窗函数、数据窗位置和粗估频率；输出是报告时刻的同步相量幅值、相角及频率估计。

实际 DFT 结果不仅包含正序旋转子相量贡献，也包含共轭项引起的影响；因此不能简单把 DFT 值当作真实相量：

$$\hat{X} = CA + DA^*$$

将该关系改写成最小二乘问题，用 $F = (G^TG)^{-1}G^T$ 对 DFT 向量进行修正，在线只需查表和矩阵乘法即可求得各子相量参数。

量化性能边界（来自 [[dynamic-synchrophasor-estimator-based-on-multi-frequency-phasor-model]]）：
- 45–55 Hz 静态频偏范围内 MFPM 的总相量误差 TVE 保持在 0.8% 以内，而 DPMA 在两端频偏时约为 8%
- 在 45 Hz 基频叠加 5 Hz 振荡时，MFPM 最大 TVE 为 0.6435%，DPMA 为 11.1224%，CT-DFT 为 0.7756%
- 采用 7 个数据窗时幅值和相角阶跃响应时间分别为 52.44 ms 和 113.47 ms

## EMT 建模方法

### 1. 保护系统闭环建模

EMTP 中的保护系统关键环节包括仪用互感器、继电器算法/模型和断路器反馈。EMTP 原本主要面向电力系统元件暂态，保护算法需要逐时步读取系统量并把跳闸决策送回主程序；同时 CT、CVT 本身会改变继电器所见波形。

保护系统 EMT 建模包含三类部件：

**第一类：CT 和 CVT 模型**——用户可定义其参数，用来把一次系统暂态电流、电压转换为继电器端看到的二次量；模型关注的不是理想变比，而是仪用互感器在暂态下对波形的影响。

**第二类：继电器模型和算法接口**——EMTP 增加"total FORTRAN capability"，允许用户自定义 FORTRAN 子程序与主 EMTP 链接。EMTP 主程序产生时域电压、电流数据，用户子程序按继电器算法处理这些输入，输出动作判据或跳闸/闭锁等决策，再传回 EMTP。

**第三类：断路器和网络状态反馈**——当继电器输出动作信号后，EMTP 据此改变电力系统状态，使后续时步的电气量反映保护动作后的网络拓扑。核心流程是"系统暂态计算—互感器变换—继电器算法处理—保护决策反馈—系统状态更新"的闭环。

可用继电器模型包括线路保护 CEY51A、SLY12C 以及变压器差动保护 D202、BDD15B，并可采用不同整定值。

### 2. EMT-动态相量协同仿真接口

EMT 协同仿真针对大型电网 EMT 仿真的计算瓶颈：电力电子变流器、故障和开关过程需要微秒级步长与详细模型，但这些快速暂态通常只出现在局部区域，若全网都用 EMT 求解，会把大量动态较慢的网络也纳入小步长节点方程。

方法由三部分组成：

**第一，网络划分**：网络被划分为 EMT 子系统和动态相量子系统，接口通过具有传播延迟的传输线模型传递历史电压、电流信息，从而避免两个求解器在同一时刻形成强耦合代数方程。

**第二，动态相量表示**：动态相量侧用滑动窗口傅里叶表示波形，核心状态量是直流、基波及扩展频率谐波分量；EMT 侧核心接口量是瞬时电压、电流样本。重构式为：

$$x(t-T+s) = x_0(t) + 2\text{Re}\left(\sum_h X_h(t) e^{jh\omega(t-T+s)}\right)$$

**第三，双向样本映射**：在 DP 到 EMT 方向，可用 $x(t) = \text{Re}(X(t)e^{j\omega t})$ 类关系把复合相量还原为瞬时量；在 EMT 到 DP 方向，把瞬时样本映射为动态相量样本。

对于多速率仿真，两侧只在同步时刻交换数据，较慢求解器步长需与接口延迟和较快步长协调；非同步时刻的数据由历史样本或插值提供，以控制相位误差和数值稳定性。

量化性能边界（来自 [[co-simulation-of-electrical-networks-by-interfacing-emt-and-dynamic-phasor-simul]]）：
- 25:1 步长比下计算时间由 694 s 降至 132 s（加速比约 5.3×）
- 测试系统：450 MW Type-4 风电场、IEEE 118 节点三相网络、三相接地故障场景

### 3. 移频相量（SFP）接口

移频相量（SFP）方法把目标系统划分为 EMT 子系统和 SFP 子系统：局部快速区域（风电场等）保留传统 EMT 细节，大交流电网用变换型 SFP 模型以把步长扩展到 500 µs，并通过多速率 MD-TLM（改进传输线模型）及频变 MD-TLM 在两个域之间传递宽频带相互作用。

SFP 子系统用复包络表示基频附近的电压、电流等量，即把原始时域波形看作随基频旋转因子调制的慢变量，从而把电网侧较高频的瞬时量转化为较慢变化的相量状态，使大交流电网可用较大步长计算。

EMT 子系统仍以瞬时电压、电流和风机开关/控制状态为状态量，用小步长解析细节。两者的接口量是分区边界处的电压、电流：SFP 侧产生相量或复包络形式的边界量，EMT 侧需要瞬时值。

### 4. 带模式切换的 EMT-相量域混合仿真

基于多区域戴维南等值（MATE）的 EMT-相量域接口，以及两阶段/多条件的切换控制器。详细系统同时维护 EMT 模型和对应的相量域模型；外部系统由相量域动态仿真器表示。

接口量主要是边界三相瞬时电压/电流、三序电压/电流、戴维南等值电压和等效阻抗/导纳。EMT 侧把边界三相波形转换为相量域可用的三序注入量，发送给相量域；相量域求解外部网络后返回边界电压或等值源，再在 EMT 侧转换为诺顿等值并并入 EMT 网络方程。

网络形式为 $I(x, V) = YV$ 表示相量域动态仿真中设备电流注入与网络导纳矩阵的耦合；戴维南到诺顿转换让外部系统等值以导纳和电流源形式直接进入 EMT 求解。

模式切换不是固定时间触发，而是先等待故障后延迟，再检查边界电压变化率是否低于阈值，以判断快暂态已衰减；随后比较详细系统相量模型与外部系统边界电压偏差，要求持续若干工频周期满足容差。

### 5. 广域保护的动作判据接口

广域保护把远方测量纳入动作判据或闭锁逻辑。与就地保护相比，它可使用更多系统状态信息，但必须处理通信异常和动作责任边界。保护应用应优先说明故障类型、动作对象、允许延迟、备用逻辑和失效处理。

广域保护在 EMT 中的建模要点：
- 通信延迟建模：保护动作时间 $T_{\mathrm{act}}$ 中需包含通信传输延迟 $T_{\mathrm{comm}}$
- 数据质量处理：PMU 数据质量受互感器暂态、同步源、滤波算法和频率偏移影响
- 降级策略：当通信不可用时，应有降级或闭锁策略

### 6. 广域控制的控制器接口

广域控制使用远方信号调节 PSS、FACTS、HVDC 功率、无功支撑或负荷。控制设计需要考虑通信延迟、信号丢失、控制饱和和多控制器相互作用。

广域控制在 EMT 中的实现包括：
- 控制信号链路：远方 PMU 测量 → 通信网络 → 控制算法 → 执行器指令 → EMT 系统响应
- 时序约束：$T_{\mathrm{meas}} + T_{\mathrm{comm}} + T_{\mathrm{alg}} + T_{\mathrm{act}} \le T_{\mathrm{allow}}$
- 控制器页面参考：[[power-system-stabilizer]] 和 [[dual-loop-pi-controller]]

## 形式化表达

### 核心等式汇总

**广域闭环时间约束**：

$$T_{\mathrm{meas}} + T_{\mathrm{comm}} + T_{\mathrm{alg}} + T_{\mathrm{act}} \le T_{\mathrm{allow}}$$

**多频率相量模型（MFPM）观测方程**：

$$\hat{X} = CA + DA^*$$

其中 $A$ 是子相量复系数向量，$C$ 和 $D$ 是与频率间隔 $\Delta f$ 相关的矩阵。

**动态相量重构式**：

$$x(t-T+s) = x_0(t) + 2\text{Re}\left(\sum_h X_h(t) e^{jh\omega(t-T+s)}\right)$$

**诺顿等值形式**：

$$I(x, V) = YV$$

**PMU 总相量误差（TVE）**：

$$\text{TVE} = \sqrt{\frac{(\hat{X}_r - X_r)^2 + (\hat{X}_i - X_i)^2}{X_r^2 + X_i^2}} \times 100\%$$

## 关键技术挑战

### 挑战 1：PMU 动态相量估计的频率偏移鲁棒性

传统 DFT 计算快但默认频率锁定在标称值，在频率偏离标称值时产生非同步采样和频谱泄漏。泰勒级数模型能描述动态变化，却未充分处理大范围频偏。插值 DFT 可提高频率估计精度，但依赖窗函数和较多采样点，运算负担更高。

应对策略是多频率相量模型（MFPM），用中心粗估频率及其两侧按 $\Delta f$ 间隔分布的若干子相量共同拟合动态基波信号。

### 挑战 2：保护系统闭环的 EMT 接口结构

EMTP 原本主要面向电力系统元件暂态，保护算法需要逐时步读取系统量并把跳闸决策送回主程序；同时 CT、CVT 本身会改变继电器所见波形。这形成了"系统暂态计算—互感器变换—继电器算法处理—保护决策反馈—系统状态更新"的闭环，其接口设计是主要挑战。

### 挑战 3：EMT-相量域混合仿真的模式切换一致性

EMT 侧有三相瞬时量、离散事件和快暂态，相量侧通常处理基波相量和较慢动态，二者在故障清除后可能状态不一致；若直接切换，边界电压、电流或设备状态会跳变。

应对策略是带判据的模式切换机制：先等待故障后延迟，再检查边界电压变化率是否低于阈值，随后比较详细系统相量模型与外部系统边界电压偏差，要求持续若干工频周期满足容差。

### 挑战 4：多控制器协调的负交互

多控制器协调可能产生负交互，尤其是在 HVDC、FACTS、PSS 和切负荷同时动作时。在 EMT 仿真中需要显式建模通信延迟和控制器带宽交互。

### 挑战 5：离线 EMT 模型到实时 HIL 的移植

离线 EMT 模型迁移到实时 HIL 环境需要：模型库兼容、控制模块等价、用户代码重建、原理图修改、信号校核和性能约束。软件层问题如缺少导出/导入和自动 place-and-route 会导致移植代价很高。

## 典型工作流

### WAMPAC 保护系统 EMT 闭环测试流程

1. **布置测点**：选择能覆盖关键联络线、发电机群、换流站、负荷中心和振荡通道的测量位置
2. **EMT 系统建模**：建立包含 CT/CVT 暂态模型的完整电网 EMT 模型
3. **保护算法接口开发**：编写继电器算法的 FORTRAN 子程序，与 EMTP 主程序链接
4. **闭环仿真执行**：EMTP 主程序产生时域电压、电流数据 → 用户子程序处理 → 输出动作判据 → 传回 EMTP
5. **结果验证**：比较保护动作前后电网状态变化，验证保护逻辑正确性
6. **参数优化**：调整保护整定值，研究 CT/CVT 暂态对保护输入的影响

### 离线到实时移植工作流

参考 [[lessons-learned-in-porting-offline-large-scale-power-system-simulation-to-real-t]]：

1. 在实时工具中实例化或导入对应电气与控制组件
2. 逐项迁移参数和连接
3. 对没有直接实时等价物的离线组件，用基础模块合成或用 UCM 手写实现
4. 信号检查和系统行为验证
5. 根据实时仿真器能力作必要修改，同时尽量不改变仿真结果
6. 波形比较和性能评估
