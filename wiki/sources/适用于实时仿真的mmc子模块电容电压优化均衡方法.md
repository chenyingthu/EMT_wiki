# 适用于实时仿真的MMC子模块电容电压优化均衡方法

**作者**: 樊强 等
**年份**: 2026
**来源**: `30/樊强 等 - 2020 - 适用于实时仿真的MMC子模块电容电压优化均衡方法.pdf`

## 摘要

针对并行全比较算法存在的高空间复杂度问题，提出一种适用于模块化多电平换流器（MMC）实时仿真的电容电压均衡优化方法。在子模块电容电压的排序方面，采用分组排序的均压策略，组内子模块采用并行全比较算法以减少排序时间，组间子模块根据所定义的能量平衡因子计算结果来平衡其电容电压值。此外，提出一种子模块电容电压值重构方法处理含相同电容电压值的子模块排序问题。在触发脉冲产生方面，提出一种串、并行触发结合的混合触发模式，将触发脉冲产生环节消耗的时间与 MMC 各桥臂子模块总数解耦。在 PSCAD／EMTDC 仿真程序和低功率 MMC 物理样机平台的阀级控制器中验证了所提 MMC 子模块电容电压优化均衡方法的逻辑有效性和工程实用性，证明了所提方法在保证低时间复杂度的同时，其空间复杂度亦有所降低。

## 研究解读

### 1. 需求、对象、挑战与贡献

这篇论文面向高电平MMC在电磁暂态实时仿真和阀级控制器硬件实现中的一个具体瓶颈：每个仿真步长内既要完成桥臂子模块电容电压排序，又要生成投切触发脉冲，否则实时步长会被排序和脉冲分配逻辑拖慢。研究对象是MMC各桥臂的N个半桥子模块电容电压均衡算法，重点不是主电路等值，而是排序与触发环节在FPGA上的时空复杂度。难点在于：传统冒泡、希尔等串行排序时间复杂度高；并行全比较算法可在4个时钟完成排序，但需要大量比较器，空间复杂度随子模块数快速上升；同时，相同电容电压值会导致排序位置不唯一，触发脉冲生成也可能随N增长而增加延迟。本文的贡献是把全桥臂全比较改为“按编号分组+组内并行全比较+组间能量平衡因子排序”，用局部并行换取固定排序时钟和较少比较器；用由最大值、最小值和中间值构成的组间平衡因子替代组内全量求和；用电压值重构处理相同电压排序；并提出组内串行、组间并行的混合触发模式，使触发脉冲产生时间不再直接随桥臂子模块总数N增长。

### 2. 模型、算法与实现技术

算法输入是每个MMC桥臂N个子模块的电容电压序列、桥臂电流方向以及需要投入的子模块数；输出是排序后的相对电压序列和各子模块投切触发信号。作者先按子模块编号把N个子模块划分为n组，每组m个，满足N=nm。组内仍采用并行全比较：每个电压与组内其他m-1个电压同时比较，将“大于”结果累加得到该元素在组内的排序位置，因此能利用FPGA并行逻辑压缩排序时间。组间不再比较全部子模块，而是为每组计算能量平衡因子f=UCmax+UCmin+UCmid，即取组内最大、最小和中间电压之和表示该组整体能量状态，再对各组f值进行并行全比较并调整组序。比较器数量由全桥臂全比较的N(N-1)类需求，变为组间比较n(n-1)加n个组内比较m(m-1)，文中写为Ncom=n(n-1)+(N/n)(N/n-1)n，并给出最优分组数的理论表达式。针对相同电压导致比较结果相同的问题，论文采用电压值重构：通过Verilog位拼接等方式给相同整数或定点电压值附加可区分的低位信息，使排序位置唯一。触发环节则根据桥臂电流方向遵循充电投低压、放电投高压的均衡原则，组内串行确定具体投入单元，组间并行决定整组投入或旁路，从机制上把触发延迟主要限制在组内规模m而非总规模N。

### 3. 验证、优势与不足

作者采用两类验证。第一类是在PSCAD/EMTDC中建立双端MMC-HVDC系统，额定直流电压为±200 kV，额定容量500 MW，每桥臂子模块数N=100，分组数取n=10、每组m=10；测试包括稳态运行和有功功率阶跃等工况，并与冒泡排序、并行全比较算法比较电容电压波动和开关频率等指标。页面抽取结果给出稳态下本文方法电容电压范围为2.917～3.490 kV，冒泡排序为2.917～3.502 kV，并行全比较为2.917～3.480 kV；开关频率分别约为1140 Hz、1220 Hz和1110 Hz，说明均衡效果与已有排序接近。第二类是在低功率MMC物理样机的阀级控制器中实现，采用Quartus II开发、Cyclone V-5CEFA9F23I7 FPGA，样机额定约120 V/1 kW，12个全桥子模块，n=3、m=4，并用示波器观测电压电流和子模块电容电压。优势主要体现在硬件资源和定步长实时性：排序过程文中统计为10个时钟，包含组内排序4个、组间排序4个、平衡因子计算1个、组间调整1个，且与N无关；当N=100、n=10时比较器数量由并行全比较的9900个降为990个。边界也很明确：验证只覆盖文中给出的MMC-HVDC仿真和低功率样机，未展示不同实时步长、不同FPGA型号、故障穿越、大规模工程控制器闭环硬件在环等场景；组间平衡因子用三点电压代表整组能量，其在强不均匀分布或异常子模块状态下的鲁棒性未见系统对比。

### 4. 价值、认知与可复用场景

这项工作的价值在于把MMC实时仿真中的“电容电压均衡”从单纯控制问题具体化为FPGA可实现的排序、资源占用和触发延迟问题。它表明，实时仿真不一定要在全桥臂范围内做全比较；通过组内精确排序、组间低维能量指标排序，可以在保持近似均衡效果的同时显著降低比较器需求，并使排序时钟数固定。该方法适合被后续MMC EMT实时仿真、阀级控制器原型、FPGA排序资源优化、子模块触发分配逻辑等页面复用，尤其适用于需要解释“为什么并行全比较时间快但硬件资源重”的研究脉络。不适合直接外推为所有MMC拓扑、所有故障工况或所有硬件平台上的最优均压策略；分组数、组内规模、平衡因子形式和电压重构位宽都应随具体实时步长、定点格式和FPGA资源重新校核。

## 方法细节

### 方法概述

本文提出一种适用于MMC实时仿真的分组排序均压策略，将N个子模块分为n组（每组m个，N=nm）。组内采用并行全比较算法（4时钟周期）实现快速排序，组间采用基于能量平衡因子f=U_Cmax+U_Cmin+U_Cmid的排序策略。针对相同电压值导致的排序失效问题，提出电压重构方法（通过位拼接添加小数位区分，如1.5→1.503,1.504）。在触发脉冲产生环节，提出串并行混合触发模式：组内采用串行触发（根据"充小放大"原则确定导通个数），组间采用并行触发（整组投切），使触发时间仅与组内子模块数m相关，与总子模块数N解耦。

### 数学公式

**公式1**: $$$N_{com} = n(n-1) + \frac{N}{n}(\frac{N}{n}-1)n$$$

*所需比较器总数计算公式，其中n为分组数，N为子模块总数。第一项为组间排序所需比较器，第二项为组内排序所需比较器总和。*

**公式2**: $$$n_0 = \sqrt[3]{\frac{N^2}{4} + \sqrt{\frac{N^4}{216} - \frac{1}{46656}}} + \sqrt[3]{\frac{N^2}{4} - \sqrt{\frac{N^4}{216} - \frac{1}{46656}}} + \frac{1}{6}$$$

*最小比较器数量对应的理论最优分组数，通过对N_com求导并令dN_com/dn=0得到极值点。*

**公式3**: $$$f = U_{Cmax} + U_{Cmin} + U_{Cmid}$$$

*新型能量平衡因子计算公式，用于组间排序，将组内子模块电容电压的最大值、最小值和中间值之和作为该组的平衡因子。*

**公式4**: $$$N_{commin} = n_0(n_0-1) + \frac{N}{n_0}(\frac{N}{n_0}-1)n_0$$$

*当分组数取最优值n0时，系统所需的最少比较器数量。*

### 算法步骤

1. 步骤1-子模块分组：将各桥臂中N个子模块按序号分为n组（U_Co1, U_Co2, ..., U_Con），每组含m=N/n个子模块，该步骤不占用FPGA时钟资源。

2. 步骤2-组内并行排序：对每组子模块采用并行全比较算法，将组内每个子模块电压与其余m-1个子模块并行比较（大于为1小于为0），通过累加比较结果确定排序位置，得到组内有序序列U_Cni和对应的能量平衡因子f_i。

3. 步骤3-电压值重构：针对组内存在的相同电压值问题，通过Verilog位拼接运算符调整序列位置，为相同电压值添加微小差异（如1.5,1.5,1.5→1.503,1.504,1.505），确保排序唯一性。

4. 步骤4-平衡因子计算：根据公式f=U_Cmax+U_Cmin+U_Cmid计算各组能量平衡因子，耗时1个时钟周期。

5. 步骤5-组间排序：采用并行全比较方法对各组平衡因子进行排序，并根据排序结果调整组间顺序（U_Cn1, U_Cn2, U_Cn3），得到桥臂N个子模块的相对有序序列U_Cn，耗时4个时钟。

6. 步骤6-混合脉冲触发：根据桥臂电流方向和导通子模块数n_on，采用串行模式确定组内具体导通的子模块（NCni中具体位），采用并行模式确定组间整组投切（NCni=0旁路整组，NCni=1投入整组）。

### 关键参数

- **N**: 桥臂子模块总数（仿真中N=100，实验中N=12）

- **n**: 分组数（仿真取n=10，实验取n=3）

- **m**: 每组子模块个数（m=N/n，仿真m=10，实验m=4）

- **n0**: 理论最优分组数（N=100时n0=17.2681）

- **f**: 能量平衡因子

- **T_sort**: 排序总时间复杂度（10个时钟周期）

- **U_C_old**: 原始子模块电容电压序列

- **U_C_new**: 重构后的子模块电容电压序列

## 量化发现

- 时间复杂度：排序过程共需10个时钟周期（组内排序4个时钟+组间排序4个时钟+平衡因子计算1个时钟+组间调整1个时钟），且与子模块总数N无关，适用于定步长实时仿真。
- 空间复杂度优化：当N=100时，并行全比较算法需9900个比较器，本文算法仅需990个比较器（n=10时，N_com=10×9+10×9×10=990），比较器数量降低为原来的1/10。
- 最优分组数：N=100时理论最优分组数n0=17.2681，实际取n=10（为使m>3），此时比较器数量接近理论最小值。
- 电容电压波动：稳态工况下波动范围为[2.917, 3.490]kV（标幺值约0.97~1.16 p.u.），与冒泡排序的[2.917, 3.502]kV相比，均衡精度相当。
- 开关频率：本文算法为1140Hz，介于冒泡排序（1220Hz）和并行全比较（1110Hz）之间，处于可接受范围。
- 触发时间解耦：采用混合触发模式后，触发脉冲产生时间仅取决于组内子模块数m（串行触发），与桥臂总子模块数N解耦，大幅减少了高电平MMC的触发延迟。

## 关键公式

### 比较器数量计算公式

$$$N_{com} = n(n-1) + \frac{N}{n}(\frac{N}{n}-1)n$$$

*用于计算分组排序策略所需的逻辑资源，通过优化n值最小化FPGA中自适应逻辑块（ALM）的使用。*

### 能量平衡因子公式

$$$f = U_{Cmax} + U_{Cmin} + U_{Cmid}$$$

*用于组间排序，替代传统的组内电压求和方法，在保证能量平衡的同时降低计算量。*
