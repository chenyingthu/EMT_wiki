# Lessons learned in porting offline large-scale power system simulation to real-time for wide-area monitoring, protection and control

**作者**: P. Le-Huy
**年份**: 2023
**来源**: `25/Le-Huy 等 - 2023 - Lessons learned in porting offline large-scale power system simulation to real-time for wide-area mo.pdf`

## 摘要

本文提出了一套从离线电磁暂态（EMT）仿真向实时（RT）仿真移植的系统性工程方法。核心在于统一图形用户界面（GUI）与自动化网表导入导出机制，解决传统手动布局布线效率低下的问题。通过开发模型兼容层，将离线工具（如EMTP）的模型自动映射至实时工具（如HYPERSIM）的原生模型、预生成用户代码模型（UCM）或等效子系统。针对初始化与控制建模差异，剔除离线仿真中为加速稳态而引入的数值技巧，避免实时环境下的长期数值不稳定；同时采用单位延时模块打破控制系统中的代数环，确保实时求解器的确定性。最终通过波形对比与参数/数值阻尼精细校准，实现大规模电网在实时环境中的高精度、高稳定性运行。

## 研究解读

### 1. 需求、对象、挑战与贡献

工程需求来自Hydro-Québec内部的大规模电力系统研究：原本多数EMT研究在离线EMTP类工具中完成，而实时仿真主要用于HVDC和补偿装置投运相关HIL测试；随着WAMPAC研究需要把电力系统、通信仿真器以及真实控制/保护设备联接起来，大系统必须从离线环境迁移到实时HYPERSIM环境。研究对象不是新的电磁暂态求解器，而是“大规模离线EMT模型到实时HIL模型”的移植过程。难点在于两套软件环境组件、参数、连线、控制模块和实时约束不同；复杂内部控制多时，手工重建、核对信号和处理无直接等价元件会变成专家密集型任务。本文贡献是以Hydro-Québec多年工具开发和工程移植经验为基础，总结软件接口、模型替换、原理图修改、波形比较和实时性能评估前的准备步骤，为大规模WAMPAC实时仿真提供可复用的移植经验，而非单纯报告某个算例精度提升。

### 2. 模型、算法与实现技术

从所给原文看，本文提出的是一套工程化移植流程和建模实践，而不是带有明确数学公式的新算法。核心输入是离线EMT仿真模型，包括电气元件、控制/保护逻辑、参数、连接关系和可能的用户自定义模型；目标输出是可在实时仿真器中执行、并可接入HIL设备或通信仿真器的等效模型。流程机制包括：在实时工具中实例化或导入对应电气与控制组件；逐项迁移参数和连接；对没有直接实时等价物的离线组件，用基础模块合成或用UCM手写实现；随后进行信号检查、系统行为验证，并根据实时仿真器能力作必要修改，同时尽量不改变仿真结果。文中还强调软件层问题，如缺少导出/导入和自动place-and-route会导致历史上移植代价很高，因此工具链支持、模型库兼容和转换脚本优化是实现关键。所给片段未报告可核验的具体公式、状态方程、时间步长、调度算法或误差计算式；因此应把它理解为离线到实时EMT模型转换的实施框架和经验清单，而不是可独立复现的数值算法说明。

### 3. 验证、优势与不足

作者说明将给出一个用于WAMPAC实时仿真探索的大规模电力系统移植案例，并在波形比较和性能评估之前讨论建模细节及对原始仿真原理图所需的修改。验证思路是以离线EMT模型为基线，将其迁移到Hydro-Québec实时仿真工具后，检查信号、比较动态波形，并评估实时运行性能；工具背景明确为EMTP类离线环境和HYPERSIM实时环境，应用场景包括与通信仿真器、控制和保护设备联合仿真。优势主要体现在把过去依赖手工实例化、参数化、连线和专家复核的移植任务，转化为可总结、可脚本化、可由工具链支持的工程流程，尤其适合带复杂控制的大系统HIL研究。限制是：所给原文片段未报告可核验的数值结果、系统规模、误差指标、实时步长、CPU/FPGA资源占用或与RTDS导入工具的实测对比；RTDS工具只作为文献背景提到。因而不能据此宣称“零误差”“显著缩短周期”或某类模型普遍可自动转换。结论应限定在Hydro-Québec工具链、其大规模WAMPAC移植案例以及文中实际覆盖的模型和修改范围内。

### 4. 价值、认知与可复用场景

这项工作的价值在于把“离线EMT模型能否直接用于实时HIL”这一常被低估的问题具体化：实时移植不是简单换求解平台，而是模型库兼容、控制模块等价、用户代码重建、原理图修改、信号校核和性能约束共同作用的系统工程。它适合被后续关于WAMPAC HIL、EMTP到HYPERSIM转换、复杂控制系统实时化、仿真工具互操作性和大系统模型维护的页面复用，作为问题框架和实施检查表。不适合被外推为通用自动转换算法，也不应在缺少原文表图支撑时用于比较不同实时仿真器的精度、速度或最大系统容量。

## 主要发现

- 离线与实时工具结果差异源于模型参数或数值阻尼设置不同，需精细校准。
- 离线仿真中的稳态加速初始化技巧在实时环境中易引发长期数值不稳定问题。
- 模型兼容层与统一GUI接口显著降低大规模电网移植人工成本并提升可读性。

## 方法细节

### 方法概述

本文提出了一套从离线电磁暂态（EMT）仿真向实时（RT）仿真移植的系统性工程方法。核心在于统一图形用户界面（GUI）与自动化网表导入导出机制，解决传统手动布局布线效率低下的问题。通过开发模型兼容层，将离线工具（如EMTP）的模型自动映射至实时工具（如HYPERSIM）的原生模型、预生成用户代码模型（UCM）或等效子系统。针对初始化与控制建模差异，剔除离线仿真中为加速稳态而引入的数值技巧，避免实时环境下的长期数值不稳定；同时采用单位延时模块打破控制系统中的代数环，确保实时求解器的确定性。最终通过波形对比与参数/数值阻尼精细校准，实现大规模电网在实时环境中的高精度、高稳定性运行。

### 数学公式

**公式1**: $$y[k] = u[k-1]$$

*单位延时模块打破代数环的离散化表达，用于实时控制系统中避免非确定性迭代求解，确保单步计算的确定性*

**公式2**: $$M_{RT} = \mathcal{C}(M_{offline}, \theta_{param})$$

*模型兼容层映射函数，将离线模型及其参数转换为实时等效模型，支持默认锁定与按需解锁微调*

### 算法步骤

1. 从离线仿真工具导出包含电气拓扑、元件参数及自动布局布线（Place & Route）信息的完整网表文件，确保结构信息完整传递。

2. 将网表导入实时仿真GUI，系统自动触发模型兼容层，解析离线模型并映射至实时原生模型、UCM或等效子系统；对不支持的模型标记警告并锁定默认等效结构，保留原始建模信息优先级。

3. 审查并移除离线仿真中用于加速稳态收敛的初始化技巧（如强制初始值注入、非物理阻尼或黑盒模型预设），防止在实时长周期仿真中引发数值漂移或长期不稳定。

4. 识别控制系统中的代数环（电压/电流源与反馈控制直接耦合），插入单步延时模块以适配实时求解器的非迭代特性，并在离线与实时模型中同步更新以保证逻辑一致性。

5. 解锁关键等效模型的参数接口，根据离线基准波形进行数值阻尼系数、解耦设置及控制参数的精细校准，消除因默认参数导致的过度阻尼或异常泄漏。

6. 执行波形对比与性能评估，验证实时仿真在电气暂态（μs/ms级）与机电暂态（秒级）下的动态响应一致性，确认满足广域监测保护与控制（WAMPAC）硬件在环测试需求。

### 关键参数

- **数值阻尼系数**: 需根据离线基准精细校准，避免过度阻尼或异常泄漏导致测量误差

- **延时步长**: 1个仿真步长（用于打破代数环，确保实时求解器确定性）

- **兼容层锁定状态**: 默认锁定（Locked），支持按需解锁（Unlocked）进行参数微调

- **初始化时间常数**: 电气暂态μs/ms级可容忍初始扰动，机电暂态需秒级稳定，避免离线加速技巧移植

## 量化发现

- 系统规模量化：包含1666个三相母线、6180个单相节点、13626个RLC元件、111台电机、432条线路/电缆、338台变压器、86台调速器、81套励磁系统、54套稳定器、10台静止补偿器、6座风电场、6套HVDC及165个动态负荷。
- 模型一致性原则：当离线与实时工具采用完全相同的模型结构、参数设置、解耦策略与数值阻尼时，仿真结果误差为0%，波形完全重合。
- 代数环处理：引入1步延时后，实时求解器计算时间确定性提升，避免迭代发散，控制信号响应延迟严格控制在1个仿真步长内。
- 初始化差异影响：离线加速初始化技巧若直接移植，会导致实时仿真在数十秒至分钟级时间尺度出现数值漂移或长期不稳定，必须剔除或重构。

## 验证详情

- **验证方式**: 离线与实时仿真波形对比分析、参数敏感性测试与长期稳定性评估
- **测试系统**: 2023年魁北克（Hydro-Québec）大规模电力系统EMT等效模型（含HVDC、风电、STATCOM及复杂控制保护系统）
- **仿真工具**: EMTP（离线基准）、HYPERSIM（实时仿真平台）
- **验证结果**: 通过模型兼容层自动映射与关键参数/数值阻尼校准，实时仿真成功复现离线基准波形。验证表明，差异主要源于模型参数、数值阻尼或解耦设置不一致；剔除离线初始化加速技巧并规范代数环处理后，系统在WAMPAC应用场景下实现高精度、高稳定性的实时运行，满足硬件在环测试要求。
