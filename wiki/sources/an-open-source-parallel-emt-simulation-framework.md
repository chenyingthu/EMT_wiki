# An open-source parallel EMT simulation framework

**作者**: Min Xiong, Bin Wang, Deepthi Vaidhynathan, Jonathan Maack, Matthew J. Reynolds, Andy Hoke, Kai Sun, Deepak Ramasubramanian, Vishal Verma, Jin Tan
**年份**: 2024
**来源**: `07&08/Xiong et al. - 2024 - An open-source parallel EMT simulation framework.pdf`

## 摘要

本文介绍 ParaEMT，一个面向大规模逆变器型资源（IBR）电网的开源 Python EMT 仿真框架。论文的核心问题是：商业 EMT 软件虽然成熟，但源码和并行架构不便于研究者修改；而高保真 EMT 模型需要 50-100 μs 小步长，大系统仿真存在明显计算瓶颈。ParaEMT 以节点法网络方程、伴随电路离散、BBD 并行矩阵分解、MPI 分布式计算、Numba JIT 和 PSSE 潮流初始化为主线，提供模型库、快照、降采样和标准 DLL 接入能力。

## 研究解读

### 1. 需求、对象、挑战与贡献

实际需求是：高比例逆变器型资源接入后，电网稳定问题包含从直流到数百赫兹的快速电磁暂态过程，规划、控制设计和保护整定需要可修改、可复现实验的大规模EMT仿真平台。研究对象是开源Python框架ParaEMT，而不是某个单一元件模型。难点在于EMT通常需50–100 μs小步长，大系统每步都要更新动态元件并求解网络代数方程；商业软件虽成熟，但源码、并行结构和新算法接入不便。本文贡献在于把节点法网络方程、伴随电路离散、PSSE潮流初始化、标准DLL接入、Numba加速和BBD/MPI分布式并行整合成一个公开、模块化框架，使研究者能在Python层面修改模型、算法和并行求解流程，并在万节点级案例中展示并行可扩展性。

### 2. 模型、算法与实现技术

ParaEMT以节点电压为网络核心未知量，每个时间步求解离散网络方程Gv(t)=i(t)+i_hist(t-Δt)。其中G是由线路、变压器、RLC支路及等效诺顿导纳组成的节点导纳矩阵，v为三相瞬时节点电压，i为外部或动态元件注入电流，i_hist保存伴随电路的历史项。电感、电容用梯形积分离散为等效电导和历史电流，并按原文公式为电感并联Rp≈40L/(3Δt)、为电容串联Rs≈3Δt/(40C)，用于抑制梯形法数值振荡。动态元件状态由模型库更新，输入通常是端口电压、电流或控制参考，输出为下一步的等效电流源、内部状态和控制量；文中还支持IEEE/CIGRE标准DLL黑盒IBR模型。实现上，框架读取PSSE .raw潮流和动态参数文件完成稳态到三相EMT初值映射；底层循环用Numba JIT编译；大规模求解将稀疏G划分为加边块对角结构，各MPI进程处理局部块分解和代入，再通过角块耦合完成全局节点电压求解。

### 3. 验证、优势与不足

作者用多类算例验证准确性和效率：改进型Kundur双区域系统用于同步机、励磁和调速等慢动态对比；含EPRI IBR DLL的3节点系统用于验证标准DLL黑盒逆变器模型接入；WECC 240节点系统用于与PSSE、PSCAD/EMTDC比较动态曲线和运行时间；合成10,024节点系统在NREL Eagle高性能计算平台上测试BBD/MPI强扩展。已抽取结果显示，WECC 240节点、50 μs步长、1秒仿真时，ParaEMT单核耗时29秒，PSCAD单核90秒；但8核并行时ParaEMT为28秒，低于单核提升很小且慢于PSCAD 8核15秒，说明中等规模系统通信和角块开销会抵消BBD收益。10,024节点案例中，MPI秩和分区数增至84时，1秒仿真最低约90–110秒，峰值加速比15–18倍。优势主要是开放源码、Python可扩展、能接入DLL并在大规模系统上利用HPC；限制是原文明确指出其尚缺成熟GUI、文档、help-desk、广泛设备库和切换插值，且验证未证明其可替代商业EMT全功能软件。

### 4. 价值、认知与可复用场景

这项工作的重要认知是：Python并不必然等于低性能EMT，只要把时间步循环JIT化、把网络矩阵组织为适合分布式求解的BBD结构，并把初始化和模型接口工程化，就能成为研究型大规模EMT平台。它适合后续复用在IBR-rich电网稳定性研究、并行矩阵求解算法测试、开源EMT模型库扩展、标准DLL接口验证、PSSE潮流到EMT初始化流程开发等页面或工程原型中。不宜外推为“所有系统并行都会加速”或“ParaEMT精度全面等同PSCAD/EMTP-RV”；从验证范围看，其优势更集中在可修改性和万节点级并行实验，而不是商业软件完整功能替代。

### 证据边界

- 来自原文：EMT仿真通常需要50–100 μs小步长；商业PSCAD、EMTP-RV成熟但不便用户修改源码和并行结构；ParaEMT用Python实现并强调开源、模块化和并行计算。
- 来自原文/抽取结果：验证系统包括改进型Kundur双区域系统、含EPRI IBR DLL的3节点系统、WECC 240节点系统和合成10,024节点系统；对比工具包括PSSE和PSCAD/EMTDC。
- 来自抽取结果：WECC 240节点单核1秒仿真ParaEMT为29秒、PSCAD为90秒；8核ParaEMT为28秒、PSCAD为15秒；10,024节点案例峰值加速比15–18倍、1秒仿真约90–110秒。
- 边界：动态响应“接近”主要来自曲线对比，当前页面未提供完整误差范数、最大偏差、事件后稳态误差等可核验数值，因此不能声称精度全面优于或等同商业软件。
- 边界：BBD/MPI加速效果依赖系统规模、分区质量、角块大小和通信开销；240节点案例已经显示并行收益很弱，因此不能把万节点案例的15–18倍加速外推到中小规模系统。
- 边界：原文明确承认ParaEMT仍缺成熟GUI、帮助文档、help-desk、广泛设备库和切换插值等商业软件能力；模型库中分布参数线路、动态负荷、饱和变压器、更多IBR模型等仍属后续扩展方向。

## 方法细节

### 方法概述

ParaEMT框架采用节点分析法构建电力系统网络方程，利用梯形积分法将微分方程离散化为代数方程，并引入人工阻尼电阻以抑制数值振荡。为突破大规模网络求解瓶颈，框架采用加边块对角矩阵（BBD）分解技术，将全局稀疏导纳矩阵自动划分为多个对角子块与耦合角块，结合MPI分布式内存并行协议实现前代与回代运算的自动化并行加速。底层计算循环通过Numba即时编译（JIT）技术转化为机器码，显著提升Python执行效率。系统初始化阶段自动解析PSSE潮流文件并映射至三相网络，动态元件采用欧拉积分法更新状态。整体架构模块化设计，支持IEEE/CIGRE标准DLL黑盒模型无缝接入，具备快照恢复、结果降采样及跨平台运行能力，专为高比例逆变器型资源（IBR）并网的大规模电磁暂态仿真优化。

### 数学公式

**公式1**: $$$Gv(t) = i(t) + i_{hist}(t - \Delta t)$$$

*离散化后的网络节点电压方程，G为节点导纳矩阵，v(t)为节点电压向量，i(t)为外部注入电流向量，i_hist为历史电流向量，用于每个仿真步长求解全网电压。*

**公式2**: $$$R_p \approx \frac{40L}{3\Delta t}$$$

*并联于电感L的人工阻尼电阻，用于消除梯形积分法在电感支路中引发的数值振荡。*

**公式3**: $$$R_s \approx \frac{3\Delta t}{40C}$$$

*串联于电容C的人工阻尼电阻，用于消除梯形积分法在电容支路中引发的数值振荡。*

### 算法步骤

1. 数据加载与拓扑转换：读取预定义的PSSE .raw潮流文件与XLS动态参数文件，自动将单相/三相数据转换为全三相瞬时电压与电流网络拓扑结构。

2. 潮流初始化与状态映射：基于收敛的稳态潮流解，自动计算并初始化同步发电机、励磁系统、调速器、负荷及IBR等动态元件的初始状态变量与控制参考值。

3. 动态状态更新：在每个仿真步长内，采用欧拉积分法求解动态元件的微分方程，更新内部状态变量，并计算对应的等效诺顿电流源。

4. 网络方程组装：利用梯形积分法将电感、电容等储能元件离散为伴随电路，结合人工阻尼电阻构建全局导纳矩阵G与历史电流向量i_hist。

5. BBD分解与MPI并行求解：调用BBD算法将稀疏矩阵G自动划分为对角块与加边块，通过MPI通信协议将各子块分配至不同计算节点，并行执行局部LU分解、前代与回代运算，最终求解节点电压v(t)。

6. 结果处理与循环推进：应用Numba JIT编译加速底层数值循环，根据用户设定步长进行结果降采样或保存快照；若检测到扰动或故障，触发重初始化流程，循环推进至仿真结束时间。

### 关键参数

- **仿真步长**: 50 μs

- **人工阻尼电阻系数**: 40/3 (用于L和C的数值振荡抑制)

- **MPI并行进程数**: 1至84

- **测试系统规模**: 240节点至10,024节点

- **核心依赖库**: Python 3.7.13, Numba 0.57.1, SciPy 1.7.0, NumPy 1.21.0

- **积分方法**: 梯形积分法(网络离散), 欧拉法(动态状态更新)

## 仿真结果

### 仿真测试

## 量化发现

- 在WECC 240节点系统中，ParaEMT单核串行仿真耗时29秒，较PSCAD（90秒）提速约210%。
- 在10,024节点合成系统中，利用HPC与MPI并行计算，BBD分解实现15-18倍的峰值加速比，1秒仿真时间压缩至90-110秒。
- Numba即时编译技术使Python底层循环计算效率显著提升；在该240节点单核算例中，ParaEMT报告耗时低于PSCAD。
- BBD并行算法在中等规模系统（240节点）中因角块规模与同步通信开销导致加速比受限（仅1.03倍），但在万节点级系统中扩展性优异。
- 框架支持IEEE/CIGRE标准DLL模型集成，在3节点测试系统中复现EPRI提供的跟网型IBR模型动态响应，验证了黑盒模型兼容性入口。

## 适用边界

- 适用于研究型、可修改、可嵌入新算法的大规模 EMT 仿真实验，尤其是需要 HPC/MPI 并行探索的 IBR-rich 系统。
- 当前 ParaEMT 不是 PSCAD/EMTP-RV 的功能等价替代品；原文明确指出其缺少成熟 GUI、帮助文档、help-desk、广泛设备库和切换插值等商业工具能力。
- 模型库仍在扩展中，文中提到分布参数线路、动态负荷、饱和变压器、更多跟网/构网逆变器等仍属后续开发方向。
- BBD 并行不保证所有系统都加速：240节点中等规模算例几乎没有并行收益，只有更大规模系统才体现强扩展性。

## 验证详情

- **验证方式**: 仿真对比验证（与商业软件PSSE、PSCAD进行动态响应曲线与计算耗时对比）
- **测试系统**: 改进型Kundur双区域系统、含EPRI IBR DLL的3节点系统、WECC 240节点系统、合成10,024节点大规模系统
- **仿真工具**: ParaEMT (Python/Numba/MPI), PSSE, PSCAD/EMTDC
- **验证结果**: ParaEMT在慢动态（同步机/励磁/调速器）与快动态（IBR控制）仿真中与PSSE/PSCAD参考结果接近，支持其模型库、初始化和DLL接口的可用性。在计算效率方面，单核240节点算例显示Numba带来的收益；万节点级系统通过BBD+MPI并行实现15-18倍峰值加速，但中等规模系统的并行通信开销会抵消收益。
