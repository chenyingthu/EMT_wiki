# A Hierarchical Low-Rank Approximation Based Network Solver for EMT Simulation

**作者**: Zhang 等
**年份**: 2020
**来源**: `01/Zhang 等 - 2021 - A Hierarchical Low-Rank Approximation Based Network Solver for EMT Simulation.pdf`

## 摘要

本文提出一种基于分层低秩近似的网络方程快速求解器，旨在突破传统EMT仿真中网络求解的计算瓶颈。传统方法依赖稀疏LU分解与前代回代，计算复杂度高且难以并行。该方法首先将节点电压方程重构为矩阵向量乘法形式$v(t)=G^{-1}i(t)$。随后，利用电力网络导纳矩阵的拓扑特性，设计自底向上的图划分算法，将母线按电气距离聚类并构建平衡二叉树。针对树中表征远距离或弱耦合母线组交互的非对角子矩阵，采用截断奇异值分解(SVD)进行低秩近似，仅保留主导奇异值及对应左右奇异向量，使近似误差严格低于预设阈值。在仿真每个时间步，通过遍历该层次数据结构执行快速矩阵向量乘法，将单步求解复杂度从降至，在维持数值精度的同时显著提升计算效率并具备高度并行化潜力。

## 研究解读

### 1. 需求、对象、挑战与贡献

工程需求来自大规模电网EMT仿真的在线化和准实时化：原文指出商业EMT程序在87000节点系统上仿真1秒暂态可耗时约80小时，且网络方程求解占总计算时间80%-97%。研究对象不是元件建模本身，而是每个EMT时间步中的网络节点方程求解，即由导纳矩阵G和注入/历史电流得到节点电压。难点在于传统稀疏LU虽适合大规模稀疏网络，但前代/回代具有较强顺序性，难以通过并行继续扩展；同时某些密集区域会形成很大的块，限制网络分解收益。本文的创新是把求解Gv=i转化为对G的逆矩阵进行分层表示和快速矩阵向量乘法，利用“远距离母线组之间相互作用子矩阵通常稀疏且可低秩近似”的结构性观察，通过图划分和分层低秩近似把单步网络求解目标推进到O(N log N)复杂度，并给出时间、内存和误差分析。

### 2. 模型、算法与实现技术

方法的核心接口量是每个时间步的总电流向量i(t)，由外部注入电流和历史电流组合而成；输出是节点电压向量v(t)。网络方程写为Gv(t)=i_in(t)+i_his(t-Δt)，再转为v(t)=G^{-1}i(t)，因此在线阶段的关键计算变成近似逆矩阵与向量相乘。算法先根据导纳矩阵对应的网络图对母线进行分层聚类，构造近似平衡的二叉层次结构，使相近母线留在较小块内，远距离母线组的交互出现在非对角块中。对这些交互子矩阵，使用截断SVD表示为A≈Σ_{k=1}^r σ_k u_k v_k*，并用被截去奇异值的Frobenius范数控制误差；若子块不满足低秩或规模阈值，则继续递归分块或直接存储。在线求解时不再做LU回代，而是遍历层次结构：显式块直接乘向量，低秩块先算V*x、再乘奇异值和U，从而以较少乘加运算累积得到全局电压。机制上，图划分决定哪些相互作用可被低秩压缩，SVD截断决定速度与误差的折中，层次树保证递归深度与乘法复杂度接近O(N log N)。

### 3. 验证、优势与不足

作者用文献中的39母线系统、179母线系统，以及由这两个系统构造的大规模算例进行数值验证。精度方面，论文将所提求解器的EMT电压暂态结果与商业软件EMTP-RV对比；性能方面，与当时先进的稀疏LU分解型网络求解器比较网络求解时间，并报告最高可达2.8倍加速。指标主要包括暂态波形一致性、网络求解速度、理论复杂度、内存和误差分析。优势在于该方法直接针对EMT中最耗时的网络解算环节，把固定拓扑下重复出现的线性求解转化为可预处理的分层矩阵向量乘法；低秩块天然具有并行计算潜力，因此相对顺序性较强的LU回代更适合进一步加速。从验证范围看，结论主要支撑39、179母线及其扩展算例上的网络求解加速和精度保持；原文摘要未报告更多商业大系统、强非线性开关密集事件、拓扑频繁变化、实时硬件平台或不同时间步长下的可核验结果。若网络拓扑或元件状态频繁改变导致G频繁更新，预处理G^{-1}和分层SVD的代价可能影响总体收益，这一点需要另行评估。

### 4. 价值、认知与可复用场景

这项工作的重要认知是：EMT网络方程虽然来自稀疏导纳矩阵，但其逆矩阵中远距离母线组的影响可呈现可压缩结构，因而网络求解不必局限于稀疏LU路线。它适合被后续研究复用于大规模EMT网络求解器、分层矩阵压缩、并行矩阵向量乘法、在线安全评估加速和暂态仿真软件内核优化等页面。它能解决的是固定或少变网络拓扑下反复求解线性网络方程的计算瓶颈，而不是替代EMT元件模型、控制器模型或故障建模。也不宜把39/179母线及扩展算例上的最高2.8倍加速外推为所有电网、所有硬件和所有扰动场景的通用加速比。

## 方法细节

### 方法概述

本文提出一种基于分层低秩近似的网络方程快速求解器，旨在突破传统EMT仿真中网络求解的计算瓶颈。传统方法依赖稀疏LU分解与前代回代，计算复杂度高且难以并行。该方法首先将节点电压方程重构为矩阵向量乘法形式$v(t)=G^{-1}i(t)$。随后，利用电力网络导纳矩阵的拓扑特性，设计自底向上的图划分算法，将母线按电气距离聚类并构建平衡二叉树。针对树中表征远距离或弱耦合母线组交互的非对角子矩阵，采用截断奇异值分解(SVD)进行低秩近似，仅保留主导奇异值及对应左右奇异向量，使近似误差严格低于预设阈值。在仿真每个时间步，通过遍历该层次数据结构执行快速矩阵向量乘法，将单步求解复杂度从$O(N^2)$降至$O(N \log N)$，在维持数值精度的同时显著提升计算效率并具备高度并行化潜力。

### 数学公式

**公式1**: $$$Gv(t) = i_{in}(t) + i_{his}(t - \Delta t)$$$

*EMT网络节点方程，G为导纳矩阵，v为节点电压向量，i_in为注入电流，i_his为历史电流*

**公式2**: $$$v(t) = G^{-1} i(t)$$$

*将网络方程重构为矩阵向量乘法形式，i(t)为总注入电流向量，作为快速求解的核心目标*

**公式3**: $$$A \approx A_r = \sum_{i=1}^r \sigma_i u_i v_i^*$$$

*截断SVD低秩近似公式，用前r个奇异值及向量近似原交互子矩阵A*

**公式4**: $$$\varepsilon = \|A - A_r\|_F = \sqrt{\sigma_{r+1}^2 + \dots + \sigma_p^2}$$$

*基于Eckart-Young-Mirsky定理的截断误差计算公式，用于控制近似精度*

**公式5**: $$$W(S_1, S_2) = \frac{\sum_{v_1 \in S_1, v_2 \in S_2} G_{v_i, v_j}}{\max\{|S_1|, |S_2|\}}$$$

*图划分权重函数，衡量两组母线间的平均电导，用于指导平衡合并*

### 算法步骤

1. 步骤1（网络图构建与自底向上划分）：基于全系统导纳矩阵G构建无向图H=(V,E)。初始化所有母线为独立节点，按层级执行平衡合并：优先合并度为1的节点与其唯一邻居；随后在剩余节点中迭代选取权重函数W(S1,S2)最大的节点对进行合并，确保每层每个节点仅合并一次。重复该过程直至所有节点合并为单一根节点，生成高度为O(log N)的平衡二叉树层次结构。

2. 步骤2（逆矩阵计算与分层低秩近似）：预处理阶段计算全网络导纳逆矩阵G^{-1}。从二叉树根节点开始递归遍历：若当前子矩阵维度或数值秩小于预设阈值r_th，则直接存储为显式矩阵；否则计算其完整SVD，根据预设误差阈值ε_th截断保留前r个奇异值及对应U、Σ、V矩阵。若秩仍大于r_th，则将子矩阵按行列索引划分为四个子块，对每个子块递归执行相同近似流程，直至所有交互子矩阵均被显式存储或低秩分解表示。

3. 步骤3（快速矩阵向量乘法求解）：在每个仿真时间步，先计算历史电流i_his(t-Δt)与注入电流i_in(t)之和得到i(t)。自顶向下遍历层次树执行矩阵向量乘法：若子矩阵为显式形式，直接计算v=G^{-1}i；若为低秩形式(U,Σ,V)，先计算中间向量x=Σ(Vi)，再计算v=Ux；若为子块组合形式，则递归调用子节点计算并合并结果。最终输出全系统节点电压向量v(t)，完成单步网络求解。

### 关键参数

- **r_th**: 预设秩阈值，控制低秩近似的最大保留秩数

- **ε_th**: 预设截断误差阈值，基于Frobenius范数控制SVD截断精度

- **N**: 电力系统母线总数

- **Δt**: EMT仿真时间步长

## 量化发现

- 网络方程求解时间占传统EMT仿真总耗时的80%-97%，本方法将其单步计算复杂度从O(N^2)降至O(N log N)

- 低秩近似截断误差严格受控于预设阈值ε_th，仿真电压波形与商业软件EMTP-RV结果高度一致，未牺牲计算精度
- 层次划分树高度为O(log N)，保证了快速矩阵向量乘法的递归深度与计算效率

## 关键公式

### 重构网络求解方程

$$$v(t) = G^{-1} i(t)$$$

*替代传统前代回代过程，将网络求解转化为矩阵向量乘法，是低秩近似加速的基础*

### 时间复杂度递推公式

$$$T(N) \leq \begin{cases} CN^2 & \text{if } N \leq r_{th} \\ CNr_{th} & \text{if rank}(G^{-1}) \leq r_{th} \\ 2T(N/2) + 2C(N/2)r_{th} + N & \text{otherwise} \end{cases}$$$

*用于严格证明快速矩阵向量乘法算法的整体时间复杂度为O(N log N)*
