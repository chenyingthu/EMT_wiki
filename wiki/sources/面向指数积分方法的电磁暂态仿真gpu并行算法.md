# 面向指数积分方法的电磁暂态仿真GPU并行算法

**作者**: 赵金利; 刘君陶; 李鹏; 富晓鹏; 王成山 等
**年份**: 2018
**来源**: `19、20、21/EMT_task_21/Zhao 等 - 2018 - GPU Based Parallel Algorithm Oriented to Exponential Integration Method for Electromagnetic Transien.pdf`

## 摘要



为满足对大规模可再生能源接入的电力系统进行快速电磁暂态仿真的需求, 提出了一种面向指数积分方法的电力系统电磁暂态仿真图形处理器(GPU)并行算法. 首先, 分析了矩阵指数积分算法求解过程所具有的高度数据并行性, 进而将该特性与GPU计算资源相结合; 利用GPU处理指数积分方法求解时所需的大规模矩阵运算, 而将较为复杂的系统状态判别与更新保留在CPU中完成, 有效提升了仿真计算速度. 最后, 分别针对17台和100台风机的风电场算例进行了测试, 验证了所提并行算法的正确性和有效性, 同时也说明了算法的加速效果会随着系统规模的增加而愈发明显.

## 研究解读

### 1. 需求、对象、挑战与贡献

这篇论文面对的需求不是一般性的“加快仿真”，而是大规模新能源并网系统在电磁暂态尺度下的快速计算：风电场、变流器、集电网络等详细模型会带来大量状态变量和微秒级时间步长，仿真时长一旦延伸，传统串行EMT计算很容易成为瓶颈。研究对象是基于指数积分，尤其指数Euler公式的电力系统电磁暂态状态方程求解过程。难点在于：一方面，电力系统模型具有刚性和多时间尺度特征，需要数值稳定性；另一方面，指数积分会产生大量矩阵-向量乘法和向量运算，计算量随系统规模增长，而系统状态切换、故障和开关逻辑又不适合完全放到GPU上。本文的贡献在于识别指数积分求解过程中的高数据并行度，将大规模矩阵运算映射到GPU并行执行，同时保留复杂状态判别和模型更新在CPU端，形成面向指数积分方法的CPU-GPU异构EMT仿真流程，并用17台和100台DFIG风机风电场算例验证其正确性和规模扩展趋势。

### 2. 模型、算法与实现技术

本文采用状态空间形式描述EMT模型：$\dot{x}=Ax+g(t,x)$，其中$x$为系统状态变量，$A$表示可线性化或线性部分，$g(t,x)$容纳非线性项、控制和外部激励等。指数积分的作用是用矩阵指数精确处理线性部分，得到$e^{hA}x_n$和$h\varphi_1(hA)g(t_n,x_n)$构成的单步推进公式。进一步利用递推和等价变形，将指数Euler写成$x_{n+1}=x_n+h\varphi_1(hA)(Ax_n+g(t_n,x_n))$，这样每一步主要分解为稀疏矩阵$A$与状态向量相乘、非线性向量更新、稠密矩阵$h\varphi_1(hA)$与向量相乘、向量加法等操作。算法实现上，CPU负责初始化、系统状态判断、故障或开关事件检测、拓扑或模型变化后的矩阵重构；GPU负责规则且数据并行度高的矩阵-向量和向量运算。输入主要是系统模型、当前状态$x_n$、步长$h$、矩阵$A$及非线性项参数；输出是下一时步状态$x_{n+1}$和所需观测量。该分工的机制意义在于避免把分支复杂、通信频繁的逻辑强行并行化，而把计算密集且结构规整的部分交给GPU。

### 3. 验证、优势与不足

作者使用风电场算例验证方法，测试对象包括17台和100台基于双馈感应发电机的风力发电机组，并通过功率因数校正并联电容器与外部电网连接。验证包含两类基线：一是与MATLAB/SimPowerSystems中搭建的相同算例进行结果对比，用于判断并行指数积分EMT结果是否可信；二是与MATLAB脚本实现的指数积分串行程序对比，用于观察GPU并行实现的加速效果。原文摘要明确说明17台和100台风机算例验证了算法正确性和有效性，并指出加速效果随系统规模增加而更加明显；但当前证据未给出可核验的加速比、运行时间、误差范数或硬件型号等具体数值，因此不能将“更快”量化为固定倍数。优势主要体现在算法结构与GPU硬件匹配：指数Euler步骤中的矩阵-向量乘法和向量加法适合SIMT并行，而系统状态判别保留在CPU可降低复杂分支对GPU效率的影响。从验证范围看，结论主要支持大规模风电场EMT场景下的离线快速仿真，不足以证明对任意电网拓扑、任意电力电子控制、频繁开关系统、实时仿真或其他GPU平台均有同样效果。

### 4. 价值、认知与可复用场景

这项工作的核心认知价值在于把“数值积分方法选择”和“硬件并行结构”联系起来：指数积分不仅因A稳定性适合刚性EMT问题，还因为其计算核可分解为规则矩阵和向量操作，天然适合GPU。它可被后续研究用于设计指数Runge-Kutta、指数梯形等指数类EMT算法的并行框架，也适合为大规模风电场、新能源并网系统、含大量状态变量的离线EMT仿真提供实现参考。复用时应关注状态方程组织、矩阵存储、CPU-GPU通信和事件处理策略。不宜外推为所有EMT仿真均能获得同等加速，也不能据此替代对具体控制模型、故障场景和硬件平台的重新验证。

## 方法细节

### 方法概述

本文提出了一种面向指数积分方法的电力系统电磁暂态仿真GPU并行算法。该方法基于状态分析框架，利用指数Euler公式进行数值积分，通过CPU-GPU协同计算架构提升仿真效率。具体而言，将指数积分求解过程中具有高度数据并行性的大规模矩阵运算（稀疏矩阵向量乘法、稠密矩阵向量乘法、向量加法等） offload 到GPU上并行执行，利用CUDA架构及CUBLAS、CUSPARSE库加速计算；而将涉及复杂逻辑判断和串行处理的系统状态判别、模型更新、故障处理等环节保留在CPU端执行，从而减少CPU与GPU间的数据传输开销，实现高效的异构并行电磁暂态仿真。

### 数学公式


**公式1**: $$$$\dot{x}(t) = F(t,x(t)) = Ax(t) + g(t,x(t))$$$$

*系统状态方程，描述电力系统电磁暂态模型，其中x为状态变量，A为线性部分矩阵，g为非线性部分*


**公式2**: $$$$x(t) = e^{(t-t_0)A}x_0 + \int_{t_0}^t e^{(t-\tau)A}g(\tau,x(\tau))d\tau$$$$

*基于矩阵指数算子的Volterra积分方程，表示系统精确动态响应*


**公式3**: $$$$x_{n+1} = e^{h_n A} x_n + h_n \varphi_1(h_n A) g(t_n, x_n)$$$$

*指数Euler积分公式，用于数值求解，其中$h_n = t_{n+1} - t_n$为仿真步长*


**公式4**: $$$$\varphi_k(hA) = \int_0^1 e^{h(1-\theta)A} \frac{\theta^{k-1}}{(k-1)!} d\theta, \quad k \ge 1$$$$

*$\varphi$族函数定义，用于将矩阵指数形式的积分转化为可计算形式*


**公式5**: $$$$z\varphi_{k+1}(z) = \varphi_k(z) - \varphi_k(0)$$$$

*$\varphi$族函数的递推关系，用于高效计算高阶$\varphi$函数*


**公式6**: $$$$x_{n+1} = x_n + h_n\varphi_1(h_n A)(Ax_n + g(t_n, x_n))$$$$

*指数Euler公式的实用计算形式，分解为矩阵向量乘法和向量加法操作，适合GPU并行*


### 算法步骤

1. 步骤1：在状态分析框架下，对待仿真的电力系统建立电磁暂态仿真模型，构建状态方程

2. 步骤2：完成系统初始化，在主机端(CPU)分配内存空间，在设备端(GPU)分配显存空间，用于存放矩阵、状态变量、输出向量等数据

3. 步骤3：更新计算所需的系统矩阵（包括线性部分矩阵A和非线性部分参数）

4. 步骤4：将数据从主机端(CPU内存)传输到设备端(GPU显存)，准备GPU计算

5. 步骤5：在GPU中并行计算当前时刻稀疏矩阵A与状态向量$x_n$的乘法运算($dx \leftarrow Ax_n$)，结果存放在GPU存储空间，采用CSR(压缩稀疏行)格式存储稀疏矩阵以提升效率

6. 步骤6：在GPU中完成非线性部分向量$g(t_n,x_n)$的更新，以及向量$dx$和$g$之间的相加运算($dx \leftarrow dx + g(t_n, x_n)$)

7. 步骤7：在GPU中计算稠密矩阵向量乘法 $dx \leftarrow h\varphi(hA)dx$，其中$h\varphi(hA)$为稠密矩阵

8. 步骤8：在GPU中进行向量加法 $x_{n+1} \leftarrow x_n + dx$，得到下一时步的状态变量

9. 步骤9：更新仿真时刻 $t = t + \Delta t$

10. 步骤10：判断仿真时间是否到达终止时刻，若是则转到步骤13，否则转到下一步

11. 步骤11：在CPU中进行系统状态判断，检测是否有故障发生或开关操作

12. 步骤12：若系统状态发生改变，则在CPU端重新建立系统仿真模型并更新矩阵，转至步骤3；若状态未变，则直接转到步骤5继续下一时步计算

13. 步骤13：仿真结束，从GPU传回最终结果，释放资源


### 关键参数

- **仿真步长**: $h_n = 50\mu s$

- **基准对比步长**: $0.1\mu s$（用于MATLAB/SPS高精度基准）

- **计算精度**: 双精度浮点数

- **稀疏矩阵存储格式**: CSR (Compressed Sparse Row)

- **GPU架构**: CUDA (统一计算设备架构)

- **加速库**: CUBLAS (稠密矩阵运算), CUSPARSE (稀疏矩阵运算)

- **测试系统规模**: 17台风机和100台风机风电场

- **操作系统**: Windows 8.1

## 量化发现

- 指数Euler方法具有与隐式梯形法相同的A稳定性，适合处理电力系统的刚性问题
- 仿真步长设置为50μs，对比基准采用0.1μs小步长积分结果
- 矩阵向量乘法的串行算法时间复杂度为$O(n^2)$，通过GPU并行可显著降低实际计算时间
- 算法涉及的大规模矩阵运算（稀疏矩阵A与向量乘法、稠密矩阵$h\varphi(hA)$与向量乘法）具有$O(n^2)$的计算复杂度，但通过GPU并行处理可实现接近线性的加速比提升
- 加速效果随系统规模增加（从17台到100台风机）而愈发明显，证明算法适合大规模系统仿真


## 关键公式

### 指数Euler实用计算公式

$$$$x_{n+1} = x_n + h_n\varphi_1(h_n A)(Ax_n + g(t_n, x_n))$$$$

*在GPU上并行执行的核心计算步骤，将原微分方程求解转化为矩阵向量乘法和向量加法，分为4个并行化计算环节：稀疏矩阵向量乘法$Ax_n$、向量更新与加法、稠密矩阵向量乘法$h\varphi(hA)dx$、以及最终的状态向量更新*

### $\varphi_1$函数定义

$$$$\varphi_1(hA) = \int_0^1 e^{h(1-\theta)A} d\theta$$$$

*指数积分方法的关键核函数，用于精确处理系统线性部分的演化，保证算法的A稳定性和高精度*



## 验证详情

- **验证方式**: 对比验证分析，包括与商业仿真软件对比和与串行算法对比
- **测试系统**: 文献[34]中的风电场测试算例，包含17台和100台基于双馈感应发电机(DFIG)的风力发电机组，经功率因数校正并联电容器与外部电网连接，包含详细的元件子系统模型（发电机、变流器、集电网络等）
