## 方法细节

### 方法概述

补偿法（CM）用于在缺乏自然传输线传播延迟或延迟过短时实现电磁暂态（EMT）并行仿真。该方法基于诺顿/戴维南等效与叠加定理，通过手动切断连接元件（导线或支路）将网络解耦为多个独立子网。各子网利用节点导纳矩阵和稀疏LU分解并行求解初始节点电压与戴维南等效参数；随后通过多线程屏障同步机制收集各子网的等效参数，串行计算补偿支路电流；最后将补偿电流作为附加注入量回代至各子网，修正得到最终节点电压。针对含电力电子开关动作的网络，动态更新并重新分解补偿阻抗矩阵。该方法可与传统线路延迟解耦策略无缝结合，通过任务映射与屏障同步实现高效的离线与实时并行计算。

### 数学公式


**公式1**: $$$Y_1 v_{n1} = i_1, \quad Y_2 v_{n2} = i_2$$$

*子网节点导纳方程，用于并行求解各子网初始节点电压向量*


**公式2**: $$$Y_1 v_1^{(k)} = i_1^{(k)}, \quad Y_2 v_2^{(k)} = i_2^{(k)}$$$

*戴维南阻抗计算方程，通过在切断节点注入单位电流并置零独立源求解*


**公式3**: $$$Z_{th1} = [v_{1nC1}^{(1)} \dots v_{1nC1}^{(nC)}], \quad Z_{th2} = [v_{2nC2}^{(1)} \dots v_{2nC2}^{(nC)}]$$$

*构建切断节点处的戴维南等效阻抗矩阵*


**公式4**: $$$Z_C i_C = v_{th2} - v_{th1}$$$

*补偿支路电流求解方程，其中$Z_C = Z_{th1} + Z_{th2} + Z_B$为总补偿阻抗*


**公式5**: $$$v_{n1}^{final} = v_{n1} + v_1^C, \quad v_{n2}^{final} = v_{n2} + v_2^C$$$

*最终节点电压更新方程，将补偿电流产生的电压修正量叠加至初始解*


### 算法步骤

1. 网络解耦与任务划分：手动选择切断位置（导线或支路），将原始网络分割为多个独立子网（如N1、N2），并为每个子网分配独立计算线程。

2. 子网并行求解：在各仿真步长内，各线程并行求解节点导纳方程$Y v_n = i$，利用稀疏LU分解计算初始节点电压$v_{n1}, v_{n2}$，并直接提取切断节点处的戴维南电压$v_{th1}, v_{th2}$。

3. 戴维南阻抗矩阵构建：在各子网内，依次在切断节点注入单位电流向量，置零所有独立源，求解线性方程组得到各列电压向量，组装成$Z_{th1}$和$Z_{th2}$矩阵。若网络拓扑未变，该矩阵可跨步长复用。

4. 屏障同步与补偿电流计算：设置第一道线程屏障，收集所有子网的戴维南等效参数。随后在专用补偿线程中串行求解$Z_C i_C = v_{th2} - v_{th1}$，得到补偿支路电流$i_C$。

5. 电压修正与结果广播：利用$i_C$计算各子网切断节点的电压修正量$v_1^C, v_2^C$，叠加至初始电压得到最终解。设置第二道屏障广播补偿结果，各子网线程继续计算历史电流并进入下一仿真步。

6. 开关事件处理：当检测到电力电子器件或断路器动作导致拓扑变化时，实时更新补偿阻抗矩阵$Z_C$并触发稀疏LU重新分解，确保非线性暂态过程的数值稳定性。


### 关键参数

- **仿真步长**: 配电网50 μs，HVDC离线/实时30 μs，HIL验证40 μs

- **计算平台**: Arch1 (Intel i7-6820HQ), Arch2/OP5031 (双路Intel Xeon E5, 32核)

- **矩阵求解器**: 稀疏LU分解（Sparse LU decomposition）

- **同步机制**: 双屏障线程同步（收集等效参数/广播补偿结果）

- **网络规模**: 600节点20 kV配电网，IFA2000 HVDC双极系统（±272 kV, 1000 MW）



## 仿真结果

### 仿真测试

| 测试场景 | 结果描述 | 对比基线 |

|---------|---------|----------|

| 600节点20 kV配电网单相接地故障 | 在t=0.2 s施加1 mΩ单相接地故障，持续100 ms。采用1次切断（2核）与3次切断（4核）并行策略。 | 2核方案执行时间降至67 μs（Arch2），实时步长73 μs，加速比1.99倍；4核方案执行时间降至38 μs，实时步长40 μs，加速比3.63倍。相对误差仅表现为数值噪声，精度无损。 |

| IFA2000 HVDC换流站解耦仿真 | 针对无自然延迟的LCC换流器内部网络，采用CM将每个换流站解耦为2个子任务，共6核并行。仿真步长30 μs，包含6.3 s换流器解锁过程。 | 相比传统方法需4核且易发生实时越限（overrun），CM方案在6核下稳定运行，成功处理晶闸管频繁换相导致的矩阵重分解，直流电压波形与基准高度一致。 |

| HVDC硬件在环（HIL）交流侧故障 | 在法国侧施加40 ms单相接地故障，传输功率1000 MW，步长40 μs。使用物理控制保护装置替换Simulink模型。 | CM方案使用4核，传统方法需3核。计算速度提升40%，有效避免实时仿真超时，直流电压动态响应误差在可接受范围内。 |

| HVDC HIL变压器空载合闸 | 法国侧详细电网模型下执行三相非同步合闸（tA=128 ms, tB=129 ms, tC=119 ms），引发换相失败并最终导致HVDC闭锁。 | CM方案使用8核，传统方法需5核。在系统跳闸前计算性能提升30%，暂态电压/电流波形与基准吻合，微小偏移源于合闸事件非同步及重分解数值误差。 |



## 量化发现

- 在600节点配电网中，4核并行实现3.63倍加速比，实时仿真允许步长从145 μs缩短至40 μs。
- 线性网络下补偿法相对误差极低，仅表现为浮点运算数值噪声，未引入额外精度损失。
- HVDC HIL交流故障测试中，计算速度较传统解耦方法提升40%，成功消除实时越限现象。
- 变压器合闸暂态测试中，计算性能提升30%，支持8核并行处理复杂开关拓扑。
- 开关网络因采用固定主元策略进行多次矩阵重分解，相对误差略高于线性网络，但未引发数值失稳。


## 关键公式

### 补偿支路电流求解方程

$$$Z_C i_C = v_{th2} - v_{th1}$$$

*在收集各子网戴维南等效电压与阻抗后，用于串行计算切断支路的补偿电流，是连接并行子网的核心桥梁。*

### 节点电压修正方程

$$$v_{n}^{final} = v_{n} + v^C$$$

*将补偿电流产生的电压修正量叠加至子网初始解，获得满足原始网络拓扑约束的最终节点电压。*

### 子网节点导纳方程

$$$Y v_n = i$$$

*各独立子网并行求解的基础方程，结合稀疏LU分解实现高效数值积分。*



## 验证详情

- **验证方式**: 离线仿真（Offline）、软件在环（SIL）、硬件在环（HIL）对比验证
- **测试系统**: 600节点20 kV主动配电网；IFA2000高压直流输电系统（法国-英国互联，±272 kV LCC双极，1000 MW）
- **仿真工具**: 自研EMT实时仿真平台（集成稀疏LU求解器）、OP5031实时仿真目标机、MATLAB Simulink（控制模型接口）、物理控制保护装置（HIL）
- **验证结果**: 补偿法在无自然线路延迟场景下成功实现高效并行解耦。线性配电网测试验证了其高精度与3.63倍加速能力；HVDC开关网络与HIL实测表明，该方法能有效处理晶闸管换相、矩阵重分解及复杂控制交互，计算速度提升30%~40%，实时步长显著缩短，且未出现数值失稳，满足工业级实时仿真需求。
