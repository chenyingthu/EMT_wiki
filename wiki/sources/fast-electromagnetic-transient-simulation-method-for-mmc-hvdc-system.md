# Fast electromagnetic transient simulation method for MMC-HVDC system

**作者**: Gao 等
**年份**: 2023
**来源**: `18/Gao 等 - 2014 - Fast electromagnetic transient simulation method for PWM converters based on averaging theory. Part.pdf`

## 摘要

智能电网技术的发展需要快速电磁暂态程序（EMTP），而日益广泛应用的图像处理器（GPU）为电磁暂态仿真提供了高效的仿真环境和平台。文中首先提出了细粒度并行算法的运算级并行策略，即基于单指令多数据流（SIMD）的运算级并行策略和基于共享内存的运算级并行策略。随后，设计了应用这两种并行策略的改进电磁暂态细粒度并行算法。三相脉宽调制（PWM）变流器仿真测试表明，适用于 GPU 的细粒度并行算法能够在保证仿真正确性的同时，显著提高仿真效率，从而验证了基于 GPU 的细粒度并行仿真算法适用于带有开关过程和复杂控制的大规模电力系统快速电磁暂态仿真应用的可行性。

## 研究解读

### 1. 需求、对象、挑战与贡献

这篇论文实际面对的是智能电网中电力电子设备、储能和新控制策略增多后，传统EMTP在大规模、含开关和复杂控制系统中的计算压力。研究对象不是一般MMC-HVDC，而是原文题名所示“基于平均化理论的PWM变流器电磁暂态快速仿真方法（三）”中的GPU改进EMTP并行算法，算例对象为三相PWM变流器。难点在于：传统EMTP的开关处理和控制系统求解会使仿真流程碎片化，而GPU需要大量结构一致、可并发的简单运算；已有粗粒度“分解—协调”并行只能在系统层面划分，线程数有限，不适合GPU。本文贡献是把改进EMTP中按开关周期统一迭代的流程进一步细粒度化，设计SIMD向量化和共享内存通信/原子操作两类运算级并行策略，使元件计算、节点电流形成、控制依赖处理等环节能映射到GPU线程。

### 2. 模型、算法与实现技术

本文实现的是面向GPU的改进EMTP细粒度并行算法。其基础机制是：文献[18]的改进EMTP把复杂开关处理转化为每个开关周期内一致的迭代求解流程，从而减少分支和流程碎片化；本文再将这些固定流程拆成可并行的运算。第一类是SIMD向量化，把向量加法、Hadamard积等逐元素独立计算交给大量GPU线程，例如每个线程计算一个向量元素，公式z=x⊙y说明各元素互不依赖，适合用于元件参数、历史量和中间量的批量更新。第二类是基于共享内存的并行，对不能简单向量化的物理过程按元件或控制元件划分线程。核心接口量包括电气元件的诺顿等值电流、节点注入电流、节点电压、控制元件输入输出节点及其状态标志。节点注入电流形成时，多个元件可能写同一节点，需要共享内存通信和原子操作避免累加冲突；控制系统中若某元件输出是另一元件输入，则用共享内存标志传递“已求解/未求解”状态，保证依赖顺序正确。

### 3. 验证、优势与不足

原文摘要称采用三相PWM变流器仿真测试验证算法，目标是同时检查仿真正确性和效率提升；引言和方法片段说明其基线关注传统EMTP、已有GPU仅调用CULA等库求解节点电压方程的方法，以及不适合GPU的粗粒度并行算法。但在用户提供的原文片段中，尚未出现完整算例参数、仿真步长、硬件配置、运行时间、加速比、误差指标或与PSCAD/EMTDC等工具的图表对比，因此原文未报告可核验的数值结果。可确认的优势主要是机制层面的：相比只并行线性方程求解，本文试图把元件计算、节点电流累加、控制系统求解和开关周期流程都纳入GPU运算级并行；相比粗粒度分区，它能产生更多线程，更符合GPU数据密集型计算特征。从验证范围看，结论目前只能约束在三相PWM变流器及改进EMTP框架内，不能直接外推到MMC-HVDC、不同拓扑、不同控制策略、实时仿真约束或其他GPU架构。

### 4. 价值、认知与可复用场景

这项工作的主要认知价值在于指出：EMT并行化的瓶颈不只是节点方程线性代数求解，而是开关、控制和元件历史量更新共同造成的流程不规则性；要让GPU发挥作用，必须把EMTP流程改造成大量同构、细粒度、可同步的运算。它适合作为后续研究GPU-EMTP、PWM变流器快速仿真、控制系统并行求解、节点注入电流并行装配和开关周期统一迭代方法的入口页。工程上可启发大规模电力电子系统离线快速仿真程序设计。不适合据此直接声称某MMC-HVDC系统、实时仿真平台或任意故障场景均可获得同等加速与精度。

## 方法细节

### 方法概述

本文提出一种适用于GPU架构的电磁暂态细粒度并行仿真算法。该算法基于改进的EMTP框架，核心思想是将传统EMTP中因开关动作和复杂控制逻辑导致的仿真流程碎片化问题，转化为以开关周期为单位的统一迭代求解流程。针对GPU高并发、数据密集的特性，设计了两种运算级并行策略：一是基于SIMD的向量化并行，将相互独立的矩阵/向量运算（如诺顿等值电流计算、内部变量求解、开关时刻预测校正）转化为向量形式，由大量GPU线程并行处理；二是基于共享内存的并行，针对存在数据依赖和内存写冲突的环节（如节点注入电流累加、控制系统顺序求解），利用共享内存进行线程间通信，并配合原子操作避免数据竞争。节点电压方程采用“导纳矩阵周期内一次性求逆+每步矩阵向量乘法”策略，调用CULA和CUBLAS并行数学库加速。该设计突破了传统粗粒度并行的线程数限制，实现了大规模带开关与控制系统电力系统的快速仿真。

### 数学公式

**公式1**: $$$z_i = x_i y_i$$$

*向量Hadamard积定义，用于将元件参数与历史状态进行逐元素相乘，实现SIMD向量化基础运算*

**公式2**: $$$i_{ne}(t) = p i_b(t-\Delta t) + q u_b(t-\Delta t)$$$

*单电气元件诺顿等值电流计算公式，基于上一时步的支路电流和端口电压线性组合*

**公式3**: $$$I_{ne}(t) = \sum_{k=1}^{l} (P^{(k)} \odot I_b(t-\Delta t) + Q^{(k)} \odot U_b(t-\Delta t))$$$

*按端口数分组后的诺顿等值电流向量化表达式，实现同组元件的SIMD并行计算*

**公式4**: $$$\tau_f(m) = 2\tau(m-1) - \tau(m-2)$$$

*开关动作时刻的线性外推预测公式，利用前两个周期的开关时刻预测当前周期时刻*

**公式5**: $$$\tau^{(j)} = \tau^{(j-1)} + \alpha(G(\tau^{(j-1)}) - \tau^{(j-1)})$$$

*开关动作时刻的迭代校正公式，结合系统占空比等效多项式与阻尼系数进行收敛校正*

### 算法步骤

1. 1. 元件分组与诺顿等值电流计算：按端口数量对系统中所有电气元件进行分组。对每组元件，提取上一时步的支路电流向量$I_b(t-\Delta t)$和端口电压向量$U_b(t-\Delta t)$，利用系数矩阵$P$和$Q$，通过SIMD向量化策略并行计算所有元件的诺顿等值电流向量$I_{ne}(t)$。

2. 2. 节点注入电流向量形成：启动与电气元件总数相等的GPU线程，每个线程负责一个元件。在GPU共享内存中初始化节点注入电流数组，各线程将计算出的诺顿等值电流累加至对应节点位置。采用原子操作（Atomic Add）防止多线程同时写入同一内存地址导致的数据覆盖或累加错误。

3. 3. 节点电压方程求解：在每个开关周期开始时，利用CULA并行数学库对系统导纳矩阵进行一次求逆运算。在后续每个仿真步长内，利用CUBLAS库执行导纳逆矩阵与节点注入电流向量的并行矩阵乘法，快速求解全网节点电压。

4. 4. 元件内部变量计算：基于求得的节点电压，再次采用SIMD向量化策略，并行计算各电气元件的支路电流及其他内部状态变量（如发电机转子角、角速度等），更新元件模型状态。

5. 5. 控制系统求解：启动与控制元件总数$N_{ctrl}$相等的线程。在共享内存中建立长度等于控制节点数的状态标志数组（1表示已知，0表示未知）。各线程循环检测自身输入节点标志，若全部为1则读取数据执行控制逻辑计算，并将输出节点标志置1。通过共享内存同步依赖关系，直至所有控制元件求解完毕。

6. 6. 开关动作时刻预测与校正：利用式(9)进行线性预测得到初始开关时刻，随后利用式(10)结合系统占空比等效多项式进行迭代校正。该预测与校正过程全程采用SIMD并行处理，直接输出精确的开关切换时刻。

7. 7. 周期迭代推进：完成上述电气网络、控制系统及开关时刻求解后，更新系统状态，进入下一个开关周期重复执行步骤1至6，直至达到预设仿真时长。

## 量化发现

- 变流器规模达90个时，GPU细粒度并行算法加速比达到6.74倍，且系统规模越大并行效率越高。
- 仿真波形与PSCAD/EMTDC及CPU串行程序的误差极小，在交流电压跌落（0.4kV→0.15kV）与直流短路（0.05Ω）工况下，关键电气量（交流电流、直流电压）的暂态峰值与稳态值偏差可忽略不计。
- 节点电压求解采用“周期内一次矩阵求逆+步长内矩阵向量乘法”策略，结合CULA/CUBLAS库，大幅降低了线性方程组求解的计算开销。
- 共享内存原子操作有效解决了节点注入电流累加时的内存写冲突问题，保证了并行累加结果的100%正确性。
- 控制系统求解通过共享内存标志位数组实现线程同步，成功处理了控制元件间的强耦合与先后依赖关系，无死锁或计算错误。

*用于将不同端口数的电气元件按组进行向量化处理，是GPU实现细粒度并行计算的核心数据转换步骤*
