# Time-Window Based Discrete-Time Fourier Series for Electromagnetic Transients in Power Systems

Arash Tavighi, Member IEEE, Jose R. Mart ´ ´ı, Fellow IEEE, Veronica A. Galv ´ an, ´ Member IEEE, and Jose A. ´ Gutierrez-Robles Member IEEE

Abstract—This paper approaches the solution of electromagnetic transients in the frequency domain from the point of view of choosing a time window with sufficient width to allow the time constants of the transients to decay to a small value. This criterion, and the required maximum frequency content of the simulation, fully determine the simulation parameters. Satisfying this criterion, a very simple methodology, like the Discrete Time Fourier Series can provide very accurate results without the inherent difficulties of time-warping solutions based on the Numerical Laplace Transform (NLT). Even though the NLT can give very accurate results, it requires expert knowledge to correctly specify the damping factors, sampling schemes, and windowing functions. The Discrete Time Fourier Series (DTFS) solution is compared with the NLT and with analytical solution for a circuit with lumped elements, and with the most-accepted frequency-dependent transmission line models in the Electromagnetic Transients Program (EMTP): the JMARTI (fdLINE) model and the Universal Line Model (ULM) for a double-circuit transmission line.

Index Terms—Discrete Time Fourier Series, Time-window width, Numerical Laplace Transform, Time constants, electromagnetic transient simulation.

## I. INTRODUCTION

F REQUENCY-DOMAIN methods have been used to solveelectromagnetic transients in power systems in the fre- electromagnetic transients in power systems in the frequency domain and then transferring the solution to the time domain using numerical Fourier or Laplace transforms. They are particularly useful when the system parameters are frequency dependent, and to verify the solutions for time-domain frequency-dependent transmission line and other component models [1]–[6].

Pioneering work on power system transients’ simulation using numerical Fourier transforms was performed in 1965 [7]. This work addressed the Gibbs oscillations due to truncation of the finite time sampling rate. The authors of [7] introduced the use of the Lanczos’ window to reduce Gibbs errors. They also investigated the aliasing errors in the time domain due to the discretization of the continuous variable in frequency. To reduce time aliasing, they proposed the “Modified Fourier Transform” (MFT) in which they smoothened the frequency response by adding a damping factor [8]. In 1978, Wilcox formulated the MFT methods in terms of the Laplace transform theory and introduced the term Numerical Laplace Transform (NLT) [9]. He also provided a criterion to calculate the associated damping factor. In 1982, Wedepohl proposed another criterion for the calculation of the damping factor based on the number of samples [10].

Unlike the most well-known frequency-dependent transmission line and cable models in the Electromagnetic Transients Program (EMTP) [11], such as fdLINE [12], fqLINE [13], and the Universal Line Model (ULM) [14], which use curve-fitting algorithms to approximate the frequency-dependent line functions, frequency-domain solutions make no approximations regarding the frequency-dependence of the line parameters or of the transformation matrices for modal decomposition. On the other hand, going back and forth between time and frequency domains imposes higher computational costs for frequencydomain methods than their time-domain counterparts. There have been several attempts to reduce the computational costs of frequency-domain methods [15]–[20].

The NLT method has been the most-widely used frequencydomain solution for power system transients’ studies. However, the use of a boosting factor in the NLT to compensate for the damping factor initially used to time-limit the duration of the transient signal often results in amplifying the Gibbs oscillations [21]. In this paper, we present examples where the use of the most well-known windowing functions to reduce these errors is not totally effective.

The application of Fourier solutions for transients’ analysis was discussed in [22] and [23]. These references highlighted the importance of the selection of the time window size to allow any system oscillation to damp out within this window. In the present paper, we start from the required size of the time window and present guidelines to choose its width. Choosing correctly the time window width and specifying the required maximum frequency in the transient [24] defines a very accurate Discrete Time Fourier Series (DTFS) solution.

The proposed DTFS solution is simpler to implement than the classical NLT solution, as it does not require the user to specify the damping factors and windowing functions required by the NLT to avoid Gibbs and aliasing errors. The DTFS can work equally well for both regular sampling and odd sampling. The use of regular sampling can double the computational efficiency of the proposed DTFS compared to the NLT, which normally requires the odd sampling set up.

## II. POWER SYSTEM TRANSIENT SOLUTION USING THE DISCRETE TIME FOURIER SERIES (DTFS)

Power system frequency-domain solutions can be formulated using nodal analysis to solve one frequency at the time,

$$
\mathbf { V } ( \boldsymbol { \omega } ) = \mathbf { Y } ( \boldsymbol { \omega } ) ^ { - 1 } \mathbf { I } ( \boldsymbol { \omega } )\tag{1}
$$

where V is the vector of calculated node voltages, Y is the matrix of admittances evaluated at each frequency, and I is the injected current sources.

For an N-phase frequency-dependent transmission line, the admittance matrix can be calculated using a coupled exactπ circuit [24]. To form the exact-π circuit, the per-unit-length line series impedance matrix ${ \bf { Z } } _ { p h }$ and shunt admittance matrix $\mathbf { Y } _ { p h }$ are initially calculated from

$$
{ \bf Z } _ { p h } ( \omega ) = { \bf R } _ { c } ( \omega ) + j \omega { \bf L } ( \omega ) , { \bf Y } _ { p h } ( \omega ) = { \bf G } _ { i n s } + j \omega { \bf C }\tag{2}
$$

In (2), $\mathbf { R } _ { c }$ is a diagonal matrix of conductor’s resistance, calculated using Bessel functions to account for skin effect [24] and L is the inductance matrix calculated using the method of complex penetration depth [25]. The shunt conductance ${ \bf G } _ { i n s }$ is a constant diagonal matrix corresponds to the losses of the insulators, and $\mathbf { C }$ is the capacitance matrix calculated from the Maxwell Potential Coefficients assuming electrostatic position for the charges [24]. In this paper, the diagonal elements of ${ \bf G } _ { i n s }$ are taken as $2 \times 1 0 ^ { - 9 }$ (S/km). For a transmission line with a given geometry and ground return parameters, ${ \bf { Z } } _ { p h }$ and $\mathbf { Y } _ { p h }$ matrices can be calculated from the Line Constants routine in EMTP packages such as MicroTran and PSCAD.

To form the π-circuits, the per-unit-length impedance and admittance matrices are then transferred from coupled phase quantities to decoupled modal quantities using modal transformation matrices [26] [27],

$$
\begin{array} { r } { \mathbf { Z } _ { m } ( \omega ) = \mathbf { T } _ { V } ( \omega ) ^ { - 1 } \mathbf { Z } _ { p h } ( \omega ) \mathbf { T } _ { I } ( \omega ) } \\ { \mathbf { Y } _ { m } ( \omega ) = \mathbf { T } _ { I } ( \omega ) ^ { - 1 } \mathbf { Y } _ { p h } ( \omega ) \mathbf { T } _ { V } ( \omega ) } \end{array}\tag{3}
$$

In (3), $\mathbf { T } _ { V }$ diagonalizes $\mathbf { Z } _ { p h } \mathbf { Y } _ { p h }$ and $\mathbf { T } _ { I }$ diagonalizes $\mathbf { Y } _ { p h } \mathbf { Z } _ { p h }$ These transformation matrices are frequency dependent with complex numbers, and are prone to the modes switching positions as the frequency changes [28]. A common approach to avoid these errors is the use of the Newton-Raphson method as proposed in [29].

To compensate for the distributed nature of the line parameters, the modal impedances and admittances in (3) are corrected for the line length \` and the frequency by applying the factors $k _ { z }$ and $k _ { y }$ of (5):

$$
Z _ { m } ^ { \prime } ( \omega ) = k _ { z } ( \omega ) Z _ { m } ( \omega ) \ell \quad , \quad Y _ { m } ^ { \prime } ( \omega ) = k _ { y } ( \omega ) Y _ { m } ( \omega ) \ell\tag{4}
$$

$$
k _ { z } ( \omega ) = \frac { \sinh \left( \gamma _ { m } ( \omega ) \ell \right) } { \gamma _ { m } ( \omega ) \ell } \quad , \quad k _ { y } ( \omega ) = \frac { \operatorname { t a n h } \left( \frac { \gamma _ { m } ( \omega ) \ell } { 2 } \right) } { \frac { \gamma _ { m } ( \omega ) \ell } { 2 } }\tag{5}
$$

where $\gamma _ { m }$ is the modal constant of propagation,

$$
\gamma _ { m } ( \omega ) = \sqrt { Z _ { m } ( \omega ) Y _ { m } ( \omega ) }\tag{6}
$$

The exact-π circuit in the phase domain can now be formed by transferring $\mathbf { Z } _ { m } ^ { \prime }$ and $\mathbf { Y } _ { m } ^ { \prime }$ from mode to phase,

$$
\begin{array} { r } { \mathbf { Z } _ { p h } ^ { \prime } ( \omega ) = \mathbf { T } _ { V } ( \omega ) \mathbf { Z } _ { m } ^ { \prime } ( \omega ) \mathbf { T } _ { I } ( \omega ) ^ { - 1 } } \\ { \mathbf { Y } _ { p h } ^ { \prime } ( \omega ) = \mathbf { T } _ { I } ( \omega ) \mathbf { Y } _ { m } ^ { \prime } ( \omega ) \mathbf { T } _ { V } ( \omega ) ^ { - 1 } } \end{array}\tag{7}
$$

To solve (1) in the frequency domain, we need to convert the injected current sources from time to frequency and, after solving (1), we need to convert the node voltages from frequency to time to form the transient response. For these conversions, we will use the conventional Discrete Time Fourier Series (DTFS) which establishes a one-to-one exact mapping between $N _ { s }$ points in the discrete time domain and the same number of $N _ { s }$ points in the discrete frequency domain.

Figure 1 illustrates the mapping from a time window of width $T _ { c }$ into its corresponding frequency window of width $f _ { c }$ using the DTFS. As will be discussed in Section III, the width of the time window should be chosen based on the time constants of the circuit. The width of the frequency window should be chosen based on the maximum frequency that we want to simulate in the transient. Once $T _ { c }$ and $f _ { c }$ have been chosen, the number of sample points (solution points) $N _ { s }$ is determined by

$$
N _ { s } = T _ { c } f _ { c }\tag{8}
$$

For non-integer values, $N _ { s }$ in (8) is rounded up. The time step size and the frequency step size are given by

$$
\Delta t = \frac { T _ { c } } { N _ { s } } \quad , \quad \Delta f = \frac { f _ { c } } { N _ { s } }\tag{9}
$$

The expressions in (9) provide an exact correspondence between the time-domain and the frequency-domain solution samples.

It should be noted that because the Fourier series to represent real time signal is complex conjugate (Fig. 1), the maximum frequency that can be captured in the solution is $f _ { N y }$ (Nyquist frequency) which is one half of the sampling frequency,

$$
f _ { N y } = \frac { 1 } { 2 \Delta t } = \frac { f _ { c } } { 2 }\tag{10}
$$

This means that for regular sampling $( \omega = 2 \pi k \Delta f , k =$ $0 , 1 , 2 , \ldots , N _ { s } - 1 )$ there will be only $N _ { s } / 2$ distinct frequencies in the interval $[ 0 , + f _ { N y } ]$ ]. The rest of the frequency components $[ - f _ { N y } , 0 ]$ are the complex conjugate of the frequencies in the positive interval. This property can be used to increase the computational efficiency of the frequency-domain solution by solving the system with frequencies in increments $f = k \Delta f$ for $k = 0 , 1 , 2 , \ldots , N _ { s } / 2$ and constructing (instead of solving for) the complementary part of the frequency spectrum (from $- f _ { N y }$ to 0), by taking the complex conjugate of the positive half solution. This property does not exist when odd sampling is used $( \omega = \pi \left( 2 k + 1 \right) \Delta f , k = 0 , 1 , 2 , \ldots , N _ { s } - 1 )$

Notice that even though the frequency window of Fig. 1a shows 11 points, there are only 10 points for five distinct frequencies since the $- f _ { N y }$ and $+ f _ { N y }$ points are 1/2 of the same point.

The mapping between time and frequency in the DTFS is established as follows. In the general case, with either regular or odd sampling, a discrete-time signal $x [ n ]$ , having an arbitrary shape and a finite number of equally spaced samples $N _ { s } ,$ can be expressed as a sum of complex frequency components such that when evaluated at $t = n \Delta t$ we get the original x[n] values,

$$
x [ n ] = \sum _ { k = \left\lfloor - { \frac { N _ { s } - 1 } { 2 } } \right\rfloor } ^ { \left\lceil { \frac { N _ { s } - 1 } { 2 } } \right\rceil } A _ { k } e ^ { j k { \frac { 2 \pi } { N _ { s } } } n } = I D T F S \{ A _ { k } \}\tag{11}
$$

where de and bc are the ceiling and floor functions, respectively. Equation (11) is the Inverse Discrete Time Fourier Series (IDTFS) of the coefficients of the Fourier series. These coefficients are the spectral coefficients and are obtained from

$$
A _ { k } = \frac { 1 } { N _ { s } } \sum _ { n = 0 } ^ { N _ { s } - 1 } x [ n ] e ^ { - j k \frac { 2 \pi } { N _ { s } } n } = D T F S \{ x [ n ] \}\tag{12}
$$

Since the complex exponentials in the Fourier series are periodical functions, the synthesis is only valid for the original time span $T _ { c } .$

Since the DTFS provides a one-to-one mapping between corresponding points in the time and frequency windows, these points constitute a complete solution set and in this solution there is no need for filtering to avoid aliasing.

In Section III, we will suggest a procedure to specify a time window of adequate width and a frequency window of adequate width to contain the desired system dynamics and the desired frequency components that are of interest in a particular transient simulation.

## III. GUIDELINES TO SPECIFY A TIME WINDOW AND A FREQUENCY WINDOW FOR THE DTFS SOLUTION

Guidelines are provided next to choose the width of the time window and the width of the frequency window for a given transient simulation.

## A. Time-window Width $( T _ { c } )$

Figure 2a shows a discrete-time source with $N _ { 1 }$ samples applied to a system at $t = 0$ . Figure 2b illustrates an example system response to the first source sample at $t = 0$ and the last source sample at $t = \left( N _ { 1 } - 1 \right) \Delta t$ . For the source samples, at $t = n \Delta t$ , for $n = 0 , 1 , 2 , \ldots , N _ { 1 } - 1$ , the response samples will be replicas of $h [ 0 ]$ , scaled by the corresponding value of i[n]. The response at any source sample will have a length $N _ { 2 }$

The total output by the time we reach source sample $N _ { 1 } - 1$ will be the sum of all responses to the previous inputs plus the response to the current input,

$$
v [ n ] = i [ n ] * h [ n ] = \sum _ { n = 0 } ^ { N _ { 1 } - 1 } i [ n ] h [ N _ { 1 } - 1 - n ]\tag{13}
$$

![](images/c4792d0b62b246bf5da967689c4eb22c3c962dc38e73f1796e434435834a08bb.jpg)  
Fig. 2. Time window selection a) Input source (Dashed line: unit step with discontinuity. Solid line: unit step with averaging of points at discontinuity). b) System response.

Equation (13) is the convolution of the input function i and the system response h at time step n. Suppose now that instead of obtaining the system response h in the time domain, our system is defined by a matrix of admittances $Y$ that has different values at different frequencies $f = k \Delta f .$ . In this case, if we express the input source i in terms of its frequency components $I [ k ] = D T F S \{ i [ n ] \}$ in (12), we can calculate $V [ k ]$ by multiplying $Y ^ { - 1 } [ k ]$ by $I [ k ]$ , and using the property that convolution in time (13) is equivalent to multiplication in frequency. Once V has been calculated for each $k ,$ we can map the $V [ k ]$ sequence to its corresponding v[n] sequence in (11).

Since the system response $h [ n ]$ (Fig. 2b) lasts for $N _ { 2 }$ samples, the time window $T _ { c }$ must be able to accommodate all the points of the source $( N _ { 1 } )$ plus the length of the response $h [ n ] ~ ( N _ { 2 } )$ to the last sample applied at $N _ { 1 } - 1$ . This gives us a length of $N _ { 1 } + N _ { 2 } - 1$ , which must be contained in the time window $T _ { c }$ . Calling the length of the source $t _ { s i m }$ and the length of the system impulse response $t _ { s e t }$ , we will have that

$$
T _ { c } = t _ { s i m } + t _ { s e t }\tag{14}
$$

Since the number of samples in the time window and in the frequency window must be equal, and since the source sequence is only of length $N _ { 1 }$ , we need to complete this sequence with zeroes (“zero completion”) for the rest of the $T _ { c }$ window. Since we stop the source at $t _ { s i m }$ , the simulation will only be valid up to $t _ { s i m }$ . The rest of the time window corresponds to “auxiliary points” for the time/frequency mapping that should be discarded in the final time-domain simulation.

As discussed, $t _ { s i m }$ is the simulation time of interest specified by the user, while $t _ { s e t }$ is the time interval needed by the system response to damp out. If we take, for example, $t _ { s e t }$ equal to seven times the time constant of the slowest mode of the system

$$
t _ { s e t } = 7 \tau _ { m }\tag{15}
$$

all transient components of the time response will decay to less than 0.1% of their initial value $( e ^ { - 7 } \overset { - } { \approx } 1 0 ^ { - 3 } )$ .

The time constants of a circuit with lumped-elements can be determined by extracting the poles of the corresponding transfer function. In general this can be achieved using many available routines such as linmod syntax in MATLAB/Simulink and eigenvalue/eigenvector analysis in [30].

Assume that the poles for a stable system of order n are given by,

$$
p _ { i } = \alpha _ { i } \pm j \beta _ { i } \quad , \quad i = 1 , 2 , \ldots , n\tag{16}
$$

The largest time constant of the system $\tau _ { m }$ can be calculated as

$$
\tau _ { m } = \frac { 1 } { \operatorname* { m i n } ( - \alpha _ { i } ) }\tag{17}
$$

The time constants for power systems that contain power electronic converters can be calculated using averaging methods to represent the converters with lumped-element circuits [31]–[33].

For a system that includes distributed-parameter frequencydependent transmission lines with coupled-conductors, we can assume that the line is transposed and use symmetrical components to calculate the approximate largest time constants $\tau _ { m }$ . The validity of this assumption will be analyzed for the double-circuit line of Fig. 3b in Section IV-B. For an $N _ { - }$ conductor transposed line, the zero/positive/negative sequence series impedance and shunt admittance are given as

$$
\begin{array} { c c c } { { Z ^ { o } = Z _ { s } + \left( N - 1 \right) Z _ { m } } } & { { , } } & { { Z ^ { + } = Z ^ { - } = Z _ { s } - Z _ { m } } } \\ { { { } } } & { { { } } } & { { { Y ^ { o } = Y _ { s } + \left( N - 1 \right) Y _ { m } } } } \end{array}\tag{18}
$$

In (18), subscript s refers to the average value of the diagonal elements whereas subscript m denotes the average value of the off-diagonal elements of the $\mathbf { Z } _ { p h }$ and $\mathbf { Y } _ { p h }$ matrices.

Since the positive sequence and negative sequence networks have the same time constants, for a given network terminal condition, we will have a combination of positive and zero sequence parameters (for example the three sequence impedances in series for a line to ground fault). Next, we use (4)–(6) to correct the zero/positive sequence parameters of (18) for the line length \` and the frequency and form π- circuits with lumped elements.

Since for transmission lines the parameters are frequency dependent and the resistances increase with frequency, while the inductances decrease and the capacitances can be assumed constant [34], the slowest time constants correspond to the lower frequencies. Normally the transmission line zero/positive sequence parameters are readily available for 60 Hz from power flow, or EMTP, studies and these parameters are sufficient for the margins required in determining the largest time constants (as we discussed in the guidelines to choose the time window width, we take a tolerance of seven times on these values).

Next, we connect the zero/positive sequence π-circuits to the rest of the system components and determine the time constants as in (16) (17). Normally the balanced case of threephase fault will give the largest $\tau _ { m } .$ In Section IV-B, we verify this assumption for our test case.

## B. Frequency-window Width $( f _ { c } )$

The width of the frequency window $f _ { c } = 2 f _ { N y }$ is determined by the maximum frequency $f _ { m }$ that we want to model in the transient. $f _ { m }$ should include the frequency of the source and the maximum frequency of the transient we are interested in. For accurate results $f _ { N y }$ should be a number of times larger than $f _ { m } .$

For a system described by its poles as in (16), we suggest that $f _ { N y }$ be sufficient to provide enough resolution (sufficient data points) to represent: a) the exponentials $( e ^ { - \alpha _ { i } t } )$ associated with the real part of the poles $\alpha _ { i }$

$$
B _ { r } = \operatorname* { m a x } ( - \alpha _ { i } ) W R\tag{19}
$$

where W R is the Waveform Resolution factor, and b) the resolution needed to capture the highest harmonic $( e ^ { - j \beta _ { i } t } )$ in the transient associated with the imaginary part of the poles $\beta _ { i } ,$

$$
B _ { i } = \frac { \operatorname* { m a x } \left( \lvert \beta _ { i } \rvert \right) } { 2 \pi } W R\tag{20}
$$

Where, as in (19), the resolution is augmented by the W R waveform resolution factor. A factor of $W R = 1 0$ is recommended for (19) and (20).

Notice that the waveform resolution criterion of (20) matches the criterion used in the EMTP [24] that advises to choose a Nyquist frequency of at least 5 times the maximum frequency in the transient. In the EMTP the recommendation is related to decreasing the distortion by the discretization rule, while here it applies to the “tracing” of the time-domain response.

For some systems, the real part of some of the poles $( \alpha _ { i } )$ might be very large (orders of magnitude larger than for the other poles). A very large α means that the associated transient decays very fast compared to the other exponentials, and the contribution of these poles to the transient in our observation window can be neglected. It is recommended that values of α larger than $1 0 ^ { 4 }$ times larger than the other alphas do not be taken into account.

For a system that includes frequency-dependent transmission lines, we recommend to calculate $B _ { r }$ and $B _ { i }$ using (19) and (20) for the positive and zero sequence systems and take the larger of the two, with the transmission line represented by decoupled π-circuits at low frequencies (e.g., 60 Hz). In addition, the bandwidth $B _ { \ell }$ should be enough to incorporate the wave travelling time for the line length \`. Using the speed of light $v _ { c } = 3 \times 1 0 ^ { 8 }$ m/s, $B _ { \ell }$ is calculated as in [24]

$$
B _ { \ell } = { \frac { v _ { c } } { \ell } } W R\tag{21}
$$

Common sources in power systems are sinusoidal, decaying exponentials, and step functions. For a sinusoidal source with frequency $f _ { 0 } ,$ the resolution needed to trace the waveform of the source is calculated as

$$
B _ { s } = f _ { 0 } W R\tag{22}
$$

For sources represented by exponential functions, the recommended resolution is the same as in (19).

The resolution for a step function of amplitude $A _ { m }$ and duration $t _ { s i m }$ can be calculated using the energy content from Parseval’s theorem [35]. The energy of the step function in time domain $E _ { t }$ is given by

$$
E _ { t } = A _ { m } ^ { 2 } t _ { s i m }\tag{23}
$$

and the energy of the frequency spectrum $E _ { f }$ is calculated as

$$
E _ { f } = 2 \int _ { 0 } ^ { B _ { s } } \left( A _ { m } t _ { s i m } \right) ^ { 2 } s i n c ^ { 2 } \left( t _ { s i m } f \right) d f\tag{24}
$$

To obtain 99% for the ratio $E _ { f } / E _ { t } , B _ { s }$ can be calculated as

$$
B _ { s } = \frac { 1 1 } { t _ { s i m } }\tag{25}
$$

After determining the associated resolutions in (19) to (25), the width of the frequency window is calculated as

$$
f _ { c } = 2 f _ { N y } = 2 m a x \left( B _ { r } , B _ { i } , B _ { \ell } , B _ { s } \right)\tag{26}
$$

## C. Source Discretization

When energizing the circuit with a source at $t = 0 ,$ , there is a discontinuity between $0 ^ { - }$ and $0 ^ { + }$ . For the DTFS, this discontinuity results in loss of accuracy for the first time step, while in the NLT solution it produces high frequencies that need to be truncated by a filtering window to avoid Gibbs oscillations. An improvement to this discontinuity problem can be achieved (Fig. 2a, dashed line) by taking the average of the values before applying the source and after applying the source, as indicated in Fig. 2a (solid line). This makes the area from zero to $\Delta t$ equal to using the trapezoidal rule between these two points.

## C. Simulation Set up

The proposed DTFS and the NLT were programmed in MATLAB. The analytical solution given in (27) will be used as a reference in Section V to compare the DTFS and NLT simulations for the RL circuit test of Fig. 3a.

In the double-circuit line test, the frequency-domain simulations are compared with the time-domain frequency-dependent line models in the EMTP: fdLINE in MicroTran v3.25 and ULM in PSCAD v4.5.2. For the curve-fitting processes of fdLINE and ULM, the maximum number of poles was set to 35, and the frequency range was set from $1 0 ^ { - 2 }$ to $1 0 ^ { 7 }$ Hz with 10 points per decade. For fdLINE, the transformation matrices were calculated at 1 kHz.

For the DTFS regular sampling was used for all test cases and no windowing function is needed. To avoid division by zero when taking the inverse of $\mathbf { Y } ,$ the DC frequency was taken as $1 0 ^ { - 4 } ~ \mathrm { H z }$

For the implementation of the NLT, we followed the instructions given in [21]. The set up was as follows. Damping factors: Wilcox criterion $\sigma _ { 1 } \ [ 9 ]$ , Wedepohl criterion $\sigma _ { 2 }$ [10], and Error criterion $\sigma _ { 3 }$ (with $\varepsilon ~ = ~ 1 0 ^ { - 4 } )$ [36]. Sampling schemes: regular and odd. Windowing functions: Lanczos, Hannnig, and Tukey [37]. For the Tukey window, $2 Q / N = 0 . 3$ gave the most accurate results.

Table III shows the implementation of the guidelines of Section III to calculate the time-frequency parameters for the DTFS for the two test cases. There are three sets of parameters in this table labelled as User, Base, and Derived.

For the User parameters, the first step is to specify the wanted simulation time $t _ { s i m }$ . The user then needs to calculate the slowest time constant $\tau _ { m }$ using (17) and the frequency of the system and the frequency of the source using (19) to (25).

After specifying the User parameters in Table III, the Base parameters $T _ { c }$ and $f _ { c }$ are calculated next. These Base parameters determine the requirements of the frequency-domain solution. As discussed, the time window $T _ { c }$ is calculated as $t _ { s i m } + 7 \tau _ { m }$ . As also discussed, the simulation with the DTFS will be valid from $t = 0 \mathrm { \ t o \ } t = t _ { s i m }$ . The frequency window $f _ { c }$ is chosen as the largest of the frequencies $B _ { r } , B _ { i } , B _ { \ell } ,$ , and $B _ { s }$ in (26).

The results of Fig. 9 show that both the DTFS and the NLT present an initial error in the first sample of the simulation. This error is due to the initial discontinuity of the source.

Figure 9 also shows that the windowing functions corrupt the results of the NLT for the initial few samples of the simulation. No corruption occurs with the DTFS since no windowing function is used.

This shows the sensitivity of the NLT to the windowing function for the initial samples of the simulation. The maximum absolute error for DTFS, which does not use a windowing function, was significantly smaller than the NLT at only 0.9 V.

Ignoring the first few samples of the simulation, the average absolute errors for both methods when the time window is chosen as $T _ { c } = t _ { s i m } + 7 \tau _ { m }$ were very small in the order of 0.1 V.

## B. Double-circuit Line

The more complex example of a double-circuit line energization (Fig. 3b) is used for this test. Figure 10 compares the simulation results obtained with the DTFS, the NLT with regular sampling, the fdLINE and the ULM. For the NLT method, no large differences were observed between Lanczos, Hanning and Tukey windows, and Lanczos window was used for the comparisons. The time window width $T _ { c }$ and the frequency window $f _ { c }$ for both the DTFS and the NLT were selected based on the guidelines presented in Table III.

The results of Fig. 10 show that with the proposed guidelines the DTFS and the NLT with $\sigma _ { 3 }$ match the EMTP line models very well. However, for the NLT the choice of $\sigma _ { 1 }$ resulted in NaN error (indeterminate value) in MATLAB for which no output was produced, and $\sigma _ { 2 }$ gave erroneous result which will be labelled as $^ { 6 6 } \times \mathrm { A } ^ { , 3 }$ in Table IV. The deviation of the NLT with $\sigma _ { 2 }$ (red curve) from the correct result given by the DTFS and the EMTP line models in Fig. 10 is similar to the discrepancies reported in [5] between the NLT method and the EMTP line models for the simulation of double-circuit lines.

We also observed that for odd sampling the solution given by the NLT with $\sigma _ { 1 }$ and $\sigma _ { 2 }$ coincided with the benchmark solutions of Fig. 10 (not shown in Fig. 10) and with $\sigma _ { 3 }$ gave incorrect result (labelled as $^ { 6 6 } \times \mathrm { D } ^ { , 3 }$ in Table IV) which will be shown in Fig. 11.

To compare the effect of the choice of time window width on the DTFS and NLT results, we repeated the simulation condition of Fig. 10 for 31 simulation runs with $T _ { c }$ varied from 0.5 to 3.5 s, in increments of 0.1 s. With $t _ { s i m } = 0 . 0 5 ~ \mathrm { s } .$ the time window width is $T _ { c } = 1 . 0 3 \ \mathrm { s } .$ which corresponds to our recommendation $t _ { s i m } + 7 \tau _ { m }$

The DTFS solution does not use damping factors and is not affected by using regular or odd sampling. For faster solution, regular sampling is used for the DTFS.

## VI. CONCLUSION

This paper uses the Discrete Time Fourier Series (DTFS) to solve electromagnetic transients in the frequency domain. The key concept of this work is that the transient response of the system must fit inside the time window of the discrete Fourier series. Since the source delivers samples up to the last simulation point, the time window must include the slowest transient component of the system response to this last point. The minimum time window width must then be equal to the source duration (simulation time) plus the time for the transient terms to decay sufficiently to zero.

As in the traditional EMTP time-domain simulation, the width of the frequency window is determined by the maximum frequency we want to simulate in the transient.

The DTFS method, with the guidelines presented in this paper, provides a reliable tool for general purpose electromagnetic transients simulation in the frequency domain, with a number of advantages over the classical Numerical Laplace Transform (NLT).

The time window of the NLT is usually taken equal to the simulation time. As a result, the NLT requires the use of a damping factor to force the transient response of the system to fit inside of the time window. Despite the compression effect of the damping factor, this window width is often not sufficient to include all the transient response. The correct time window width to use with the NLT often requires trial and error. On the other hand, using the DTFS and choosing the time window width according to the guidelines and methodology proposed in this paper, provides very reliable results. This was verified in all cases presented in this paper.

The simulation results for the of the RL circuit show that the use of damping factors to smoothen the frequency response for the NLT may result in numerical errors at the signal’s tail, and that windowing functions are required to reduce Gibbs errors. These errors do not appear in the proposed DTFS solution.

Because the DTFS method provides a countable mapping between discrete time and discrete frequencies, it does not require, like the NLT, the use of windowing functions to avoid artifacts like aliasing and Gibbs that are typical of the NLT solution.

The simulations in this paper suggested that better results for the NLT were obtained using odd sampling, while the DTFS works equally well with regular sampling. This makes the DTFS solution twice as fast for the same time window width compared to the NLT.

Finally, the proposed DTFS is simpler to implement than the NLT since it does not require the user to specify damping factors and windowing functions, for which it is difficult to find simple guidelines, and is not sensitive to the use of sampling schemes.
