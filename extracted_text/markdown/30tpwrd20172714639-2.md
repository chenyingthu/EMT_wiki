# Parallel Electromagnetic Transients Simulation with Shared Memory Architecture Computers

Shengtao Fan, Member, IEEE, Hui Ding, Member, IEEE, Anuradha Kariyawasam, Student Member, IEEE, and Aniruddha M. Gole, Fellow, IEEE

Abstract-- This paper proposes a graph based matrix level partitioning methodology for parallel electromagnetic transient (EMT) simulation. By partitioning the graph associated with the admittance matrix, an efficient bordered block diagonal (BBD) matrix form is obtained, which is particularly suitable for parallel implementation. Even without the presence of distributed transmission lines, fully automatic system partitioning and parallelization of the simulation are achieved. The parallelized algorithm is implemented on a shared memory computer and assessed using two scalable test cases. The first involves an underground cable with many distributed parameter cascaded sections. The second is an interconnection of multiple instances of the IEEE 14 bus system, where only lumped transmission line models are used. The tests show that EMT simulation can be significantly accelerated. The scalability is tested with up to 64 cores.

Index Terms-- Bordered block diagonal, electromagnetic transient simulation, LU factorization, graph based partitioning, parallel processing

## I. INTRODUCTION

LECTROMAGNETIC Transient (EMT) simulation has become a fundamental tool in the design, operation and analysis of modern power systems. Application scenarios include transient simulations of large-scale power systems such as FACTS and HVDC systems [1], microgrids, shipboard power systems [2], cross-bonded cable systems [3] and so on. Such simulations require significant computer time. Traditional speedup approaches such as increasing clock speed or exploiting instruction-level parallelism (ILP) have now saturated. However, new trends in processor development have made multi-core processors commonplace, and thread level parallelism (TLP) has become a method of choice to increase a system's overall performance [4]. Therefore, it is necessary to investigate efficient approaches for parallel EMT simulation to fully utilize existing computer technology.

To parallelize EMT simulations, many attempts have been made for system decomposition. One commonly used approach uses the natural decoupling introduced by traveling wave propagation on transmission lines [5]. If the travel time exceeds the time-step, the two sides are computationally decoupled and hence paralelizabe, as one side cannot affect the other in the same time step. However this technique becomes difficult for simulation cases with no transmission lines or very short transmission lines (or cables) such as shipboard systems, microgrids or systems with dense power electronic clusters, where the small travel time may demand an unreasonably small time step. One approximate solution is to introduce artificial transmission lines (‘stubs’ lines) with travel times greater than one time step to implement the partition [6]. Alternatively other approximate approaches intentionally introduce a one time step delay to decouple portions of a network. Information is exchanged between the decoupled portions using controlled voltage and current sources [7]. However, the one time step delay can lead to inaccuracy or even numerical instability. To overcome these difficulties, the concept of diakoptics [8]-[11] and related methods [12]-[15] have been proposed. These do not rely on any natural parallelism such as travel times. Instead, parallelism is identified by analyzing the structure of the network. In diakoptics, the full network is reduced to a sparse interconnection of several subsystems with internal nodes collapsed. Due to this collapse, the solution of this network only involves a much smaller number of nodes. Once this network is solved, the terminal node solution of each collapsed subsystem becomes available, from which its internal node voltages can be obtained independently, i.e., in parallel. However, existing diakoptics type methods are circuit based and partitions are performed on the original electrical network. Although this has the merits of clear physical interpretation, fully automating the process is challenging and usually involves some amount of user intervention [15].

In this paper, a new fully automated methodology of partitioning and parallelization is proposed. To facilitate automation, instead of performing system partitioning at the network level, the proposed method uses a graph based approach to directly partition the admittance matrix into a bordered block diagonal (BBD) form. The resulting BBD matrix is further factored into a LU product, with blocked L and U matrices. As the diagonal blocks are only loosely coupled with each other through the boundary nodes, the method is particularly suitable for coarse-grain parallel implementation [16]-[19].

Additionally, the implementation uses a multiple instruction multiple data (MIMD) shared-memory architecture in which explicit instructions for data exchange between processors are not required, greatly facilitating the parallel implementation of the EMT simulation. In this paper, discussion is restricted to tightly coupled shared memory systems, e.g., a single CPU with multiple cores, or multiple single core CPUs. Such computers have small inter-processor communication latency (i.e., time to communicate between processors), which is very important for efficient thread synchronization.

Using the above algorithm and hardware implementation, the resulting EMT solver has the following desirable features: a) Parallelism is determined from the network structure. Systems with transmission lines need no special treatment.

b) The parallelized EMT algorithm utilizes the available multiple processors without any user interaction.

## II. PARALLELIZATION OF EMT SIMULATION

An overview of the EMT simulation outlining the main computational stages of a typical EMT simulation is first presented. Block LU factorization based parallelization and the partitioning scheme for obtaining bordered block diagonal (BBD) matrices are then introduced.

## A. Typical Process of EMT Simulation

The entire EMT simulation period is divided into small time steps. The network response in any time step is calculated from a known solution from the previous time step. In order to achieve this, the network elements are represented by their companion circuits consisting of resistors and current sources, the latter representing ‘history terms’ related to conditions in prior time steps or actual sources [20]-[22]. The values of the companion circuit being used depend on the choice of integration method, with trapezoidal integration being the most common [22]. Using nodal analysis, the network equation becomes a linear vector equation of the form YV J  , where V is the vector of node voltages to be solved for, Y is the nodal admittance matrix, and J is the vector of known currents (sources or history terms). When ideal voltage sources or some controlled sources are present, the method can be extended to a “modified” nodal analysis (MNA) method [23], where the vector of unknowns may also include some currents, and the admittance matrix may include elements that have units other than pure admittance, giving a more generalized form as in equation (1), with Y , V and J replaced by A , x and b . Once the unknown vector x is solved for, the history terms and the admittance matrix (in case any switches have operated) for the next time step are calculated. Time is then advanced by the time step and the solution for this new time step is generated. Fig. 1 shows the basic algorithm.

To solve nodal equations (1), a widely used procedure is first factorizing the admittance matrix into lower and upper triangular matrices. The solutions are then obtained through one forward and one backward substitution step [24]. If the nodal admittance matrix has been modified due to switching events etc., a refactorization of the admittance matrix into one lower and upper triangular matrix is also required.

Speeding up the EMT simulation requires careful identification of all parallelizable operations. These are discussed in the subsections to follow.

## B. Parallelization in calculation of history terms

The calculation of companion circuits for network component and updating of network states is inherently parallel as it only requires known voltages and currents e.g., from the previous time step. Parallelizing the network solution is more challenging as discussed in the section below.

## C. Parallel Solution of Nodal Equations

## 1) BBD Matrix and Block LU Factorization

A bordered block diagonal (BBD) matrix based method as [16] is used to achieve decomposition through exploring the sparse structure of the admittance matrix. A similar approach for power flow and transient stability calculations was reported in [16], where the matrix entries are phasors and all calculations are with complex numbers. Here it is adapted for electromagnetic transients simulation, where the representation is in the time domain and all elements are real numbers.

Suppose the modified nodal equation system to be solved is

$$
\mathbf { A } \mathbf { x } = \mathbf { b }\tag{1}
$$

where b is the right hand vector and x is the vector of unknown variables. The matrix A is a highly sparse matrix, which can be converted into the BBD form (2) through proper row and column permutations. As a decomposition method, BBD based approaches have been used in many areas of parallel calculation, such as circuit simulation [25], power flow calculation [26], and power system transient stability simulation [16], [27]. The construction of the BBD form will be discussed latter.

$$
{ \begin{array} { r l r } { \left[ \mathbf { A } _ { 1 1 } \right. } & { } & { \left. \mathbf { A } _ { 1 n } \right] \left[ \mathbf { X } _ { 1 } \right] } \\ & { } & { \mathbf { A } _ { 2 2 } } \\ & { } & { \ddots } \\ & & { } & { \left. \mathbf { A } _ { m n } \quad \mathbf { A } _ { m n } \right] \left[ \begin{array} { l } { \mathbf { X } _ { 1 } } \\ { \mathbf { X } _ { 2 } } \\ { \vdots } \\ { \mathbf { X } _ { m } } \\ { \mathbf { A } _ { n } } \end{array} \right] = \left[ { \begin{array} { l } { \mathbf { b } _ { 1 } } \\ { \mathbf { b } _ { 2 } } \\ { \vdots } \\ { \mathbf { b } _ { m } } \\ { \mathbf { b } _ { n } } \end{array} } \right] } \\ { \mathbf { A } _ { n 1 } \quad \mathbf { A } _ { n 2 } \quad \dots \quad } & { \mathbf { A } _ { n m } \quad \mathbf { A } _ { n n } \left[ \begin{array} { l } { \mathbf { X } _ { m } } \\ { \mathbf { X } _ { n } } \end{array} \right] } \end{array} }\tag{2}
$$

with,

$$
n = m + 1\tag{3}
$$

Variables $\mathbf { x } _ { i } , i = 1 , \cdots m$ , are the internal variables and $\mathbf { X } _ { n }$ denotes the boundary variables which couple with the internal variables.

A diakoptics like method as in [13]-[15] used previously for EMT simulation also generates similar BBD type matrices and thus can provide parallelism. However, to automatically exploit parallelism and achieve additional speedup, this paper explores the use of block LU factoring of the coefficient matrix. To solve (2), the matrix A is factored into a product of lower and upper triangular matrices L and U as in (4), which, due to the BBD structure of A , have the special structure as in (5) and (6).

$$

(6)

Internal submatrices $\mathbf { L } _ { i i }$ and $\mathbf { U } _ { i i }$ which lie along the block diagonal of L and U are obtained by a standard LU decomposition of $\mathbf { A } _ { i i }$ ，

$$
\mathbf { A } _ { i i } = \mathbf { L } _ { i i } \mathbf { U } _ { i i } , i = 1 \cdots m\tag{7}
$$

Border submatrices $\mathbf { L } _ { n i }$ and $\mathbf { U } _ { i n }$ are then readily obtained as in (8) and (9).

$$
\mathbf { L } _ { n i } = \mathbf { A } _ { n i } \mathbf { U } _ { i i } ^ { - 1 } , ~ i = 1 \cdots m\tag{8}
$$

$$
\mathbf { U } _ { i n } = \mathbf { L } _ { i i } ^ { - 1 } \mathbf { A } _ { i n } , \quad i = 1 \cdots m\tag{9}
$$

To obtain the lowermost submatrices $\mathbf { L } _ { \boldsymbol { n } \boldsymbol { n } }$ and $\mathbf { U } _ { n n }$ , we apply another LU decomposition to the left hand side of (10).

$$
{ \bf A } _ { n n } - \sum _ { i = 1 } ^ { m } { \bf L } _ { n i } { \bf U } _ { i n } = { \bf L } _ { n n } { \bf U } _ { n n }\tag{10}
$$

To solve the unknown variables, a forward substitution process is first performed on (11), which will give the solutions of intermediate variables y .

$$
\mathbf { L y } = \mathbf { b }\tag{11}
$$

where,

$$
\mathbf { y } = \left[ \mathbf { y } _ { 1 } \quad \mathbf { y } _ { 2 } \quad \cdots \quad \mathbf { y } _ { m } \quad \mathbf { y } _ { n } \right] ^ { t }\tag{12}
$$

After y is available, a backward substitution is then performed on (13), which finally gives the solutions for x .

$$
\mathbf { U } \mathbf { x } = \mathbf { y }\tag{13}
$$

From (11), one obtains:

$$
\mathbf { L } _ { i i } \mathbf { y } _ { i } = \mathbf { b } _ { i } , i = 1 { \cdots } m\tag{14}
$$

$$
\mathbf { L } _ { n n } \mathbf { y } _ { n } = \left( \mathbf { b } _ { n } - \sum _ { i = 1 } ^ { m } \mathbf { L } _ { n i } \mathbf { y } _ { i } \right)\tag{15}
$$

Similarly, from(13) :

$$
{ \mathbf U } _ { n n } { \mathbf x } _ { n } = { \mathbf y } _ { n }\tag{16}
$$

$$
\mathbf { U } _ { i i } \mathbf { x } _ { i } = \mathbf { y } _ { i } - \mathbf { U } _ { i n } \mathbf { x } _ { n } , \quad i = 1 \cdots m\tag{17}
$$

2) Parallelizing the solution of nodal equations

From inspection of (7) it is evident that each $\mathbf { L } _ { i i }$ and $\mathbf { U } _ { i i }$ can be calculated independently and in parallel. As soon as any $\mathbf { L } _ { i i }$ and $\mathbf { U } _ { i i }$ are calculated, they can immediately be used in (8) and (9) to calculate $\mathbf { L } _ { n i }$ and $\mathbf { U } _ { i n }$ . This gives the parallel threads shown in the block labelled $^ \circ \mathbf { A } ^ { \prime }$ in Fig. 2. However, calculating of $\mathbf { L } _ { \boldsymbol { n } \boldsymbol { n } }$ and $\mathbf { U } _ { n n }$ as in (10) is a serial task, which needs insertion of a synchronization point, i.e., all parallel paths in the previous step must be completed before proceeding further.

Similarly, the forward and backward substitutions for obtaining $\mathbf { y } _ { i }$ and $\mathbf { X } _ { i }$ in (14) and (17) can be implemented in parallel (block B in Fig. 2). Calculation of the boundary variables ${ \bf y } _ { n }$ and $\mathbf { X } _ { n }$ is necessary before the remaining unknowns $\mathbf { X } _ { i }$ can be determined. As values $\mathbf { y } _ { i } , i = 1 , \cdots m$ are needed for this purpose, another synchronization point appears between the forward and backward substitution steps.

From (10), (15) and (17), it is seen that some variables $( { \mathbf { L } } _ { n i } , { \mathbf { U } } _ { i n } , { \mathbf { x } } _ { n } , { \mathbf { y } } _ { n } . )$ are shared by different threads. As the shared memory environment provides a unified address for each shared data point in all parallel threads, the accessing of data between threads is straightforward and efficient.

After the forward and backward substitution steps, the node voltages are obtained. These are used for calculating the new history terms as described in sub-section II.B above. This step is depicted in the block labelled $\mathbf { \tilde { C } } _ { } ^ { \prime }$ in Fig. 2.

## D. System Partitioning

An important contribution of this paper, key for automated factoring and high speedup of the EMT algorithm, is the approach of partitioning. This includes two separate steps.

## 1) Clustering of similar components

In EMT simulation, elements such as inductors, capacitors transmission lines etc. are modelled using companion circuits consisting of a fixed conductance in parallel with a history current source, which is updated in each time step [20]. Updating network states as well as deriving the companion circuit for each element are independent processes, as only historical data is required. In order to achieve good load balance, all the components are firstly grouped according their type (e.g., all the capacitor components are divided into one group, all transformers belongs to another group). Then the components in each group are equally assigned to each thread. Since the computation burdens of similar component types are usually equal, this approach achieves good load balance.

## 2) Partioning to obtain BBD Structure

For solving of nodal equations, the parallel algorithm based on BBD structure as described in Section II-C is used. The success of the algorithms relies heavily on how BBD matrix is constructed. In power flow and transient stability applications, the single-line diagram of the power electrical network can be readily used to give high quality partitions with graph based approaches [28]-[31]. However, due to the complexity of the EMT models, the sparse structures of the admittance matrices are not directly evident from the single-line diagram of the system. Therefore, in contrast to previous approaches, this paper transforms the construction of the BBD into a graph partitioning problem. A graph is directly obtained from the structure of the modified admittance matrix and is used to partition and parallelize the nodal equations [20].

The admittance matrix A is typically a highly sparse mainly-symmetrical matrix, which can be represented by an undirected graph. The graph has n vertices, where n is the order of the matrix. To generate the graph, an edge is added between vertices i and j , if there is a pair of none zero elements $A _ { i j }$ and $A _ { j i }$ in the admittance matrix. With modified nodal analysis, A can be asymmetric. Then $\mathbf { A } + \mathbf { A } ^ { T }$ is used to generate the corresponding graph.

The procedure is explained with the simple 5 x 5 modified admittance matrix given in (18), whose graph is shown in Fig. 3. As there is an pair of off diagonal elements $( G _ { 2 } { \mathrm { a n d } } G _ { 3 } )$ between node 1 and node 2, the graph in Fig. 3 contains an edge between nodes 1 and 2. There is no off diagonal nonzero elements between nodes 1 and 4, hence the corresponding edge is absent in Fig. 3, and so on. This graph is partitioned into three parts, consisting of the block diagonals BD1 and BD2 and the boundary. The partitioning follows the rule that the diagonal blocks only connect to the boundary and must have no direct connections with each other. Typically this partitioning is not unique. Here only the single node 2 is included in the boundary and the block diagonals are formed with nodes {1} and nodes {3,4,5} to give the BBD matrix structure as in (19).

In the above example, the size of blocks is 1x1 and 3x3 and the boundary has one node. For the greatest computational efficiency, the block sizes must be nearly equal and the boundary nodes must be as small as possible. For large systems, portioning by hand as was done with the above graph is not practical, so automated tools are necessary. In this work, the graph partitioning tool METIS [32] was used, which produces high quality partitions, i.e., ones with small borders and roughly equal block sizes. The METIS's multilevel k-way partitioning scheme firstly reduces the size of the original graph by collapsing vertices and edges, yielding a smaller graph. Then a initial k-way partitioning of this smaller graph is found. Finally, refinement of the partitioning is performed to construct a high quality k-way partitioning for the original graph [31]. The appendix gives an example to illustrate the process of admittance matrix partitioning in a realistic system.

It should be noted that after obtaining the BBD structure, certain rows and columns may still be permutated without destroying the BBD structure. For example in (19) if rows 2 and 4 as well as columns 2 and 4 are interchanged, the BBD structure of the matrix is preserved. Some of these permutations result in more non-zero elements in L and U matrices in (5) and (6) than other choices, making the solving of the nodal equation slower. A re-ordering within each partition can be performed on A to give L and U matrices with less non-zero fill-ins using special ordering algorithms. The approximate minimum degree ordering algorithm [33] is used in this work.

E. Overcoming Limitations of Previous Partitioning Approaches

In non EMT simulations, i.e., load-flow or transient stability simulations, the single-line diagram of the original system itself is typically used to perform the system partitioning using graph based approaches. These programs use a phasor representation for the network, which gives a set of algebraic equations (i.e. an admittance matrix with complex elements). A sequence transformation is further used to decouple the admittance matrix into three independent ones. The branches typically include transmission lines and transformers, which are usually represented as T or Π equivalents. Hence, from the viewpoint of balancing computational burden, all the edges in the graph contribute almost equally. However, this methodology doesn’t work well with electromagnetic transients simulation, as different branches can have very different computation burdens.

In the past, partitioning in EMT simulation has been implemented using either the natural decoupling from transmission lines [5] or diakoptics type approaches [12]-[14]. However, branches in EMT simulation can range from complex frequency-dependent transmission line models to much simpler inductors or resistors. These have very different computational burdens, and good partitioning typically requires user intervention [15]. In contrast, the proposed partitioning approach is completely matrix based, and hence can be entirely automated. The proposed BBD approach also generates high quality partitions directly from the admittance matrix structure, which incorporates parallelism resulting from transmission delays as well as from sparsity.

To lower the circulating sheath currents and prevent sheath over-voltages [36], many cable systems are cross-bonded as shown in Fig. 4. When voltages and currents at all cross bonding points are a concern, individual cable segments of the cascaded power transmission system need to be modeled separately.

In this test case, the simulated cable system consists of 320 sections, each represented by a universal transmission line model [37]. Simulation using the traditional serial EMT approach is extremely time consuming due to the highly detailed representation, the large number of sections, and the small time step of 1 micro-second which is necessary due to the short length of each cable section. ## C. 7000 Bus Test System

The 7000 bus test system is constructed by replicating the IEEE 14 bus system shown in Fig. 7, five hundred times. Adjacent systems are connected at buses 13 and 14. The parameters for each 14 bus system are as in [34]. The transmission lines are modeled by coupled Π sections; the loads are modeled as three phase resistors, inductors and capacitors.

As all transmission lines are modelled as lumped Π sections, there is no intrinsic decoupling due to travel time delays. Nevertheless, the proposed factorization via BBD with block LU matrices can automatically partition the solution to parallelize the simulation. Due to this lack of decoupling, the border size is not zero and so the non-parallelizable equations such as (10) must be solved. These equations involve boundary element matrices such as $\mathbf { L } _ { \boldsymbol { n } \boldsymbol { n } } \mathrm { ~ , ~ } \mathbf { U } _ { \boldsymbol { n } \boldsymbol { n } }$ , etc. The smaller the size of the boundary elements in A, the lesser the number of equations in (10), (15) and (16). Therefore, the size of boundary element matrices should be minimized. The METIS multilevel k-way partitioning approach [31] is used to minimize the border size of the BBD form matrix. Fig.8 gives the border size with an increasing number $" n '$ of partitions. It can be seen that the size of the border increases with the number of partitions. However, even when the number of partitions is 64 (the limit of our hardware, with one core used for each partition), the border size is 612, which is still relatively small (2%) when compared with the total size (\~30K) of the full nodal admittance matrix.

## IV. CONCLUSION

This paper investigates a new method for parallelization of electromagnetic transient simulation on tightly coupled shared memory architecture computers. To fully automate the parallelization, a graph based matrix level partitioning approach is proposed to obtain the BBD form of the admittance matrix. This is followed by the block LU factorization of the BBD form. A highly efficient parallel algorithm for solving the nodal admittance equations is developed. The decoupling effect of cables or transmission lines need not be explicitly modelled, as the approach automatically caters for it based on the admittance matrix structure.

Cases with and without distributed cable or transmission line models are used to test the performance of the implementation, with the number of cores ranging from minimum of 1 to a maximum of 64. In the case with distributed cable models, a speedup gain of more than 40 is observed when using 50 cores. However, increasing the partitions (i.e. number of cores) beyond that value actually results in a reduced speed-up as the thread-synchronization overhead becomes dominant.

For the network with no obvious circuit level partitioning resulting from travel delays across cables or transmission lines, significant maximum speedup of about 20 was still possible with 30 cores, before it began to level off due to more nonzero fill-ins.
