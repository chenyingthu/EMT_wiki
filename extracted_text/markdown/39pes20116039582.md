# Use of XML to import SCADA data into EMTP

S.Dennetiere,Member, IEEE,A. Parisot, Member, IEEE,E.Milin,A. Dalmau Pons

Abstract-In the next few years many installations of long EHV AC cables and HVDC links are planned on the French grid. Prior to these huge system changes various EMT studies have to be conducted to prevent problems.Some of these studies require large scale network models.To limit errors when building large networks and to gain time, the French TSO RTE has decided to develop an interface between system tools and EMTP.This XML based interface enables SCADA data to be automatically imported into an EMTP graphical user interface.This paper describes the environment developed in EMTP-RV,imported data from SCADA to EMTP and studies performed with the full models.Simulation speed, data availability and model reduction issues are addressed in this paper.

Index Terms— data portability，EMTP,Electromagnetic transient studies,large scale networks modeling.

## I．INTRODUCTION

I HE French TSO RTE (Reseau de Transport d'Electricite) operates,designs and maintains the largest network in Europe,at the center of the European electricity market. To carry out needed reinforcements in the French grid in a context of dwindling public acceptance for new overhead infrastructure,many installations of long EHV cables, capacitor banks and HVDC links are planned in the next few years. These new components may give rise to unusual electromechanical and electromagnetic phenomena with potential adverse effects on the existing network. Therefore, specific studies have to be conducted，from the earlier planning to final commissioning stages.

The classical approach to conduct studies based on simulation tools is to establish a compromise between precision， computational speed and available data. The ultimate goal is to achieve the most precise calculations with allrelevantmodelingdataandwithinminimal computational time. Depending on the type of studies the power system engineer will therefore use different methods, databases and simulation tools,such as load-flow, transient stability packages and EMTP software. As a consequence there has always been an issue concerning the data portability between these tools.

This paper presents a solution implemented and tested at RTE on real cases to

save time building EMTP network models as they have already been modeled for load-flow or transient stability studies,

ensure that the models used from load-flow studies to EMTP studies are consistent,

ensure proper management of data when modeling large scale networks,

· be able to run load-flow， time-domain and frequency scan calculations in the same platform,

· harmonize the visual drawing of the network in EMTP with other standard representation used in network operation and planning.

The proposed solution is based on :

· XML data import from SCADA to EMTP,

·library of components to create a network model in EMTP that looks like standard RTE grid representations in system studies.

To the author's best knowledge it is the first time that such an approach is implemented to get a dynamic link between classical large scale network simulation tools and EMTPtype software. Frequencyand time domain simulations on the entire 40o kV French grid have been successfully performed. The software used in this paper is EMTP-RV [3].This choice is related to previous work conducted by the authors and also the required user interface capabilities.Nevertheless the approach proposed in this paper is independent from the EMTP simulation tool.

Simulation of large scale networks in EMTP has already been presented in previous works. The approach proposed in [1] does not include data portability between simulation tools.Usage of fundamental frequency multi-port network equivalents is another solution presented in [2] to reduce system size.However, this solution is not optimal because network equivalents have to be rebuilt for every network change (topology, parameters,...).

## II.NEEDS AND REQUIREMENTS FOR LARGE SCALE NETWORK MODELING ATRTE

Every installation of new components requires EMTP studies with a network model of the area. A large scale network model that represents a specific network topology and a load situation is useful to get a relevant starting point of EMTP studies and to get an accurate model of the network for frequency scan studies.

## A. Starting point of EMTP studies

In the case of time-domain simulation of transient phenomena a limited part of the network is usually modeled.

A large scale network model coming from a load-flow or a transient stability study is useful in order to get a relevant starting point for the electromagnetic transient studies. Load-flow datacan be based either on SCADA measurements and state estimations or on calculations on planned networks.

## B. Network models validity

Long EHV cable installations can lead to harmonic temporary overvoltages [4]-[5]. The cable-network system impedance can present low order harmonic resonances that can lead to harmonic resonances at energization of no-load power transformers. Based on the network short-circuit power at the cable supply bus bar and the charging reactive power associated to total capacitance seen at cable supply busbar,a preliminary assessment of the risk of resonance can be obtained [5]. This simple expression only provides a quick estimate of the first resonance frequency.As low order harmonic resonance is of particular concern in power systems,a more detailed analysis can be carried out for assessing the network harmonic impedance.

For EHV network modeling,a Joint CIGRE/CIRED WG CC02 document [6] recommends to model accurately at leastthecompleteprimarytransmissionnetwork. Furthermore it is appropriate to consider the loads on the secondary transmission network in order to decide whether these should be modeled explicitly or as an equivalent circuit.If these loads are placed directly on the secondary side of the transformer, the damping they cause can be overestimated when using simple equivalents.

Asa consequence to accurately assess the harmonic impedance close to new EHV cable installation, it is needed to model at least the 400 kV French grid.

## C. Graphical User Interface (GUI) requirements

When data of large scale networks are imported in EMTP, the next challenge is to handle such a large amount of data. Building and maintaining large networks using the GUI mouse based functions is not viable. Making data changes in many devices,searching for equipments or updating models to the type of studies must be achieved efficiently.

Moreover large networks schematics are already available and do not depend on a specific simulation tool. Thus it is relevanttodisplaythesenetworksinthesame representation as is used in system operation and planning.

## III.INTERFACE DESCRIPTION

## A. CONVERGENCE platform

Many years ago RTE identified the need of a platform gatheringnetworkdata andtoolsforsteady-state simulations (load-flow and short-circuit calculations).As of now,a first version of this platform has been developed and provides the following features :

· Description of the entire RTE network from 400 kV to 63 kV: sequence impedances of lines (directandzero-sequence)，transformer parameters(impedances, tap changer positions,...)，generators (sequence impedance, voltageandpowerreferences)，loadsand substation configurations

·network data in real-time: RTE network snapshots are available every 5 minutes,

· network data for planning studies,

· load-flow and short-circuit calculations in the same platform.

Network data and load-flow results can be exported from the CONVERGENCE platform (see Fig.1） into an XML file.

## B. EMTP-RV graphical user interface

The Graphical User Interface of EMTP-RV is named EMTPWorks. As explained in [3] almost all the aspects of this GUI are scriptable and modifiable by the user. Data input is based on web pages.It provides quasi unlimited capabilities in the development of data forms through dynamic HTML and XML programming. Users can create their own user interface.Forms, device drawings, simulation options can be controlled through JavaScript language. The scriptinglanguageisJavaScriptwithEMTPWorks extensions.

To build a network in EMTP that replicates standardized system representations,a set of service devices has been developed as presented in Fig.2.The network in EMTP is built manually by the user based on these service devices.

Parameters of these devices can be modified through custom made forms as presented in Fig.2.Busbar connections are automatically updated on the drawing depending on data available in data input forms.

These standard devices (for instance an outgoing line feeder connected to a 3 busbar substation） are used in several places in the drawing.As explained in [1]a mechanism is available in EMTPWorks for updating all subcircuit contents by modifying the internal details of only one instance.This object oriented mechanism is called unique type definition. A unique type definition can be maintained for contents， but mask data canvary independently.

To import the XML data into EMTPWorks the Microsoft

COM implementation of the XML Document Object Model (DOM) has been used. The XML DOM provides a navigable set of classes that directly reflect the W3C Document Object Model Level 1 specification. This DOM implementation provides JavaScript interfaces and thus can be used in EMTPWorks.

Toillustrate the XML DOM implementationin EMTPWorks, the following JavaScript source is provided. The active power reference of a generator named 'PORC2P7' is updated in the EMTP network from the value available in the XML document. XML DOM is activated through the JavaScript COM interface.The XML file exported from the CONVERGENCE platform is read by EMTPWorks JavaScript functions and converted into a string format in less than ls.This string contains the entire XML document. For example, to locate the power reference of a specific generator, it is possible to use path expressions to select nodes in the XML document.

```javascript
//create the xml Document Object Model
xmldoc = new ActiveXObject("Msxml2.DOMDocument.3.0");
//EMTPWorks JScript functions to work with files
var myfileXML=SPFile();
myfileXML.fulIName='topo_reseau_04012010.xml'
//Reads the contents of the file and returns a string
//This string contains the entire XML document
var xmlContent = myfileXML.readText();
//Loads an XML document using the supplied string
xmlDoc.loadXML(xmlContent);
//Locate generator named 'PORC2P7' in XML document and get its
active power reference
var XpathPref = "//generator[name=" 'PORC2P7'"]//Pref"
var prefValue = xmlDoc.selectNodes(XpathPref);
//Locate generator named 'PORC2P7' on network schematic
var cct = currentCircuit()
var myGen = cct.devices('Name','PORC2P7")
//Use Generator class definition in EMTPWorks
parseScriptFile('generator_m.dwj");
//Create the object from device selection
oGen = new Generator(myGen[0]);
//Update power reference
oGen.Pref = prefValue
```

Following the XML document loading, scripts are used to update the network parameters: lines parameters, generators voltage and power references,circuit breakers /disconnect switch positions, etc. When network parametersare updated, the discrepancies between the network schematic in EMTP and the network data in the XML document are reported into a log file.Additional functions are also implemented to:

· locate a substation based on its name,

·apply a ratio on loads for selected load devices,

·replace CP-line models with propagation delays smaller than the integration time-step by PI-line sectionswith their correspondingsequence impedances.

## IV. TEST CASES

## A. 400 kV French grid

Data of the entire RTE network and a limited part of ENTSO-E network is available in the CONVERGENCE platform. The listing of main devices in XML document contains:

· 4731 transmission lines

· 557 generators

·880 power transformers

· 4482 loads

The XML document is loaded in less than 1s.The network schematic developed in EMTP includes the entire 400 kV French grid. The total system model is composed of 419 transmission lines operating at 40o kV and 91 synchronous generators. The total number of electrical nodes is 1,211 (around 400 3-phase buses). Transmission lines are modeled using the CP model where lines with propagation delays smaller than the integration time-step are automatically replaced by PI-line sections with their corresponding sequence impedances.

## B.Frequency scan studies for long EHV cable installations

The 400 kV network presented in the previous section has been used to analyze the relevant study area for frequency scan studies.Frequency scan studies are performed in RTE as part of projects for long EHV cable installations.In these cases the local 225 kV network is added as well.

To analyze the impact of the study area, various network reductions are compared. The network is modeled up to 1, 2 and 3 nodes away from the study points. Study points are the cable connections to the EHV grid. Links between successive nodes are lines as well as transformers.Networks beyond the border nodes are represented by short-circuit impedances calculated at power frequency.

Frequency-scan studies are carried out with reduced and complete networks.Harmonic impedance amplitudes at connection points are presented in Fig. 3． Network reduction based on short-circuit impedance leadsto overestimation in the magnitudes of resonances.Asa consequence RTE decided to use the approach proposed in this paper to model the entire 40o kV network and the local 225kV network for EHV cable installation studies. CIGRE/CIRED working group CC02 [6] came to the same conclusion but this paper allows to justify and support such a conclusion.

## V. CONCLUSIONS

Due to numerous installations of long EHV cables and HVDC links on the French grid,many EMTP studies will have to be conducted in the next few years at RTE.The large number of studies and the extended size of network models require robust and fast tools to deliver highest accuracy.. It has been demonstrated that XML data usage for the representation of large scale networks in EMTP is a time-saving feature for power system engineers.It also contributes to ensuring the validity of the studies.

The approach presented in this paper is based on a proprietary XML data format used to import SCADA data into EMTP.The problem of information exchange between various types of power system software is not new. The Common Information Model (CIM) attempts to facilitate information exchange in power system software.A CIM/EMTP-RV interface has already been developed in the past [8]. This interface has not been used by RTE because it is based on bus-branch topology description. Moreover, up to now the CONVERGENCE platform that gathers network data, does not propose a CIM compliant export feature.

The approach proposed in this paper gives practical examples to encourage the definition of a standard data format for power system analysis tools.It should be valuable to analyze potential integration of a standard data exchange format as CIM-XML in the CONVERGENCE platform. Use of a standard format, such as CIM-XML, would provide a reliable service in this context.
