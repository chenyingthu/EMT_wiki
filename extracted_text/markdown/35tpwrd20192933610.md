# Small Time-Step FPGA-based Real-Time Simulation of Power Systems Including Multiple Converters

Ramin Mirzahosseini, Reza Iravani, Fellow, IEEE.

Abstract—This paper presents an FPGA-based Digital Real-Time Simulation (DRTS) environment for the analysis of Electromagnetic Transients (EMTs) of power systems that include multiple Power Electronic Converters (PECs). The main features of the proposed DRTS platform are that it i) employs two-value resistor switch model for PECs and thus avoids inaccuracies of Associate Discrete Circuit (ADC) model and ii) does not require separation of PECs by transmission lines (stub lines) and thus accurately represents PECs in close electrical proximity of each other. These features respectively result in a time-variant and large system admittance matrix which significantly increases the computation burden in each simulation time-step and can render it infeasible for real-time applications. To overcome this problem this paper introduces i) Switching-Network Partitioning (SNP) method which is tailored for PECs representation in FPGA and ii) Reformulated Modified Nodal Analysis (RMNA) method to enable computational parallelism between the solution of network and component models in FPGA. SNP enables solutionlevel partitioning of the PECs and the rest of the system. The developed DRTS platform enables sub-microsecond simulation time-step and provides a fixed hardware irrespective of the system configuration. Performance of the platform is evaluated for a study system and verified versus off-line simulation studies.

Index Terms—Real-time systems, Field programmable gate arrays, Power electronics, Electromagnetic transient analysis

## I. INTRODUCTION

Real-time simulation of power system EMTs is the basis for the Hardware-In-the-Loop (HIL) environment to evaluate and test control/protection platforms [1]. Nodal Analysis approach is mainly employed in the existing production- and researchgrade DRTS platforms [2] [3] [4]. Continuously increasing number of PECs in power system necessitates real-time simulation of large systems with a small simulation time-step (less than 1 µs). This has become major challenge for these platforms.

Simulation of power system EMTs based on GPU and FPGA has been reported in the literature. GPU, due to its memory and processor structure, has been employed to accelerate off-line simulation [5] or relatively large time-step DRTS applications [6]. Reconfigurable parallel processing capability and on-chip memory of the FPGA make it suitable for small time-step DRTS applications. The objective of this work is to introduce an FPGA-based DRTS platform to address the above-mentioned challenges. This is based on a fresh reevaluation of i) PEC models, ii) partitioning of the overall system model and iii) formulation of the system model for enhanced parallelism.

ADC switch model [7] provides computational advantage for simulation of PECs, and has been implemented on CPUbased [8] and FPGA-based [9] [10] DRTS platforms. However, the drawbacks of this method, i.e., artificial switching loss and spurious oscillations, limit its applications for systems embedding High-frequency PECs. Several attempts to enhance the ADC model have been reported, including: incorporation of conduction losses [11], reducing the oscillations by adjusting the ADC model parameters, based on the eigenvalues of the system model in z-domain, for a simple single PEC system [12], and optimizing the model parameters to damp certain oscillatory modes for specific PEC configurations [13].

To overcome the drawbacks of the ADC switch model, i.e., artificial switching loss and spurious oscillations, we adopt the two-value resistor model. The latter model results in timevariant system Y matrix which in turn imposes unacceptable computational burden for real-time calculations. This paper introduces a novel partitioning method to overcome this drawback of the two-value resistor model.

Partitioning methods divide the power system mathematical model into subsystems to enable parallel computation in each time-step, and can be devised at modeling, discretization, or solution stages. Modeling-level partitioning exploits physical lines or artificial stub-lines [14] [15] [16] to achieve partitioning, however, introduces parasitic elements which may lead to spurious oscillations. This approach is not suitable for multiple PECs in close electrical vicinity of each other.

Discretization-level partitioning adopts explicit integration methods, e.g., Forward Euler (FE), [17] [18] [19] and can lead to numerical instability due to limitations in their stability characteristics [20] and stiff nature of EMT equations [21] [22]. A necessary condition for stability in the specific case of modular multilevel PEC is reported in [23] .

Solution-level partitioning is based on implicit integration methods [24] [25] [26]. As compared with the former two methods, solution-level partitioning is computationally more demanding, but does not cause numerical instability or oscillations. An FPGA-based solution-level partitioning is proposed in [27] based on customized floating-point cores and an iterative process to find the switch states. The method is applied to a neutral-point clamped converter and the system time-variant admittance matrix is handled using Gauss-Jordan elimination. This method is effective in cases where the system level matrix size is relatively small, i.e., 4×4 in [27].

In [28] the system model is partitioned by predicting the capacitor voltages. This method partitions the system if there exist enough capacitors connected to the partitioning nodes. In [29] an explicit method is proposed for simulation of PECSNs using fixed-point numerical representation. Since PECSNs render the system equations stiff, using explicit integration methods for their representation can lead to instability. High-Level Synthesis (HLS) is employed to implement the ADC switch model in [30]. In [31] FE is employed to avoid solving an admittance matrix. Instead, a switch state matrix is constituted and solved on-the-fly using Cholesky factorization based fixed-point numerical representation.

This paper introduces Switching-Network Partitioning (SNP) method, which is a solution-level partitioning method for multiple PECs applications and suitable for FPGA-based implementation, based on implicit integration method. This method assumes that the external circuit to the PECSNs in the system features constant admittance and focuses on simulating the network and the PECs, using two-value resistor model. This assumption enables handling large system matrices and therefore simulating relatively large systems. Simulating other power system component models in this method requires modeling approaches that satisfy the assumption. It is also assumed the switching process of each PECSN is independent of those of the other PECSNs in the system.

Furthermore this paper i) adopts RMNA formulation to enable parallel solution of network equations and component models and ii) presents the design of the envisioned FPGAbased DRTS, based on standard floating-point cores, to achieve sub-microsecond time-steps.

The rest of this paper is organized as follows. Section II presents RMNA formulation. Section III explains the SNP method for simulation of PECs. The design of FPGA-based DRTS platform is presented in Section IV, and section V presents simulation results using the devised platform and compares them against those obtained from off-line simulation cases. Section VI concludes this paper.

## II. FORMULATION OF NETWORK EQUATION

## A. Modified Nodal Analysis (MNA)

Consider a network with a time-invariant admittance matrix, i.e, with no switching converters. The network differentialalgebraic equations constitute a stiff system which is best numerically integrated by implicit numerical integration methods, e.g., Trapezoidal (TR) or Backward Euler (BE) method. The network equations based on Modified Nodal Analysis (MNA), which also enables inclusion of voltage sources, are [20]

$$
\underbrace { \left[ \mathbf { B } _ { g } ^ { \phantom { \dagger } } \mathbf { \Lambda } \mathbf { \Sigma } \mathbf { \Sigma } \mathbf { \Sigma } \mathbf { \Sigma } \mathbf { \Sigma } \mathbf { \Sigma } \right] } _ { \mathbf { Y } } \underbrace { \left[ \mathbf { \bar { v } } _ { { n + 1 } } ^ { n + 1 } \right] } _ { \mathbf { \Sigma } \mathbf { \Sigma } \mathbf { \Sigma } \mathbf { \Sigma } \mathbf { \Sigma } \mathbf { \Sigma } \mathbf { \Sigma } \mathbf { \Sigma } \mathbf { \Sigma } \mathbf { \Sigma } \mathbf { \Sigma } \mathbf { \Sigma } \mathbf { \Sigma } \mathbf { \Sigma } \mathbf { u } _ { n o d e } ^ { n + 1 } } = \underbrace { \left[ \mathbf { u } _ { i } ^ { n + 1 } \right] } _ { \mathbf { \Sigma } \mathbf { \Sigma } \mathbf { \Sigma } \mathbf { \Sigma } \mathbf { \Sigma } \mathbf { \Sigma } \mathbf { \Sigma } \mathbf { \Sigma } \mathbf { \Sigma } \mathbf { \Sigma } \mathbf { u } _ { n o d e } ^ { n + 1 } } ,\tag{1}
$$

or equivalently

$$
\mathbf { Y } \mathbf { x } _ { n o d e } ^ { n + 1 } = \mathbf { u } _ { n o d e } ^ { n + 1 } ,\tag{2}
$$

where Y is the admittance matrix, $\mathbf { u } _ { n o d e } ^ { n + 1 }$ is the nodal injection vector and $\mathbf { x } _ { n o d e } ^ { n + 1 }$ is the vector of unknowns. $\mathbf u _ { i } ^ { n + 1 }$ is the current injection vector. $\mathbf { v } _ { s } ^ { n + 1 }$ is the vector of voltage sources, $\mathbf { v } ^ { n + 1 }$ and $\mathbf { i } _ { v s } ^ { n + 1 }$ are vectors of unknown node voltages and voltage source currents, respectively. G is the conductance matrix. Matrices $\mathbf { A } _ { g }$ and $\mathbf { B } _ { g }$ define node incidence of independent/dependent voltage sources and dependent current sources.

Let’s assume the network includes components that do not change the admittance matrix, e.g., independent voltage/current sources, capacitive/inductive branches and transformers. Therefore,

$$
\mathbf { u } _ { i } ^ { n + 1 } = \mathbf { i } _ { s } ^ { n + 1 } + \mathbf { i } _ { h } ^ { n + 1 } + \mathbf { i } _ { c } ^ { n + 1 } ,\tag{3}
$$

where $\mathbf { i } _ { s } ^ { n + 1 } , \mathbf { i } _ { h } ^ { n + 1 }$ and $\mathbf { i } _ { c } ^ { n + 1 }$ are vectors of independent, branch history and component history current sources, respectively. Contribution of each component to Y of (1) is as follows.

1) branch: The differential equation of each branch is discretized, e.g., that of an inductor branch is [21]

$$
i _ { h } ^ { n + 1 } = c _ { h } i _ { h } ^ { n } + c _ { v } v ^ { n } ,\tag{4}
$$

where $i _ { h }$ is history current and $c _ { h }$ and $c _ { v }$ are respectively history and voltage coefficients.

2) Transformer: Figure 1 depicts a lumped model of a single-phase transformer and the equivalent circuit corresponding to its discretized model. The inductive branches are treated based on (4) and the ideal transformer is represented by dependent sources described by

$$
\begin{array} { c } { { v ^ { n + 1 } ( n _ { 1 } ) - v ^ { n + 1 } ( n _ { 2 } ) = n _ { T r } ( v ^ { n + 1 } ( n _ { 3 } ) - v ^ { n + 1 } ( n _ { 4 } ) ) , } } \\ { { i _ { p r } ^ { n + 1 } = n _ { T r } { \bf i } _ { v s } ^ { n + 1 } ( m ) , } } \end{array}\tag{5}
$$

where $n _ { T r }$ is the transformer turns ratio and $i _ { p r }$ is the current source representing the primary side. Based on (5), the contribution of transformer model is

$$
\begin{array} { l l } { { { \bf { A } } _ { g } ( n _ { 1 } , m ) = n _ { T r } , } } & { { { \bf { B } } _ { g } ( m , n _ { 1 } ) = n _ { T r } , } } \\ { { { \bf { A } } _ { g } ( n _ { 3 } , m ) = 1 , } } & { { { \bf { B } } _ { g } ( m , n _ { 2 } ) = - n _ { T r } , } } \\ { { { \bf { A } } _ { g } ( n _ { 2 } , m ) = - n _ { T r } , } } & { { { \bf { B } } _ { g } ( m , n _ { 3 } ) = 1 , } } \\ { { { \bf { A } } _ { g } ( n _ { 4 } , m ) = - 1 , } } & { { { \bf { B } } _ { g } ( m , n _ { 4 } ) = - 1 . } } \end{array}\tag{6}
$$

The tasks to solve (1), in each time-step, include:

1) Calculating independent voltage (current) sources $\mathbf { v } _ { s } ^ { n + 1 }$ $( \mathbf { i } _ { s } ^ { n + 1 } )$ ;

2) calculating branch history current source vectors $\mathbf { i } _ { h } ^ { n + 1 }$

3) calculating component history current $\mathbf { i } _ { c } ^ { n + 1 }$

4) deducing vector of unknowns $\mathbf { x } _ { n o d e } ^ { n + 1 }$ from (1).

To enable small time-step for real-time solution, parallelism within each task and also between the tasks is desirable. The latter is referred to as “task-level” parallelism. The above tasks do not feature task-level parallelism and therefore not enable efficient FPGA-based implementation. Next section addresses this issue.

## B. Reformulated Modified Nodal Analysis (RMNA)

For time-invariant Y either pre-calculated inverse $( \mathbf { Y } ^ { - 1 } )$ or matrix triangularized factors, e.g., LU factors [20], can be used. The former involves matrix-vector multiplication and the latter necessitates forward-backward substitution which i) is inherently sequential and ii) requires floating-point division operation. The latter approach cannot best exploit the FPGA parallelism properties, therefore, we adopt pre-calculated $\mathbf { Y } ^ { - 1 }$ to achieve solution of (1) within a small time-step, i.e.,

$$
\mathbf { x } _ { n o d e } ^ { n + 1 } = \mathbf { Y } ^ { - 1 } \mathbf { u } _ { n o d e } ^ { n + 1 } ,\tag{7}
$$

which provides specific features for parallelism. The structure of $\mathbf { u } _ { n o d e } ^ { n + 1 }$ however prevents task-level parallelism and makes (7) unsuitable for FPGA implementation. The reason is that the first three tasks contribute to $\mathbf { u } _ { n o d e } ^ { n + 1 }$ entries and thus, in a general case, it is available only after their computations are complete, i.e., the last task is sequential to the first three tasks. Task-level parallelism is more critical than parallelism within each task, since in a fully task-level parallel scheme all tasks are carried out simultaneously, and the time-step is decided by the computation time of the longest task. We propose the new nodal injection vector

$$
\mathbf b _ { n o d e } ^ { n + 1 } = \big [ \mathbf b _ { s } ^ { n + 1 } \mathbf b _ { h } ^ { n + 1 } \mathbf b _ { c } ^ { n + 1 } \big ] ^ { T } ,\tag{8}
$$

which enables parallelism in (7) and is related to $\mathbf { u } _ { n o d e } ^ { n + 1 }$ by

$$
\mathbf { u } _ { n o d e } ^ { n + 1 } = \mathbf { K } \mathbf { b } _ { n o d e } ^ { n + 1 } ,\tag{9}
$$

where the node incidence matrix K is

$$
\mathbf { K } = \left[ \mathbf { K } _ { s } \quad \mathbf { K } _ { i } \quad \mathbf { K } _ { c } \right] .\tag{10}
$$

$ { \mathbf { b } } _ { s } ^ { n + 1 }$ is the vector of independent voltage/current sources, $\mathbf b _ { h } ^ { n + 1 }$ is the vector of branch history currents and $\mathbf { b } _ { c } ^ { n + 1 }$ is the component history sources. ${ \bf K } _ { s } , \ { \bf K } _ { i }$ and $\mathbf { K } _ { c }$ are respectively node incidence matrices corresponding to $\mathbf b _ { s } ^ { n + 1 } , \mathbf b _ { h } ^ { n + 1 }$ and $\mathbf { b } _ { c } ^ { n + 1 }$ . Subscripts s, i and c denote parts of the matrix corresponding to $\mathbf b _ { s } ^ { n + 1 } , \mathbf b _ { h } ^ { n + 1 }$ and $\mathbf { b } _ { c } ^ { n + 1 }$ , respectively.

Note that each entry of $\mathbf { b } _ { n o d e } ^ { n + 1 }$ is associated with only one voltage or current source, whereas, each entry of vector $\mathbf u _ { i } ^ { n + 1 }$ is a summation of current sources connected to the corresponding node. Only one task contributes to each entry of ${ \bf b } _ { n o d e } ^ { n + 1 }$ and K creates a desirable order between these entries based on (i) data dependency of each task to the previous timestep solution and (ii) the task computation time-interval. Since independent sources do not require information from previous time-step(s), they are arranged to form the first vector $\mathbf { b } _ { s } ^ { n + 1 }$ of (8). Calculation of $ { \mathbf { b } } _ { h } ^ { n + 1 }$ takes a relatively shorter time as compared to $\mathbf { b } _ { c } ^ { n + 1 }$ , therefore, establishes the second vector of (8).

Substituting for $\mathbf u _ { i } ^ { n + 1 }$ from (9) in (7),

$$
\mathbf { x } _ { n o d e } ^ { n + 1 } = \mathbf { H } _ { R } \mathbf { b } _ { n o d e } ^ { n + 1 } ,\tag{11}
$$

where $\mathbf { H } _ { R }$ is the RMNA matrix and expressed by

$$
{ \bf H } _ { R } = { \bf Y } ^ { - 1 } { \bf K } = \left[ { \bf H } _ { R s } \quad { \bf H } _ { R i } \quad { \bf H } _ { R c } \right] .\tag{12}
$$

Based on (12), contribution of each source $\mathbf { b } _ { n o d e } ^ { n + 1 } ( i )$ to $\mathbf { x } _ { n o d e } ^ { n + 1 }$ is calculated based on superposition principle. $\mathbf { b } _ { n o d e } ^ { n + 1 } ( i )$ is either a current source or a voltage source and the $i ^ { t h }$ column of $\mathbf { H } _ { R }$ determines its contribution to $\mathbf { x } _ { n o d e } ^ { n + 1 }$

The first three tasks respectively calculate $\mathbf { b } _ { s } ^ { n + 1 } , \mathbf { b } _ { h } ^ { n + 1 }$ and $\mathbf { b } _ { c } ^ { n + 1 }$ , and the fourth task calculates their contributions to $\mathbf { x } _ { n o d e } ^ { n + 1 }$ . Using (11), the fourth task can start calculating the contributions of the entries of $\mathbf { b } _ { n o d \epsilon } ^ { n + 1 }$ that were calculated earlier, and need not wait for completion of tasks 2 and 3.

The proposed reformulation offers three advantages. First, it enables fixed hardware architecture design irrespective of the connectivity of the network, since the structure of $\mathbf { b } _ { n o d e } ^ { n + 1 }$ is independent of the network configuration. Second, the number of rows of the matrix-vector multiplication is kept relatively small, therefore, a large-size system can be accommodated by FPGA. Third, it considers a proper order between vectors $\mathbf { b } _ { s } ^ { \mathsf { ^ { n + 1 } } } , \mathbf { b } _ { h } ^ { n + 1 }$ and $\mathbf { b } _ { c } ^ { n + 1 }$ in vector $\mathbf { \widehat { b } } _ { n o d e } ^ { n + 1 }$ to maximize task-level parallelism.

Next section explains RMNA formulation that can accommodate PEC models as subsystems with time-variant Y, using a new partitioning method.

## III. REAL-TIME SIMULATION OF PECS

This section presents a new network partitioning method for mathematical modeling of networks that include multiple PECs. The proposed formulation provides task-level parallelism properties discussed in the previous section and thus enables small time-step parallel solution of the network and PEC equations in real-time in an FPGA environment. The ADC switch model [7], which is widely accepted/used as the defacto method for real-time simulation of PECs, results in time-invariant Y matrix and facilitates achieving real-time simulation. However, it also results in i) spurious oscillations and ii) inaccuracies due to the artificial R, L and C components of the ADC model.

In contrast to the widely used ADC switch model, this work adopts the two-value resistor switch model to overcome the drawbacks of the ADC switch model. However, the two-value resistor model results in time-variant Y and necessitates switch status (on-off) identification. Identifying on-off status of a forced-commutated switch is straightforward, however, that of a naturally-commutated switch depends on time-evolution of a set of variables. The switch status, considering voltage/current direction convention of Fig. 2, satisfies [10]

$$
s _ { S W } ^ { n + 1 } = g _ { S W } ^ { n + 1 } + ( \bar { s } _ { S W } ^ { n } ( v _ { S W } ^ { n + 1 } < 0 ) + s _ { S W } ^ { n } ( i _ { S W } ^ { n + 1 } < 0 ) ) ,\tag{13}
$$

where $g _ { S W } ^ { n + 1 }$ is the switch gate signal at the present time-step, and is always zero for a diode switch. One can observe that in $( 1 3 ) \ s _ { S W } ^ { n + \mathbf { \breve { 1 } } }$ depends on the switch current and voltage, i.e., $v _ { S W } ^ { n + 1 }$ and $\ddot { i } _ { S W } ^ { n + 1 }$ , which themselves depend on $s _ { S W } ^ { n + 1 }$ and states of other switches. The implicit relationships of $i _ { S W } ^ { n + 1 } , ~ v _ { S W } ^ { n + 1 }$ and $s _ { S W } ^ { n + 1 }$ require iterative solution of network equations to determine the switch states after a switching event.

The iteration process can impose significant computationtime burden and prevents small time-step solution of multiple

PECs with high switching frequency. This paper overcomes limitations of the two-value resistor switch model by the proposed novel network partitioning method.

The proposed partitioning method is based on defining each PEC Switching-Network (PECSN) as a subsystem and we refer to it as Switching-Network Partitioning (SNP) method. The rest of the PEC components, i.e., capacitors and inductors, are treated as conventional branches. The proposed SNP method comprises

• Stage 1: Obtaining the equivalent circuit of the system external to each PECSN,

• Stage 2: Obtaining the PECSN terminal variables from the equivalent circuit,

• Stage 3: Representing the PECSN in the solution of network equations by sources equal to values of terminal variables.

The following subsections describe the procedures to carry out Stage 1 and Stage 2 for a system that embeds only one PEC and thus one PECSN. Since for such a case the rest of the system features time-invariant Y, Stage 3 is to represent the PEC as a component in the network as was explained in Section II. Last subsection generalizes the method for multiple PECs.

## A. Stage 1: Solution of external equivalent circuit

Consider the system of Fig. 3(a) which includes one PEC and thus one PECSN. Figure 3(b) shows the discretized model of the circuit external to the PECSN. The external circuit model can be expressed by

$$
\begin{array} { r } { \left[ \mathbf { \check { v } } ^ { n + 1 } \right] = \left[ \mathbf { \check { v } } _ { e q } ^ { n + 1 } \right] + \mathbf { Y } _ { N e q } ^ { - 1 } \left[ \mathbf { \check { i } } _ { o } ^ { n + 1 } \right] , } \\ { \left[ \mathbf { \check { i } } ^ { n + 1 } \right] = \left[ \mathbf { \check { i } } _ { e q } ^ { n + 1 } \right] + \mathbf { Y } _ { N e q } ^ { - 1 } \left[ \mathbf { \check { v } } _ { o } ^ { n + 1 } \right] , } \end{array}\tag{14}
$$

where $\mathbf { i } _ { o } ^ { n + 1 }$ and $\mathbf { v } _ { o } ^ { n + 1 }$ are the PEC terminal variables, in+1 and $\mathbf { v } ^ { n + 1 }$ are the network variables. $\mathbf { v } _ { e q } ^ { n + 1 }$ and $\mathbf { i } _ { e q } ^ { n + 1 }$ are the equivalent sources, and ${ \bf Y } _ { N e q }$ is the admittance matrix. Vector $\left[ \mathbf { \check { v } } _ { e q } ^ { n + 1 } \mathbf { i } _ { e q } ^ { n + 1 } \right] ^ { T }$ can be obtained by

$$
\mathbf b _ { e q } ^ { n + 1 } = \left[ \mathbf v _ { e q } ^ { n + 1 } \right] = \mathbf H _ { e q } \mathbf b _ { n o d e _ { e x t } } ^ { n + 1 } ,\tag{15}
$$

where $\mathbf { b } _ { n o d e _ { e x t } } ^ { n + 1 }$ is the RMNA injection vector of the external circuit, i.e,

$$
\mathbf b _ { n o d e _ { e x t } } ^ { n + 1 } = \left[ \mathbf b _ { s } ^ { n + 1 } \mathbf b _ { h } ^ { n + 1 } \mathbf { 0 } \right] ^ { T } .\tag{16}
$$

Matrices ${ \bf H } _ { e q }$ and ${ \bf Y } _ { N e q } ^ { - 1 }$ are time-invariant and determined by directly applying node elimination to Y. It should be noted that these matrices represent the equivalent circuit of the overall system external to the PECSN. This is in contrast to [27] where the equivalent circuit of each sub-system is obtained in a block matrix form, known as Multi-Port Hybrid Equivalent (MPHE), and used to formulate system equations.

The equivalent of the external circuit is characterized by ${ \bf Y } _ { N e q }$ and vector $\mathbf { b } _ { e q } ^ { n + 1 }$ which, according to (15), is a function of $\mathbf { b } _ { n o d e _ { e x t } } ^ { n + 1 }$ . If we assume source outputs remain unchanged in each time-step (one time-step delay), then $\mathbf { b } _ { s } ^ { n + 1 } = \mathbf { b } _ { s } ^ { n }$ and therefore $\mathbf { b } _ { n o d e _ { e x t } } ^ { n + 1 }$ can be calculated prior to time-step $n + 1$ i.e., at time-step n, using

$$
{ \bf b } _ { n o d e _ { e x t } } ^ { n + 1 } = { \bf C } _ { h } { \bf b } _ { n o d e } ^ { n } + { \bf C } _ { v } { \bf x } _ { n o d e } ^ { n } ,\tag{17}
$$

where

$$
{ \bf C } _ { h } = d i a g \left[ { \bf I } \mathrm { ~ } { \bf { c } } _ { h } \mathrm { ~ } { \bf { 0 } } \right] ,\tag{18}
$$

and

$$
\mathbf { C } _ { v } = \left[ \mathbf { 0 } \quad \mathbf { c } _ { v } \mathbf { K } _ { i } \quad \mathbf { 0 } \right] ^ { T } .\tag{19}
$$

$\mathbf { c } _ { h }$ and $\mathbf { c } _ { v }$ are diagonal matrices and their entries are coefficients of each branch defined in (4). Substituting for $\smash {  { \mathbf { b } } _ { m e d e } ^ { n + 1 } }$ nodeext from (17) in (15),

$$
\mathbf { b } _ { e q } ^ { n + 1 } = \mathbf { H } _ { e q } ( \mathbf { C } _ { h } + \mathbf { C } _ { v } \mathbf { H } _ { R } ) \mathbf { b } _ { n o d e } ^ { n } ,\tag{20}
$$

or equivalently

$$
\mathbf { b } _ { e q } ^ { n + 1 } = \mathbf { H } _ { e q R } \mathbf { b } _ { n o d e } ^ { n } ,\tag{21}
$$

where

$$
\mathbf { H } _ { e q R } = \mathbf { H } _ { e q } ( \mathbf { C } _ { h } + \mathbf { C } _ { v } \mathbf { H } _ { R } ) .\tag{22}
$$

Since $\mathbf { b } _ { e q } ^ { n + 1 }$ is a function of $ { \mathbf { b } } _ { n o d e } ^ { n }$ , it can be computed in parallel with $\mathbf { x } _ { n o d e } ^ { n }$ which is enabled by incorporating matrix ${ \bf { H } } _ { e q R }$ in the existing task of (11), i.e.,

$$
\begin{array} { r } { \left[ \mathbf { \Delta } \mathbf { { x } } _ { n o d e } ^ { n } \right] = \left[ \mathbf { \Delta } \mathbf { { H } } _ { R } \right] \mathbf { b } _ { n o d e } ^ { n } . } \\ { \left[ \mathbf { b } _ { e q } ^ { n + 1 } \right] = \left[ \mathbf { \Delta } \mathbf { { H } } _ { e q R } \right] \mathbf { \Delta } \mathbf { b } _ { n o d e } ^ { n } . } \end{array}\tag{23}
$$

We refer to (23) as Matrix-Vector Multiplication (MVM) task. Based on (23), the equivalent circuit external to PECSN for time-step n+1 is deduced at time-step n, i.e., at the beginning of each time-step, thus Stage 1 of the SNP method is already completed which enables task-level parallelism. It should be noted that assuming one-time step delay for the independent sources does not significantly affect the accuracy, since the time-step is relatively small, as compared to the independent source periods. It also does not impact simulation stability since $\mathbf { b } _ { s } ^ { \bar { n } + 1 }$ is an input signal.

## B. Stage 2: Calculation of PECSN terminal variables

Stage 2 comprises interfacing the equivalent circuit obtained at Stage 1 with the PECSN model and solving the resulting circuit as of Fig. 3(c). At this stage, in contrast to Stage 1 which is carried out by the existing MVM task, we define a new task and refer to it as “PEC simulation $t a s k ^ { \prime }$ . To solve the system of Fig. 3(c) we derive the equations that relate the external circuit parameters ${ \bf Y } _ { N e q }$ and $\bar { \mathbf { b } _ { e q } ^ { n + 1 } }$ to the PECSN terminal variables and switch currents/voltages. Since current and voltage of each two-value resistor model are proportional, only switch current variables are used. The equations of PECSN, using MNA formulation, are

$$
\mathbf { Y } _ { \mathit { C o n v } } ( \mathbf { s } ^ { n + 1 } ) \left[ \mathbf { \bar { i } } _ { o } ^ { n + 1 } \right] = \left[ \mathbf { \bar { i } } ^ { n + 1 } \right] ,\tag{24}
$$

where $\mathbf { Y } _ { C o n v } ( \mathbf { s } )$ is the admittance matrix corresponding to the switch state vector s. If a switch is on (off), then its corresponding entry in the vector s is 1 (0). From (14) and (24), PECSN terminal variables at switch state $\mathbf { s } ^ { n + 1 }$ are

$$
\mathbf b _ { c } ^ { n + 1 } = \left[ \mathbf { i } _ { o } ^ { n + 1 } \right] = \mathbf { H } _ { C } ( \mathbf s ^ { n + 1 } ) \mathbf { b } _ { e q } ^ { n + 1 } ,\tag{25}
$$

where

$$
\mathbf { H } _ { C } ( \mathbf { s } ^ { n + 1 } ) = ( \mathbf { Y } _ { C o n v } ( \mathbf { s } ^ { n + 1 } ) - \mathbf { Y } _ { N e q } ^ { - 1 } ) ^ { - 1 } .\tag{26}
$$

$\mathbf { b } _ { c } ^ { n + 1 }$ represents PECSN in the network. Switch currents are

$$
\mathbf i _ { S W } ^ { n + 1 } = \mathbf M _ { S W } ( \mathbf s ^ { n + 1 } ) \mathbf b _ { c } ^ { n + 1 } ,\tag{27}
$$

where $\mathbf { M } _ { S W } ( \mathbf { s } )$ is the PECSN switch incidence matrix multiplied by the switch admittances. From (27) and (25)

$$
\mathbf i _ { S W } ^ { n + 1 } = \mathbf H _ { S W } ( \mathbf s ^ { n + 1 } ) \mathbf b _ { e q } ^ { n + 1 } ,\tag{28}
$$

where

$$
\mathbf { H } _ { S W } ( \mathbf { s } ^ { n + 1 } ) = \mathbf { M } _ { S W } ( \mathbf { s } ^ { n + 1 } ) \mathbf { H } _ { C } ( \mathbf { s } ^ { n + 1 } ) .\tag{29}
$$

Equations (25) and (28) relate $\mathbf { b } _ { e q } ^ { n + 1 }$ to the PECSN terminal voltages/currents and switch currents, respectively, and the PEC simulation task is carried out as follows.

Firstly, the switch states are determined. Using (13), multiple iterative solution of (1) might be required in one timestep to determine all switch states [21]. The reason is that a switch state transition, e.g., a change in a gate signal, can affect the state of other switches. These effects are detected after (1) is solved. We propose to determine the state of PECSN switches as one entity, rather than examining state of each switch separately. This avoids costly iterative solution of (1) and determines the switch states after an inexpensive iteration of (28), based on the following:

• It is assumed that $\mathbf { s } ^ { n + 1 }$ is affected by two factors, i.e., (i) switch gate signal vector $\mathbf { g } ^ { n + 1 }$ and (ii) electrical state variables of the external circuit. ${ \bf Y } _ { N e q }$ and $\mathbf { b } _ { e q } ^ { n + 1 }$ represent the net effect of these variables on PECSN.

• A combined switch and diode pair is assumed as one switch, therefore, asserting a gate signal implies that the switch pair is ON regardless of its current direction.

$\mathbf { g } ^ { n + 1 }$ is used as the initial guess for $\mathbf { s } ^ { n + 1 }$ . The temporary switch currents are determined by

$$
\mathbf i _ { S W t } ^ { n + 1 } = \mathbf H _ { S W } ( \mathbf g ^ { n + 1 } ) \mathbf b _ { e q } ^ { n + 1 } .\tag{30}
$$

Based on the current convention of Fig. 2, switch states are

$$
\mathbf { s } ^ { n + 1 } = ( \mathbf { i } _ { S W t } ^ { n + 1 } < 0 ) | | \mathbf { g } ^ { n + 1 } .\tag{31}
$$

Entries of $\mathbf { s } ^ { n + 1 }$ are not treated as state variables and $\mathbf { g } ^ { n + 1 }$ is used as an initial guess, therefore, a switch is on only if it is a forced-commutated switch and its gate signal is asserted. This will enforce current through the natural-commutated switches and (31) detects these commutations. Solving (30) is relatively inexpensive since the dimensions of ${ \mathbf { H } } _ { S W }$ are very small compared to Y. Once the switch states are known, $\dot { \mathbf { b } } _ { c } ^ { n + 1 }$ is obtained from (25) and is used to represent the PECSN in the system as Fig. 3(d) shows. This completes PEC simulation task. To summarise:

1) calculate switch current vector $\mathbf { i } _ { S W t } ^ { n + 1 }$ from (30);

2) determine switch states from (31);

3) determine PEC output variables $\mathbf { b } _ { c } ^ { n + 1 }$ from (25).

## C. Generalization to multiple PECs

Matrices ${ \bf Y } _ { N e q }$ and ${ \bf { H } } _ { e q R }$ of each PECSN, in a system embedding multiple PECs, are theoretically functions of switch states of other PECSNs. In a given time-step each entry of $\mathbf { x } _ { n o d e } ^ { n + 1 }$ of (1), i.e., network electrical variables, affects the value of other entries i) implicitly, i.e., in the same timestep as represented by the Y matrix, and ii) explicitly, i.e., in the subsequent time-steps as represented by history current sources. We assume that the external system of a PECSN explicitly changes due to the change of switch status in other PECSNs. The justification is that PECs are intentionally designed/deployed to exhibit independent switching process with respect to each other. Thus we can assume ${ \bf Y } _ { e q N } , { \bf H } _ { e q }$ and therefore ${ \bf { H } } _ { e q R }$ matrices of each PECSN remain time-invariant in each time-step. To achieve this, in the process of node elimination for each PECSN, other PECSN are replaced by zero voltage/current sources. Hence for the generalized SNP method of a system including m PECs

$$
\mathbf b _ { c } ^ { n + 1 } = \left[ \mathbf b _ { c 1 } ^ { n + 1 } . . . \mathbf b _ { c m } ^ { n + 1 } \right] ^ { T } ,\tag{32}
$$

where $\mathbf { b } _ { c j } ^ { n + 1 }$ represents the $j ^ { t h }$ PECSN. ${ \bf H } _ { e q }$ corresponding to each PECSN is the same as (16). For such a system $\mathbf { b } _ { e q } ^ { n + 1 }$ and ${ \bf { H } } _ { e q R }$ of (23) are also defined by

$$
\mathbf b _ { e q } ^ { n + 1 } = \left[ \mathbf b _ { e q 1 } ^ { n + 1 } . . . \mathbf b _ { e q m } ^ { n + 1 } \right] ^ { T } ,\tag{33}
$$

and

$$
\mathbf { H } _ { e q R } = \left[ \begin{array} { c } { \mathbf { H } _ { e q R 1 } } \\ { \vdots } \\ { \mathbf { H } _ { e q R m } } \end{array} \right] .\tag{34}
$$

Each $\mathbf { b } _ { c i } ^ { n + 1 }$ is obtained by conducting a PEC simulation task, using $\mathbf { b } _ { e q j } ^ { \breve { n } + 1 }$ , and used to represent the PECSN in the solution of the overall network , i.e., (23). All PEC simulation tasks are conducted in parallel, assuming time-invariant ${ \bf Y } _ { e q N j }$ and $\mathbf { H } _ { e q j }$ matrices.

Configuration of a number of switches that are grouped as one PECSN is important for this generalization to be applicable. The generalization holds and the switching process of these switches is independent of the ones in the other PECSNs if these switches are part of one PEC deployed separately or there exist energy storage elements, i.e., capacitors and inductors, between them and the ones in the other PECSNs. For instance, switches in a two-level VSC or each leg of a neutral-point clamped converter lend themselves well to be considered as one PECSN. However, in the case of a modular multi-level PEC, each sub-module cannot be considered as one PECSN, since the switching networks of the sub-modules are directly cascaded. In such cases the switches that have dependent switching process should be considered as one PECSN.

## IV. FPGA-BASED DESIGN

This section presents design of i) FPGA-based modules to carry out tasks of Section II-A, and ii) the DRTS platform using Xilinx’s VC707 board that houses a VC7VX485T chip. Vivado High Level Synthesis (HLS) was used to design the simulation task modules, based on single-precision floatingpoint numerical representation. VHDL coding was used to design the accessory modules. Then, all modules were integrated in the Vivado environment.

To avoid trigonometric calculation on FPGA, a Block RAM (BRAM) containing 1200 sample points of one period of a sine wave is considered for representation of each AC source. BRAMs are abundant in FPGA and provide adequate precision for sine wave calculation in real-time. The independent source calculation module outputs $\mathbf { b } _ { s } ^ { n + 1 }$ at the rate of one entry per clock cycle.

## VI. CONCLUSION

This paper proposes Reformulated Modified Nodal Analysis (RMNA) network equations, tailored for FPGA-based realtime simulation, featuring solution parallelism of the network and component models. It introduces the Switching-Network Partitioning (SNP) method to overcome the challenges associated with the two-value resistor model for FPGA-based realtime simulation. The method is generalized for application to systems including multiple converters. This paper also presents the FPGA-based design and implementation of the envisioned digital real-time simulation platform with a fixed structure which also enables complete task-level parallelism. The developed prototype real-time simulator can simulate systems embedding up to 6 PECs and up to 100 outputs (nodes), with the time-step of 0.8µs. Performance and accuracy of the proposed platform are compared with and verified versus off-line simulation case studies. To accommodate other power system component models in the SNP method, it is important to adopt modeling approaches that retain the constant parameter feature of the external circuit to PEC Switching Networks (PECSNs). As an example Induction Machines (IMs) are simulated using constant parameter Voltage Behind Reactance (VBR) modeling approach.
